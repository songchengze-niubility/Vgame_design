# 吞吐量改变合并理念

> 纠错成本低，等待成本高。

在智能体辅助设计和配置分析中，产出速度会明显高于传统流程。为避免大改动堆积，Vgame 采用小步提交和短反馈循环。

## 实践规则

1. 每次提交聚焦一个主题。
2. 先跑 `scripts/check.ps1`。
3. 文档与规则同步更新。
4. 临时判断进入 `tech-debt-tracker.md`。
5. 发现错误快速修正，不把错误藏进大分支。

## 提交前 Checklist

- [ ] 门禁脚本通过。
- [ ] 变更范围清楚。
- [ ] 相关 design/proposal/task 已更新。
- [ ] 新增坑点或低置信结论已登记。
- [ ] 若涉及真实配置，已记录验证方式和回滚方式。
