# 仓库即记录系统

> 不在仓库里的东西，对智能体不存在。

Vgame 的设计判断不能只存在于聊天记录、个人记忆或临时 Excel 标注里。只要它会影响后续设计、配置、数值或上线判断，就应该进入仓库。

## 必须记录的内容

- 系统设计和重大规则变更。
- 配置链路、字段含义和引用关系。
- 经济产销判断与风险结论。
- 战斗强度标杆、模型权重和校准结论。
- 临时方案、历史坑点、未确认字段。
- 跨仓变更记录。

## 记录位置

| 内容 | 位置 |
|---|---|
| 系统设计 | `design/` |
| 落地方案 | `proposals/` |
| 执行拆解 | `tasks/` |
| 规则与门禁 | `harness/` |
| 债务与坑点 | `tech-debt-tracker.md` |
| 跨仓记录 | `CROSS-REPO-CHANGES.md` |

## 决策记录模板

```markdown
# ADR-XXXX: 标题

**状态:** 提议 / 采纳 / 废弃
**日期:** YYYY-MM-DD
**背景:** 为什么需要这个决策
**决策:** 我们决定怎么做
**理由:** 为什么这样做
**影响:** 影响哪些系统、配置表、skill 或流程
```
