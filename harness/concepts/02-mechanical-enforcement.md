# 机械化执行

> 文档会腐烂，脚本不会忘。

只靠“记得看规范”不可靠。Vgame design harness 要把能自动检查的内容放进 `scripts/check.ps1`。

## 初版检查层级

| 层级 | 检查内容 | 当前状态 |
|---|---|---|
| L1 结构检查 | 入口文档、模板、目录存在 | 已接入 |
| L2 文档检查 | 待办标记、过大文档、模板缺失 | 已接入基础版 |
| L3 skill 检查 | skill 目录、config.toml、快速校验 | 待扩展 |
| L4 配置审查 | 表注册、字段引用、奖励/解锁闭环 | 待扩展 |

## 原则

- 结构缺失是失败。
- 可疑内容先警告，不直接阻断。
- 每条失败信息要说明修复方向。
- 脚本先轻量，随真实流程逐步增强。
