# 熵管理

> 技术债是高息贷款，设计债也是。

Vgame 的配置、角色、成长、奖励和开放链路都可能随版本积累熵。智能体会模仿仓库里的模式，因此坏模式必须被及时标记和回收。

## 熵增来源

- 紧急配置修正没有记录原因。
- 历史玩法、草稿玩法和正式玩法混在一起。
- skill 里的经验没有跟随真实配置更新。
- 数值模型权重没有标杆校准。
- 表字段含义靠口头记忆传递。

## 回收流程

```text
扫描 -> 发现偏差 -> 登记债务 -> 形成 proposal/task -> 修复 -> 更新 skill 与文档
```

## 周期建议

| 项目 | 频率 |
|---|---|
| 门禁检查 | 每次提交 |
| 质量评分 | 每个版本或迭代 |
| skill 经验回写 | 每次重要分析后 |
| 配置链专项审查 | 每次系统改动前后 |
| 战斗/经济标杆校准 | 每轮大平衡调整后 |
