# Vgame Design Coding And Script Standards

本仓库目前以 Markdown 和轻量脚本为主。若后续加入分析脚本，遵循以下规则。

## 通用规则

- 脚本默认只读，写入脚本必须在文件头说明输出位置和影响范围。
- 输入路径、输出路径、核心参数必须可配置。
- 不把真实配置覆盖写回源目录，除非任务明确要求。
- 错误信息要说明修复方向。
- 分析结论要写明数据来源和模型假设。

## PowerShell

- 使用 `-LiteralPath` 处理中文路径。
- 脚本开始设置 UTF-8 输出。
- 对外部目录缺失使用警告，不直接崩溃，除非该目录是本仓库必需结构。

## Python

- 读取 Excel 优先使用稳定库，例如 `openpyxl` 或 `pandas`。
- 表结构解析优先使用表头和 schema，不手写脆弱字符串规则。
- 输出文件放到明确的 `output/` 或任务指定目录。
- 对百分比、倍率、持续时间、冷却等口径必须显式标注。

## Markdown

- 标题层级不要跳跃。
- 表格列名使用中文。
- 重要结论标注置信度和来源。
- 不把大段聊天记录原样贴入长期文档。
