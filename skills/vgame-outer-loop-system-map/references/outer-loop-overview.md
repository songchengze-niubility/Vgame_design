# 外循环系统总览

## 总体关系

Vgame 的外循环不是主战斗玩法本身，而是组织玩家留存、资源补充、box 扩展和商业化支撑的系统层。

默认闭环：

```text
主线推进
-> 系统开放
-> 日常/周常/活动/商店/签到/战令等外循环
-> 资源与抽卡补充
-> 角色、专武、旅券、映射装备成长
-> 正式玩法验证
-> 回到留存与长期目标
```

## 职责分层

| 职责 | 系统 |
|---|---|
| 日常留存 | 每日任务、每日活跃、签到、扫荡、日常资源副本。 |
| 周期留存 | 每周任务、周活跃、战令、竞技场/爬塔/大秘境赛季奖励。 |
| 活动留存 | 活动入口、活动任务、活动活跃、双倍掉落、限时投放。 |
| 资源补充 | 任务、签到、战令、邮件、商店、玩家升级、活动奖励。 |
| box 扩展 | 抽卡、新手卡池、专武抽卡、抽卡券投放。 |
| 商业化支撑 | 商店、月卡、战令付费档、礼包、快捷购买、付费商品。 |
| 社交留存 | 公会、好友、聊天。 |
| 补偿/运营 | 邮件、预发邮件、限时活动。 |

## 外循环与正式玩法的关系

- 外循环提供资源，不直接替代主线和验证玩法。
- 抽卡和专武负责 box breadth 与机制深度。
- 商店和战令是资源补充与商业化承接。
- 签到、任务、活动负责留存节奏。
- 邮件和补偿是一次性投放，不能纳入常态日收入。
- 社交系统主要看留存和协作，不默认承担核心成长供给。

## 边界

- 单个 ItemId 的产销细查交给 `vgame-economy-source-map`。
- DropId 内容展开交给 `vgame-reward-drop-sync`。
- 系统开放和 Unlock 交给 `vgame-level-progression-map`。
- 价格、奖励价值、付费压力判断交给 `senior-game-economy`。
