# 周期与商业化关系

## 周期分类

| 周期 | 常见系统 | 审查重点 |
|---|---|---|
| 一次性 | 主线奖励、章节奖励、玩家升级奖励、部分邮件、首通奖励 | 不能计入长期日收入。 |
| 日常 | 每日任务、每日活跃、每日签到、月卡每日奖励、日常资源副本、扫荡 | 影响基础留存和稳定供给。 |
| 周常 | 每周任务、周活跃、周奖励 | 影响中周期目标和资源节奏。 |
| 赛季 | 战令、竞技场、爬塔、大秘境 | 需要确认赛季长度、重置和排行奖励。 |
| 活动 | 活动活跃、14 日活动、双倍掉落、限时投放 | 需要确认 `Activity.AvyId` 和活动时间。 |
| 刷新 | 商店刷新、限购重置 | 需要确认 `Shop.RefreshType`, `LimitType`, `LimitNum`。 |
| 付费 recurring | 月卡、战令付费档 | 需要确认真实发奖字段和展示字段一致性。 |

## 商业化敏感点

| 系统 | 敏感原因 |
|---|---|
| 抽卡 | 直接连接抽卡券、元晶、卡池、UP、保底和 box 扩展。 |
| 快捷购买 | 抽卡或资源不足时可能把付费货币转化为关键资源。 |
| 商店 | 可能同时消耗免费货币、付费货币、活动货币和兑换币。 |
| 月卡 | 长期稳定供给，必须区分展示字段和真实发奖字段。 |
| 战令 | 免费线和付费线同时存在，会影响赛季经济。 |
| 活动礼包/付费商品 | 大额一次性投放和限时压力。 |
| 邮件/补偿 | 一次性大额投放，不应当视为常态收入。 |

## 外循环风险

- 高频系统投放高价值资源，会压缩付费和长期追求。
- 活动投放如果与常驻投放叠加，容易造成短期超发。
- 预览字段和真实奖励字段不一致，会造成展示和实际发奖风险。
- 商店无限购或刷新周期过短，会把原本受控资源变成可购买资源。
- 抽卡券、元晶、专武抽卡券等资源必须单独标高风险。
- 双倍掉落会改变重复产出效率，不能只看基础 Drop。

## 审查问题

做外循环分析时至少回答：

- 这个系统是一次性、日常、周常、赛季、活动，还是付费 recurring？
- 真实发奖字段是什么，展示字段是什么？
- 奖励是否经 DropId 间接发放？
- 是否涉及付费货币、抽卡券、月卡、战令或礼包？
- 是否有 Unlock/SystemOpen/Activity 时间控制？
- 是否存在刷新、限购、补领、过期或排行条件？
