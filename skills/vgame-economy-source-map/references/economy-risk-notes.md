# Economy Risk Notes

## 常见坑

| 坑 | 正确处理 |
|---|---|
| 把 `Items.AccessList` 当真实产出 | 只当展示途径和跳转线索，真实产出必须查 Drop/任务/活动/商店等表。 |
| 把 UI 预览当真实奖励 | `UIlevel.Reward/FirstReward`、`RewardItemPreview`、`PassRewardPreview` 多为展示或预览，真实奖励要查 DropId 或直接 PropType。 |
| 只查一个 Drop 文件 | 逻辑 `Drop` 由多个文件合并：draw、dungeon、mainlevel、other、GiftBox、TimeLimitedActivity、world、BattlePass。 |
| 把 `Drop.IsValid` 当有效性判断 | Drop 行里的 `IsValid` 是无效/占位字段，产销扫描和奖励展开不要按它过滤。 |
| 混淆系统入口和层级解锁 | `SystemOpen.UnlockId` 管系统入口，`UIlevel.Unlock` 管层级/关卡可见或进入。 |
| 混淆体力字段 | `level.Vitality` 和 `UIlevel.Vitality` 可能不同，真实消耗需要结合运行逻辑确认。 |
| 忽略周期 | 日常、周常、通行证、活动、双倍掉落、排行奖励都必须确认周期和领取次数。 |
| 忽略付费敏感性 | 元晶、抽卡价格、月卡、人民币商品、混合价格商品必须单独标高风险。 |
| 忽略同名/旧系统 | 肉鸽、能效本、旧装备本等可能还留表，不要当正式系统事实，除非任务明确要求。 |

## 高风险资源

| 资源/系统 | 风险点 |
|---|---|
| 元晶 | 付费、抽卡、商店、活动赠送都会影响商业化和玩家信任。 |
| 抽卡券/抽卡消耗 | 与卡池 Drop、保底、UP 池、快捷购买联动。 |
| 储能/体力 | 影响日常可玩时长、付费补充、资源副本效率。 |
| 金币 | 基础成长刚需，过多会失去消耗压力，过少会卡养成。 |
| 映射装备和随机装备资源 | 随机深度强，需区分保底、分解、商店和掉落效率。 |
| 通行证奖励 | 同时涉及免费线、付费线、等级经验和周期。 |
| 双倍掉落活动 | 直接改变重复产出效率，应按活动周期折算。 |
| 邮件/补偿 | 容易形成一次性大额投放，不能纳入常态日收入。 |

## 经济判断边界

这个 skill 可以指出“哪里产出、哪里消耗、哪里有风险”，但不要直接说“数值合理/不合理”。

判断合理性时读取：

- `senior-game-economy`：经济结构、奖励价值、长线节奏、商业化压力。
- `game-numerical-analysis`：日/周/月产出量、消耗量、概率期望、成长曲线。

## 建议提问模板

- “某个 ItemId 的已观察来源和消耗口有哪些？”
- “某资源在日常、周常、活动、付费中分别从哪里来？”
- “某个商店商品的产出道具和价格消耗怎么追？”
- “某个 DropId/RewardId 是哪些表引用的，内容在哪展开？”
- “某资源是否涉及付费、抽卡、体力或周期性重复供给风险？”
