# Vgame 战斗属性与战斗计算规范

基于 [Vgame基本战斗框架.xlsx](${VGAME_ROOT}/策划/数值组/2026-03/Vgame基本战斗框架.xlsx) 中“战斗属性”和“战斗公式”两个页签整理。

本文档定位为 Vgame 当前版本的战斗规则正式整理稿，面向数值策划、战斗策划、程序和技术设计共同使用。

## 1. 文档目的

- 统一 Vgame 当前战斗属性和战斗结算口径
- 作为后续数值设计、技能设计、平衡验证的基础参考
- 降低多人协作时对 Excel 原表逐条翻读的成本
- 作为 Codex 处理 Vgame 战斗相关问题时的优先依据

## 2. 体系概览

Vgame 当前战斗体系具备以下特征：

- 属性体系按“基础属性、战斗属性、百分比成长属性、等级属性”分层
- 战斗外成长与战斗内 buff 分层计算
- 伤害流程采用固定顺序：命中 -> 伤害类型 -> 基础伤害 -> 暴击 -> 格挡 -> 最终伤害 -> 护盾/生命结算
- 所有标签型伤害加成共用同一乘区，先求和，再统一参与最终伤害
- 真实伤害独立于普通伤害公式，不受护盾影响

## 3. 属性体系定义

### 3.1 基础属性

- 生命（`Hp`）
- 物理攻击（`PhysicsAttack`）
- 魔法攻击（`MagicAttack`）
- 防御力（`Armor`）

说明：

- 基础属性是角色战斗能力的核心底层输入
- 生命决定生存
- 物攻、魔攻决定伤害基数来源
- 防御决定普通伤害的减伤衰减

### 3.2 战斗属性

- 能量上限（`Energy`）
- 能量初始值（`EnergyPreAdd`）
- 能量回复速度（`EnergyPreAdd`）
- 暴击（`Crit`，配置表中文为“暴击率”）
- 暴击伤害（`CritDmgBonus`）
- 闪避（`Miss`）
- 命中（`Hit`）
- 效果命中（`EffectHit`）
- 效果抵抗（`EffectResist`）
- 无视防御（`ArmorBreak`）
- 普攻攻击速度（`AttackSpeed`）
- 伤害加成（`DmgBonus`）
- 物理伤害加成（`PhysicsDmgBonus`）
- 魔法伤害加成（`MagicDmgBonus`）
- 普攻伤害加成（`NormalAttackDmgBonus`）
- 技能伤害加成（`SkillDmgBonus`）
- 必杀伤害加成（`UltimateDmgBonus`）
- 单体伤害加成（`SingleTargetDmgBonus`）
- 穿透伤害加成（`PenetrationDmgBonus`）
- 范围伤害加成（`AOEDmgBonus`）
- 持续伤害加成（`DurationDmgBonus`）
- 流血伤害加成（`BleedDmgBonus`）
- 灼烧伤害加成（`BurnDmgBonus`）
- 中毒伤害加成（`PoisonDmgBonus`）
- 伤害减免（`DmgImmune`）
- 治疗加成（`Heal`，配置表中文为“治疗效率”）
- 受到伤害增加（`DmgTakeBonus`，配置表中文为“受伤加成”）
- 受治疗加成（`CauseHeal`，配置表中文为“受到治疗效果”）
- 施加护盾加成（`ShieldCastBonus`）
- 受到护盾加成（`ShieldGainBonus`）
- 释放盾牌加成（`BucklerHpBonus`）
- 格挡（`Block`，配置表中文为“格挡概率”）
- 格挡等级（`BlockLevel`）
- 格挡强度（`BlockImmune`）
- 破击（`BlockBreak`，配置表中文为“破击/破格挡”）
- 冷却缩减（`CoolDownReduce`）
- 玩法伤害加成（`ExtraDmgBonus`）
- 玩法伤害减免（`ExtraDmgImmune`）

当前版本特别说明：

- `效果命中`、`效果抵抗` 暂时不使用，可忽略
- `能量回复速度` 定义为角色每秒获得的能量值
- `普攻攻击速度` 定义为角色每秒发射的普攻次数
- 例如攻击速度为 `5`，表示角色每秒可进行 `5` 次普攻

### 3.3 百分比成长属性

- 生命%（`HpBonus`）
- 物理攻击%（`PhysicsAttackBonus`）
- 魔法攻击%（`MagicAttackBonus`）
- 防御力%（`ArmorBonus`）
- 攻击速度%（`AttackSpeedBonus`）
- 能量回复%（`EnergyPreAddBonus`，配置表中文为“能量回复效率”）

说明：

- 百分比成长属性主要作用于战斗外成长阶段
- 它们与战斗内 buff 百分比不是同一层计算

### 3.4 等级属性

- 暴击等级（`CritLevel`）
- 暴击伤害等级（`CritDmgBonusLevel`）
- 能量回复等级（`EnergyPreAddBonusLevel`）
- 技能冷却等级（`CoolDownReduceLevel`）
- 格挡等级（`BlockLevel`）

统一转换规则：

`加成属性 = 等级属性 / 转化参数`

示例：

- 暴击等级 = 100
- 转化参数 = 5000
- 暴击等级提供的暴击率 = 2%

## 4. 战斗外属性生成规则

### 4.1 同类属性合并

战斗外来自不同系统模块的同一种属性，统一按加法合并。

示例：

- 模块 A 提供 10% 暴击
- 模块 B 提供 20% 暴击
- 最终合并为 30% 暴击

### 4.2 裸体属性

#### 生命

`裸体生命 = [初始生命 + 等级生命 + 星盘生命 + 突破生命] * 当前品阶系数`

#### 攻击

物理攻击与魔法攻击共用同一计算结构：

`裸体攻击 = [初始攻击 + 等级攻击 + 星盘攻击 + 突破攻击] * 当前品阶系数`

#### 防御

`裸体防御 = [初始防御 + 等级防御 + 星盘防御 + 突破防御] * 当前品阶系数`

### 4.3 面板属性

#### 生命

`面板生命 = 裸体生命 * (1 + 各来源百分比生命和) + 各来源固定生命和`

#### 攻击

`面板攻击 = 裸体攻击 * (1 + 各来源百分比攻击和) + 各来源固定攻击和`

#### 防御

`面板防御 = 裸体防御 * (1 + 各来源百分比防御和) + 各来源固定防御和`

#### float 类面板属性

暴击、暴伤、命中、闪避、无视防御、伤害加成、伤害减免、治疗加成等 float 属性，统一按加法合并：

`面板属性 = 基础值 + 装备 + 宝石 + 天赋 + 其他来源`

#### 能量回复速度

`最终能量回复速度 = 基础能量回复速度 * (1 + 能量回复速度%)`

补充口径：

- 除自然回复外，角色还可以通过关卡中捡拾能量球获取能量

## 5. 战斗内属性计算规则

### 5.1 总原则

- 战斗外先形成入场初始值
- 战斗内再由 buff 实时改写
- 战斗外百分比成长和战斗内百分比 buff 严格分层

### 5.2 int 属性

战斗内 int 属性统一公式：

`最终属性 = 入场初始属性 * (1 + 所有 buff 乘法值之和) + 所有 buff 加法值之和`

示例：

- 战斗外攻击 = 1000
- 战斗外攻击% = 50%
- 入场初始攻击 = 1500
- 战斗内再获得 20% 攻击 buff 和 1000 固定攻击 buff
- 最终攻击 = 1500 * 1.2 + 1000 = 2800

### 5.3 float 属性

战斗内 float 属性统一公式：

`最终属性 = 入场初始属性 + 所有 buff 属性之和`

示例：

- 战斗外暴击 = 50%
- 战斗内再获得 20% 暴击 buff
- 最终暴击率 = 70%

## 6. 伤害结算规范

### 6.1 流程总览

普通伤害按如下顺序结算：

1. 判定命中
2. 判定伤害类型
3. 计算基础伤害
4. 判定暴击
5. 判定格挡
6. 计算最终伤害
7. 护盾吸收与生命扣减

### 6.2 命中判定

命中概率公式：

`命中概率 = 攻击者命中 - 受击者闪避`

规则：

- 若结果大于等于 1，则必定命中
- 若结果小于等于 10%，则按 10% 保底
- 中间区间按概率随机

补充规则：

- 技能未命中时，默认不触发后续机制
- 若技能、状态或组件有特殊说明，则以特殊说明为准

### 6.3 伤害类型判定

#### 真实伤害

- 不进入普通伤害公式
- 不受护盾影响
- 直接扣除生命
- 可为固定值，也可为百分比
- 会参与最终乘区计算

#### 物理伤害 / 魔法伤害

- 使用同一伤害主体公式
- 区别主要体现在标签和对应伤害加成来源

### 6.4 基础伤害计算

定义：

`防御有效值 = 受击者防御 * (1 - 攻击者无视防御)`

基础伤害公式：

`基础伤害 = 伤害基数 * [1 - 防御有效值 / (防御有效值 + 2400)]`

参数说明：

- `伤害基数` 由攻击组件、子弹或技能组件提供
- `2400` 为当前版本防御衰减常数

伤害保底：

`基础伤害 >= 伤害基数 * 5%`

即：

`基础伤害 = max(伤害基数 * 5%, 防御衰减后伤害)`

### 6.5 暴击判定

暴击由攻击者暴击属性决定。

规则：

- 暴击率 >= 1，必暴击
- 暴击率 <= 0，必不暴击
- 0 到 1 之间按概率随机

补充规则：

- 只有命中后才判定暴击
- 若技能勾选“必定暴击”，则直接暴击

公式：

`暴击后伤害 = 基础伤害 * 攻击者暴击伤害`

### 6.6 格挡判定

格挡由受击者格挡和攻击者破击共同决定。

格挡概率公式：

`格挡概率 = 受击者格挡率 - 攻击者破击率`

边界：

- 最低为 0
- 最高为 1

格挡后伤害：

`格挡后伤害 = 暴击后伤害 * (1 - 基础格挡减伤 - 格挡强度)`

当前版本参数：

- 基础格挡减伤 = 50%
- 格挡强度取值范围 `[0,1]`

表现规则：

- 若同时发生暴击和格挡，表现上仅显示格挡

### 6.7 最终伤害

最终伤害公式：

`最终伤害 = 格挡/暴击处理后的伤害`

`* (1 + 攻击者伤害加成)`

`* (1 - 受击者伤害减免)`

`* (1 + 受击者受到伤害增加)`

`* (1 + 攻击者玩法伤害加成)`

`* (1 - 受击者玩法伤害减免)`

补充规则：

- `受击者伤害减免` 计算结果最大不超过 75%
- 最终伤害保底为 1
- `冷却缩减` 上限为 75%

### 6.8 护盾吸收

对于物理伤害或魔法伤害：

- 优先扣除护盾值
- 护盾不足时，再扣除生命值

真实伤害不受护盾影响。

## 7. 伤害加成乘区规范

所有标签型伤害加成共享同一乘区。

包括但不限于：

- 通用伤害加成
- 物理伤害加成
- 魔法伤害加成
- 普攻伤害加成
- 技能伤害加成
- 必杀伤害加成
- 单体伤害加成
- 穿透伤害加成
- 范围伤害加成
- 持续伤害加成

处理方式：

- 先将所有满足标签条件的伤害加成相加
- 再统一进入最终伤害乘算

示例：

若某角色拥有：

- 10% 伤害加成
- 20% 单体伤害加成
- 30% 范围伤害加成
- 40% 技能伤害加成
- 50% 必杀伤害加成
- 60% 物理伤害加成
- 70% 魔法伤害加成

则：

- 释放单体物理技能时，总伤害加成 = 10% + 20% + 40% + 60% = 130%
- 释放群体魔法必杀时，总伤害加成 = 10% + 30% + 50% + 70% = 160%

## 8. 治疗与恢复规范

### 8.1 基础公式

`加血数值 = 加血基数 * (1 + 治疗者治疗加成 + 被治疗者受治疗加成)`

结果保底：

- 至少为 1

### 8.2 标签区分

- `治疗`：允许暴击
- `恢复`：不允许暴击

### 8.3 治疗暴击

由治疗者暴击率判定是否暴击。

若发生暴击：

`最终治疗值 = 基础治疗值 * 治疗者暴击伤害`

补充设计口径：

- 治疗更偏向固定值设计
- 恢复更偏向百分比设计

## 9. 持续伤害规范

当前版本中，持续伤害类效果独立结算，互不干扰。

当前已明确独立结算的类型：

- 持续伤害
- 流血
- 灼烧
- 中毒

设计口径：

- 各类型分别计算
- 各类型互不覆盖
- 各类型互不合并

## 10. 护盾规范

护盾与“释放盾牌/加护盾”不判定暴击。

护盾公式：

`护盾值 = 护盾基数 * (1 + 护盾释放者施加护盾加成 + 护盾受到者受到护盾加成)`

## 11. 当前版本关键常数

- 最低命中率：`10%`
- 防御衰减常数：`2400`
- 基础伤害最低保底：`伤害基数 * 5%`
- 基础格挡减伤：`50%`
- 伤害减免上限：`75%`
- 最终伤害最低保底：`1`
- 冷却缩减上限：`75%`

## 12. 可直接用于数值设计的结论

- 防御的核心平衡常数为 `2400`
- 命中和闪避采用差值模型，且命中存在 10% 保底
- 暴击先于格挡判定
- 格挡通过“格挡减破击”决定概率
- 格挡在当前版本不仅有格挡强度，还自带 50% 基础减伤
- 所有伤害标签共用同一加成乘区
- 真实伤害绕过普通伤害公式与护盾，但会参与最终乘区
- 治疗可暴击，恢复不暴击，护盾不暴击
- 攻击速度直接表示每秒普攻次数
- 能量除自然回复外，还可通过能量球补充
- 持续伤害、流血、灼烧、中毒独立结算

## 13. 后续使用建议

在基于本体系做数值设计时，建议默认按以下顺序思考：

1. 先判断某个新属性属于哪一类属性。
2. 再判断该属性发生在战斗外成长，还是战斗内 buff。
3. 若讨论输出，优先按完整伤害链路逐步推导。
4. 若讨论生存，优先看生命、防御、减伤、格挡、护盾的组合关系。
5. 若讨论增伤，先判断标签，再判断其是否进入同一乘区。
