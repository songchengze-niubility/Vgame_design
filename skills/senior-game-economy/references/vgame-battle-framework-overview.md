## 1. 文档定位

`Vgame基本战斗框架.xlsx` 不是单纯的“战斗公式表”，而是一套完整的战斗数值母表，主要覆盖：

- 战斗属性定义
- 战斗公式定义
- 标准玩家模型
- 技能效果定价
- 怪物标准模型
- 玩家战力成长映射
- 主线等级与关卡映射
- 多玩法难度线

它本质上是 Vgame 战斗数值体系的总底盘。

## 2. 使用口径

当前建议按以下优先级使用本表：

### 2.1 主规则页

- `战斗属性`
- `战斗公式`

这两页定义战斗规则主口径，应作为战斗相关讨论的优先依据。

### 2.2 主模型页

- `标准模型`
- `技能系数`
- `怪物标准模型`
- `玩家战力曲线`
- `爬塔难度线`
- `玩法难度线`
- `无限挑战难度线`

这些页是当前战斗数值分析、技能定价、怪物成长和玩法难度的主干页。

### 2.3 参考或辅助页

- `草稿-技能系数公式`
- `怪物标准模型趋势`
- `养成属性规划（从养成属性复制）`

其中：

- `养成属性规划（从养成属性复制）` 已确认只是参考页，里面的数据不作为当前实际生效口径。
- `草稿-技能系数公式` 更像技能系数的临时推导页。
- `怪物标准模型趋势` 当前基本空白，可视为预留辅助页。

## 3. 页签逐页说明

## 3.1 战斗属性

作用：

- 定义战斗属性字典
- 明确属性类别、说明、数据类型、默认值、展示位置

核心字段：

- 属性类别
- 具体属性
- 属性说明
- 数据类型
- 默认值
- 展示位置

内容结构：

- 基础属性
- 战斗属性
- 百分比成长属性
- 等级属性

关键口径：

- `普攻攻击速度` 表示每秒普攻次数，不是动作倍率
- `效果命中`、`效果抵抗` 当前版本暂不使用
- `普攻攻击速度` 默认值已明确为“每个角色不同”，不作为统一固定值

与其他文档关系：

- 名词层与 [vgame-terminology.md](D:/数值文档/策划agent/项目专属/Vgame/skills/senior-game-economy/references/vgame-terminology.md) 对齐
- 规则层与 [vgame-battle-system.md](D:/数值文档/策划agent/项目专属/Vgame/skills/senior-game-economy/references/vgame-battle-system.md) 对齐

## 3.2 战斗公式

作用：

- 定义完整战斗结算顺序
- 明确命中、伤害、暴击、格挡、治疗、护盾的计算口径

内容结构：

1. 命中判定
2. 伤害类型判断
3. 基础伤害计算
4. 暴击判定
5. 格挡判定
6. 最终伤害结算
7. 治疗与恢复
8. 护盾

当前正式口径：

- 命中保底：`10%`
- 防御衰减常数：`2400`
- 基础伤害保底：`伤害基数 * 5%`
- 格挡概率：`受击者格挡率 - 攻击者破击率`
- 基础格挡减伤：`50%`
- 格挡强度范围：`[0,1]`
- 伤害减免上限：`75%`
- 最终伤害保底：`1`
- 治疗可暴击，恢复不暴击，护盾不暴击
- 真实伤害不受护盾影响，但参与最终乘区

说明：

- 页内格挡举例已按正式口径修正
- 本页与 battle-system 文档应视为同一规则来源

## 3.3 标准模型

作用：

- 定义标准战斗节奏
- 定义标准玩家攻防血模板
- 定义标准 boss 战斗表现
- 定义普攻、技能、必杀的输出结构和提升比例

内容结构：

- 前期 / 中期 / 后期战斗时长
- boss 战时长占比
- 裸体属性组成
- 基础属性投放系数
- 标准人属性
- 减伤率、eHP、ttk
- boss 战斗系数组成
- 普攻 / 技能 / 必杀每秒伤害占比
- 天赋、专武、旅券、主角装备、星盘等的提升比例

当前基础属性组成口径：

- 初始属性
- 升级属性
- 升阶属性
- 星盘属性固定值
- 专武属性
- 旅券
- 主角装备属性

说明：

- 本页是玩家和怪物战斗体验标定的核心基准页
- 它不直接配置具体角色，而是定义“标准人”

## 3.4 技能系数

作用：

- 将不同技能效果统一折算到同一数值口径
- 作为技能定价和技能平衡的基准表

当前覆盖效果：

- 单体伤害
- 穿透子弹
- 跟踪子弹
- AOE 子弹
- DOT
- 回血
- 护盾
- 眩晕
- 冰冻
- 沉默

常见定价因子：

- 技能当前品质
- 品质系数
- 攻速
- 冷却时间
- 作用人数
- 范围半径
- DOT 持续时间
- DOT 最大层数
- 控制持续时间

当前说明：

- 页中的 `SR` 是当前测试示例，不应理解为全局固定标准品质
- 本页更像技能效果的“等价值折算器”

## 3.14 无限挑战难度线

作用：

- 定义无限挑战玩法的怪物等级爬升公式

当前公式：

`怪物等级 = b + c * (n - 1) / ((n - 1) + a)`

参数：

- `a = 45`
- `b = 10`
- `c = 940`
- 怪物等级上限 = `500`

含义理解：

- `n` 为关卡序号
- `b` 决定起始等级
- `c` 决定整体抬升幅度
- `a` 决定前期爬升速度和中后期放缓节奏

曲线特征：

- 前期爬升较快
- 中后期逐渐放缓
- 最终逼近等级上限

## 4. 页签之间的关系

可以将这本表理解为如下结构：

1. `战斗属性` + `战斗公式`
   定义规则

2. `标准模型` + `技能系数`
   定义标准玩家和标准技能

3. `怪物标准模型`
   定义怪物基础母线

4. `主角等级` + `玩家等级对应关卡` + `玩家战力曲线`
   定义玩家进度如何映射到战力

5. `爬塔难度线` + `玩法难度线` + `无限挑战难度线`
   定义各玩法如何接入怪物等级与难度节奏

## 5. 当前已确认口径

- `战斗属性`、`战斗公式` 为主规则页
- `养成属性规划（从养成属性复制）` 只是参考页，数据不作为当前正式口径
- `技能系数` 页中 `SR` 为测试示例
- `队伍人数进度` 表示上阵人数变化
- `普攻攻击速度` 默认值不是统一固定值，而是每个角色不同
- `旅券副本` 为现行口径
