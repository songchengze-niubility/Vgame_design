# Vgame 角色套件关系地图

## 套件层级

Vgame 角色套件按以下层级理解：

1. 角色身份：编号、名字、简介、品级、阵营。
2. 战斗定位：攻击、防御、辅助、输出，以及更细的关键词。
3. 基础循环：普攻、自动技能、能效被动、必杀、天赋。
4. 成长扩展：升星节点、专武技能、通用被动。
5. 装备/套装扩展：旅券套装、映射装备词条。
6. 验证场景：主线、Boss、无限挑战、竞技场、爬塔、大秘境、资源/装备副本效率。

## 字段到设计含义

| 字段/区域 | 设计含义 | 常见追问 |
|---|---|---|
| 编号/名字 | 角色识别，但不等于最终配置主键 | 这个角色在真实配置里是哪一个 ID？ |
| 简介/阵营 | 世界观、阵营、角色包装 | 是否服务抽卡包装或剧情定位？ |
| 品级 | 稀有度和预期强度/获取层级 | A/S/SSR 是否有强度预算差异？ |
| 类型 | 队伍定位粗分 | 攻击、防御、辅助是否和技能表现一致？ |
| 普攻 | 自动输出基础节奏 | 射速、段数、弹幕、命中、触发器 |
| 能效被动 | 自动技能或被动机制 | 冷却、触发条件、伤害/控制/治疗/护盾 |
| 能效额外 | 对能效被动的强化 | 是否改变循环、范围、目标、冷却或收益 |
| 必杀 | 玩家择机释放的核心爆点 | 能量节奏、爆发窗口、清怪/Boss 价值 |
| 天赋 | 角色独有机制核心 | 是否定义角色玩法身份 |
| 升星 | 长期成长与机制解锁 | 是属性补点还是机制质变 |
| 思路 | 设计意图关键词 | 是否和实际技能描述一致 |
| 邮票/旅券 | 套装搭配方向 | 是否强化角色核心机制或提供泛用收益 |
| 专武 | 角色专属成长线 | 是否解决角色短板或放大核心机制 |
| 映射装备 | 随机深度和词条搭配 | 是否形成可追求的 build |

## 角色套件闭环检查

一个完整角色套件应能回答：

- 角色主要输出或贡献来自哪里：普攻、技能、必杀、召唤、Dot、护盾、治疗、控制、增益，还是机制转化。
- 自动循环是否自洽：冷却、能量、触发条件、持续时间、目标选择是否能自然形成节奏。
- 玩家操作点是否明确：必杀释放时机是否有收益差异，而不是单纯亮了就按。
- 升星是否提供成长感：至少部分节点应该强化核心机制，而不只是堆属性。
- 专武是否有专属性：应强化角色关键机制或补足关键短板。
- 旅券/映射装备是否有 build 空间：能形成选择，而不是所有角色都用同一套泛用增伤。
- 是否能被正式玩法验证：不同角色应在不同验证玩法中有清晰价值。

## 常见机制标签

从已观察文档中，Vgame 角色和装备会反复出现这些机制标签：

- 弹幕/子弹/穿透/范围/单体
- 幻象/召唤/护盾/黑洞/盾牌
- 暴击/暴击伤害/破挡/无视防御
- 能量回复/必杀增强/冷却缩减
- 治疗/溢出治疗/治疗率
- 低血/高血/已损失生命/护盾强度
- DOT/控制/受控目标增伤
- 普攻命中/普攻暴击/攻速叠层
- Boss 增伤/首领适配
- 移动距离、切换目标、站位前后 N 人

当机制涉及 runtime 支持时，转到 `vgame-battle-content-map` 确认技能、Buff、子弹、召唤、伤害效果或选择器配置链。

## 与成长系统的关系

角色套件和成长系统不要混在一起判断：

- 角色套件定义“角色为什么有趣、为什么有用”。
- 成长系统定义“投入资源后这件事如何变强”。
- 调参定义“这件事在具体场景中强到什么程度”。

因此：

- 问“这个升星是不是让角色质变”：先用本 skill 看机制，再用 `vgame-growth-combat-conversion-map` 看成长转化。
- 问“这个系数够不够”：先用本 skill 明确技能目的，再用 `vgame-battle-tuning-helper` 算 DPS/TTK/生存。
- 问“这个技能在哪个表配”：先用本 skill 定位设计语义，再用 `vgame-hero-skill-config-map`、`vgame-battle-content-map` 和 `vgame-config-schema` 找配置链。

## 与装备/套装的关系

### 专武

专武应优先和角色核心机制绑定。审查时看：

- 是否只提供泛用增伤。
- 是否强化角色已有循环，例如幻象、护盾、能量、必杀、普攻。
- 是否导致低星/无专武角色体验缺失过大。
- 是否需要美术需求、技能 ID、技能包装和效果文本同步。

### 旅券/邮票套装

旅券套装应看：

- 2 件套是否是轻量基础收益。
- 4 件套是否形成明确 build 条件。
- 关键词是否能支持推荐搭配和玩家理解。
- 是否出现 PVP 或特定玩法中过强的爆发窗口。

### 映射装备

映射装备包含机制型和属性型词条。审查时看：

- 机制型词条是否有触发条件、冷却、对象和叠加限制。
- 属性型词条是否和已有属性枚举/成长线一致。
- 是否掉落只作为设计表信息，真实来源和概率仍需走奖励/经济 skill。

## 风险模式

常见风险：

- 角色 ID 复用导致误认角色。
- 留档内容被误当正式设计。
- 备忘录中的实现需求被误当技能文本。
- 技能描述存在“触发触发器”的死循环风险。
- 子弹、召唤、护盾、站位选择、Buff 条件需要 runtime 支持但未确认。
- 升星、专武、旅券、映射装备全部叠在同一输出轴，造成乘区膨胀。
- 套件只在木桩环境强，移动关卡、Boss、爬塔或大秘境中验证失败。
- 角色定位写辅助，但实际全是输出增伤，或写防御但没有团队保护价值。
