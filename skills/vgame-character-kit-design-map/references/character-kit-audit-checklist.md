# Vgame 角色套件只读审查清单

## 使用方式

当用户要求“看一下这个角色/技能/专武/套装有没有问题”“这个设计能不能做”“帮我审一下角色强度方向”时使用本清单。

默认只读，不修改 Excel、配置表、JSON 或代码。

## 1. 证据范围

- 是否明确了文件、sheet、角色名、行范围或 ID？
- 是否区分了 `角色技能设计`、`角色（留档）`、`角色技能备忘录（后期删除）`？
- 是否存在合并单元格、公式引用、重复 ID、空值、异常品质/类型？
- 是否需要与真实配置表或 runtime 实现交叉验证？

输出要求：

- 写明已读证据。
- 对未读或无法确认的内容标 `待确认`。

## 2. 角色身份和定位

- 名字、简介、阵营、品级是否完整？
- 类型是攻击、防御、辅助还是输出？和技能实际贡献是否一致？
- 是否有明确关键词，例如暴击、幻象、护盾、能量、Boss、DOT、控制、普攻、必杀？
- 角色在 5 人小队中是否有清晰职责？

风险信号：

- 定位与技能文本矛盾。
- 角色只是泛用数值包，没有可识别机制。
- 角色过度依赖某个未确认玩法或装备。

## 3. 技能循环

- 普攻是否定义了射速、段数、伤害类型、范围或弹幕方式？
- 自动技能/能效被动是否定义冷却、触发、目标、持续、效果和限制？
- 必杀是否有玩家释放时机和收益差异？
- 天赋是否是角色身份核心，而不是普通被动换皮？
- 能效额外是否真正改变循环或强化关键机制？

风险信号：

- 描述中缺冷却、持续、触发上限、目标或范围。
- 触发条件可能造成递归触发或死循环。
- 技能全靠被动数值，没有战斗节奏。

## 4. 成长扩展

- 升星节点是否包含机制变化，而不只是属性？
- 专武是否绑定角色核心机制？
- 通用被动是否与角色定位匹配？
- 旅券套装是否支持角色关键词？
- 映射装备词条是否提供 build 分支，而不是唯一最优答案？

风险信号：

- 升星、专武、旅券、映射装备全部叠同一乘区。
- 关键机制被锁在高星或专武后，基础体验不完整。
- 装备词条和角色机制完全脱节。

## 5. Runtime 可实现性

若出现以下内容，转到 `vgame-battle-content-map` 确认配置和 runtime 支持：

- 子弹 ID 条件
- 子弹触发位置召唤
- 幻象复制技能或继承属性
- 护盾叠加、护盾死亡、护盾爆炸
- Buff 类型比较器、Buff 条件限制、触发后消失
- 站位前后 N 人选择
- 已损失生命值读取
- 直接伤害不触发任何效果
- 召唤物、黑洞、盾牌、控制、DOT、弹幕冻结

## 6. 数值和平衡

本 skill 只判断设计结构，不批准最终数值。

需要进一步计算时：

- DPS、爆发、持续输出、TTK -> `vgame-battle-tuning-helper`
- 成长曲线、长期强度、经济压力 -> `game-numerical-analysis` 或 `senior-game-economy`
- 星级、专武、旅券、映射装备对战斗表现的转化 -> `vgame-growth-combat-conversion-map`

审查时可先指出风险：

- 爆发窗口过短或过长。
- Boss 和清怪价值同时过高。
- 护盾/治疗/减伤形成无限续航。
- 能量回复造成必杀循环过密。
- 触发概率叠加导致体验不稳定。

## 7. 正式玩法验证

为每个角色或套件指出应该在哪类玩法验证：

- 主线：基础可用性、教学友好、队伍基础搭配。
- 资源副本：效率和稳定清场。
- 旅券副本：Boss 节奏、特定套装目标。
- 映射装备副本：高频 farming 和 build 追求。
- Boss 挑战：单体、机制处理、爆发窗口。
- 无限挑战：长线续航、弹幕压力、耐久。
- 竞技场 PVPVE：竞速效率、基地保护、技能爆发。
- 爬塔：单队 DPS 和垂直强度。
- 大秘境：box 宽度、环境适配和后期深度。

## 9. 强度验证补充

当角色审查进入“强不强”“哪里强”“是否泛用过强”时，转到 `vgame-battle-tuning-helper` 的这些 reference：

- `references/peer-comparison-rules.md`：没有同定位、同投入、同场景对照时，只能输出单角色体检或风险判断，不能输出最终强弱结论。
- `references/role-strength-benchmarks.md`：按攻击、防御、辅助、混合角色拆贡献维度。
- `references/standard-validation-scenarios.md`：统一主线、Boss、资源副本、旅券副本、映射装备副本、无限挑战、竞技场、爬塔、大秘境验证场景。
- `references/mechanic-value-conversion.md`：把护盾、治疗、召唤、DOT、控制、能量、冷却、概率触发折算成可比较假设。
- `references/character-strength-report-template.md`：输出完整角色强度报告。

角色套件审查输出不能只停在“设计合理”。如果用户问强度，必须补充：

- 同定位对标。
- 同投入、同场景对照条件。
- 至少一个标准验证场景。
- 直接 DPS 与非 DPS 机制折算。
- 已测和待测项。
- 是否存在泛用过强、场景过窄、投入后跳变过大的风险。
