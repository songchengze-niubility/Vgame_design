## DropId 范围约定

来自 `Datas\Drop\说明.txt`：

| 范围 | 用途 |
|---|---|
| 4 位数及以下 | 抽卡。 |
| `1` 开头 8 位 | 主线关卡、支线、精英。 |
| `2`、`3` 开头 | 挑战关卡、装备、锦囊、资源、幻境、过关斩将。 |
| `4` 开头 8 位 | 通行证奖励。 |
| `5` 开头 | 任务、每日任务、战令、支线任务。 |
| `6` 开头 8 位 | 活动、世界 BOSS 等。 |
| `65` 开头 8 位 | 大世界奖励。 |
| `7` 开头 8 位 | 公会奖励。 |
| `8` 开头 8 位 | PVPVE 奖励。 |
| `85` 开头 8 位 | 商城礼包。 |
| `9` 开头 8 位 | 限时活动、版本活动、开服活动。 |

补充约定：

- 1 级掉落用 8 位数字。
- 2 级掉落用 5 位数字。

## 分配新 DropId 前

1. 确认奖励属于哪个系统和源文件。
2. 在所有逻辑 Drop 源文件中查重，而不是只查目标文件。
3. 优先沿用附近已有序列。
4. 避免复用历史空洞，除非确认该空洞由同一系统保留。
5. 在影响矩阵中写明新 ID 范围和分配理由。
