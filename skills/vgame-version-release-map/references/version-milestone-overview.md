# Version Milestone Overview

## 版本节奏模型

### 版本周期（待确认实际节奏）

| 版本类型 | 预期周期 | 内容规模 |
|---|---|---|
| Major（大版本） | 3-6个月 | 新核心玩法/大型系统重做 |
| Minor（常规版本） | 2-4周 | 新角色/新关卡/新活动/系统迭代 |
| Patch（修复版本） | 按需 | Bug修复/数值调整/小优化 |
| Hotfix（紧急修复） | 即时 | 线上崩溃/严重漏洞 |

### 版本命名规范（待确认）

可能的命名方式：
- 语义版本号: `1.2.0`
- 内部代号: 里程碑名称
- 构建号: 递增数字
- 日期标记: `20260616`

## 版本内容规划模型

### 每个 Minor 版本通常包含

| 类别 | 内容举例 |
|---|---|
| 新角色 | 1-2个新可玩角色 |
| 新关卡 | 新主线章节/新副本层 |
| 新活动 | 1-2个限时活动 |
| 系统迭代 | 现有系统优化/新功能模块 |
| 数值调整 | 平衡性修改/经济调整 |
| Bug修复 | 累积bug清理 |

### 版本与配置的关系

| 版本阶段 | 配置状态 |
|---|---|
| 开发中 | 配置在开发分支，频繁变更 |
| 内测 | 配置冻结，只修bug |
| 灰度 | 配置锁定，通过开关控制可见 |
| 正式 | 配置上线，全量可见 |
| 维护 | 只接受Patch级修改 |

## 环境映射

| 环境 | 用途 | 配置来源 |
|---|---|---|
| Dev | 开发调试 | 开发分支最新 |
| Staging/QA | 测试验证 | 即将发布的版本 |
| Gray | 灰度验证 | 生产环境+灰度开关 |
| Production | 正式上线 | 正式发布版本 |

## 版本发布检查清单

### 发布前必查

| # | 检查项 | 说明 |
|---|---|---|
| 1 | 配置完整性 | 所有新增配置ID引用闭合 |
| 2 | SystemOpen 配置 | 新功能开放条件正确 |
| 3 | 活动时间 | 限时活动开始/结束时间正确 |
| 4 | 经济安全 | 无无限刷资源漏洞 |
| 5 | 向后兼容 | 旧版本客户端不崩溃 |
| 6 | 回滚方案 | 每个新功能有独立关闭方式 |
| 7 | 数据迁移 | 存量玩家数据兼容 |
| 8 | 灰度方案 | 高风险功能分批放量 |
