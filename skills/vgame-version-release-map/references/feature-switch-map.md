# Feature Switch Map

## 功能开关层级

Vgame 使用多层功能开关控制特性可见性和可用性：

### 第一层：SystemOpen（系统开放表）

SystemOpen 是 Vgame 最核心的功能开关机制，基于玩家进度控制系统开放：

| 字段 | 说明 |
|---|---|
| SystemId | 系统标识 |
| OpenCondition | 开放条件（玩家等级/主线进度/引导完成） |
| UIEntry | UI入口显示控制 |
| RedDot | 红点提示规则 |

**已知 SystemOpen 控制的系统（部分）：**

| 系统 | 开放条件类型 | 说明 |
|---|---|---|
| 抽卡 | 主线进度 | 完成特定主线关卡后开放 |
| 商店 | 玩家等级 | 达到指定等级开放 |
| 竞技场 | 主线进度 + 等级 | 双条件 |
| 公会 | 主线进度 | 较晚开放 |
| 旅券副本 | 主线进度 | 分批开放4个副本 |
| 大秘境 | 主线完成 + 等级 | 后期验证玩法 |
| 通行证 | 首次登录后 | 即时可见 |
| 映射装备副本 | 主线进度 | 较晚开放 |

### 第二层：服务器功能开关

服务器侧控制，不依赖玩家进度：

| 类型 | 用途 | 控制粒度 |
|---|---|---|
| 全局开关 | 整个功能on/off | 全服 |
| 灰度开关 | 按百分比/分组放量 | 用户组 |
| AB Flag | 实验分组 | 用户ID哈希 |
| 时间开关 | 定时开关 | 全服+时间段 |

### 第三层：活动时间控制

限时活动通过 Activity 表的时间范围控制：

| 字段 | 说明 |
|---|---|
| ActivityId | 活动ID |
| StartTime | 开始时间 |
| EndTime | 结束时间 |
| RepeatType | 重复类型（一次性/每日/每周/每赛季） |
| TargetVersion | 目标版本（待确认） |

### 第四层：客户端版本门控

| 机制 | 说明 |
|---|---|
| 最低版本号 | 低于此版本不显示功能入口 |
| 资源版本 | 热更资源不够不加载新UI |
| 强制更新 | 重大版本必须更新客户端 |

## SystemOpen 与 Level Progression 的关系

SystemOpen 的开放条件通常引用：
- `LevelId` — 需要通过某个关卡
- `PlayerLevel` — 需要达到某个等级
- `TaskId` — 需要完成某个任务/引导

这些条件由 `vgame-level-progression-map` 管理的解锁链决定。
