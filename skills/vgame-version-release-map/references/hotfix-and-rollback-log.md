# Hotfix & Rollback Log

## 定位

记录 Vgame 的紧急修复和回滚事件，用于：
- 事后追溯问题根因
- 评估变更风险
- 积累回滚方案经验
- 避免重复踩坑

## 日志格式

每条记录包含：

| 字段 | 说明 |
|---|---|
| 日期 | 事件发生时间 |
| 版本 | 影响的版本号 |
| 严重度 | P0(崩溃/数据丢失) / P1(功能不可用) / P2(体验问题) |
| 问题描述 | 简要描述问题现象 |
| 根因 | 技术/配置根因分析 |
| 影响范围 | 影响多少玩家/什么功能 |
| 修复方式 | Hotfix / Rollback / 服务器开关 / 补偿 |
| 配置变更 | 涉及的配置表和字段 |
| 回滚方案 | 如果修复失败的兜底方案 |
| 补偿措施 | 是否需要补偿玩家 |
| 后续改进 | 如何避免再次发生 |

## 常见问题模式

从项目经验可归纳的高频问题类型：

| 问题类型 | 典型表现 | 常见根因 |
|---|---|---|
| 配置引用断裂 | 关卡/商店/奖励显示异常 | DropId/ItemId 不存在 |
| 空引用崩溃 | 战斗中崩溃 | 实体已销毁但仍被引用 |
| 经济漏洞 | 资源异常增长 | 奖励重复发放/无限领取 |
| 开关失效 | 未开放功能可进入 | SystemOpen 条件配错 |
| 活动时间错误 | 活动提前/延迟开始 | 时区/时间配置错误 |
| 数值异常 | 怪物一击即死/打不动 | 属性缩放公式/系数错误 |
| 客户端兼容 | 旧版本崩溃 | 新配置格式不兼容 |

## 回滚方案模板

### 配置回滚

```
1. 确认问题配置文件和字段
2. 从备份恢复上一版本配置
3. 重新导出 JSON
4. 服务器热加载新配置
5. 验证问题消失
6. 评估是否需要补偿
```

### 功能回滚（服务器开关）

```
1. 通过服务器后台关闭功能开关
2. 确认功能入口不可见
3. 已进入功能的玩家妥善处理（踢出/保存状态）
4. 排查问题根因
5. 修复后重新开启
6. 评估是否需要补偿
```

### 版本回滚

```
1. 评估是否必须整版回滚（通常避免）
2. 通知运营和客服
3. 切换到上一稳定版本
4. 回退数据库变更（如有）
5. 全量通知玩家
6. 补偿方案执行
```
