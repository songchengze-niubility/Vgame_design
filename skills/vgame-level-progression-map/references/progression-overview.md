# Vgame 关卡进度总览

## 定位

本文件只描述“正式玩法如何挂在关卡进度链上”。奖励数值、战斗难度、经济价值不在此处定论。

## 正式玩法分层

| 层级 | 玩法 | 进度职责 |
|---|---|---|
| 主轴 | `主线` | 剧情、新手、功能开放、早中期推进。 |
| 稳定成长 | `资源副本-金币`、`资源副本-角色经验`、`资源副本-专武经验` | 日常基础资源供给。 |
| 稳定成长 | `旅券副本` | 中后期稳定装备成长。 |
| 稳定成长 | `映射装备副本` | 后期装备和随机深度成长。 |
| 验证玩法 | `无限挑战` | 长线耐久和综合账号深度验证。 |
| 验证玩法 | `竞技场（PVPVE）` | 同图竞速、基地承压和赛季竞争验证。 |
| 验证玩法 | `爬塔` | 单队纵向输出和周期进度验证。 |
| 验证玩法 | `大秘境` | 多环境、赛季、终局深度验证。 |
| 验证玩法 | `BOSS挑战` | 一次性 Boss 难度门槛。 |

## 已确认开启口径

来自 `vgame-core-understanding/references/current-gameplay.md` 的高层口径：

| 玩法 | 开启口径 |
|---|---|
| 主线 | 默认主轴，始终开放。 |
| 资源副本-金币 | `主线1-6`。 |
| 资源副本-角色经验 | `主线1-6`。 |
| 资源副本-专武经验 | `角色等级27`。 |
| 旅券副本 | `角色等级34`。 |
| 映射装备副本 | `主线2-15`。 |
| 无限挑战 | `主线2-10`。 |
| 竞技场（PVPVE） | `角色等级20`。 |
| 爬塔 | `角色等级25`。 |
| 大秘境 | `角色等级37`。 |
| BOSS挑战 | `角色等级30`。 |

这些是项目理解口径。做配置级结论时，还必须检查 `Unlock.xlsx`、`SystemOpen.xlsx` 和目标玩法表。

## 默认进度链

```text
主线
-> 章节/关卡推进
-> Unlock 条件满足
-> SystemOpen 入口开启
-> UIlevel 展示玩法/关卡
-> level 或玩法专表进入战斗/结算
-> Reward/Drop 字段引用奖励系统
```

## 任务边界

当用户问：

- “这个玩法什么时候开”
- “某个 LevelId 是什么关卡”
- “这个玩法涉及哪些配置表”
- “某个系统入口和关卡怎么连”
- “体力、扫荡、章节任务、章节星级奖励在哪查”

优先使用本 skill。

当用户问：

- “这个奖励数量对不对”
- “要把首通奖励改成什么”
- “DropId 怎么配”

转给 `vgame-reward-drop-sync` 和 `senior-game-economy`。

当用户问：

- “关卡难度、怪物血攻、防御公式、推荐战力是否合理”

转给战斗/数值相关 skill。

## 首版排除项

以下内容首版不主动纳入正式进度地图：

- 肉鸽相关表，例如 `level\Rouge\*.xlsx`、`Rogue\*.xlsx`
- 神经漫游相关表，例如 `neural_roam\*.xlsx`
- 小游戏相关表，例如 `WorldMap\SmallGame\*.xlsx`
- 历史、废弃、草稿玩法

如用户明确点名这些系统，可作为单次任务补充观察。
