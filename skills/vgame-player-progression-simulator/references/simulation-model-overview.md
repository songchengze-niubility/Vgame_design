# Simulation Model Overview

## 模拟框架

### 目标

给定玩家画像和行为假设，模拟 N 天内的资源积累、养成进度和卡点位置，帮助策划验证经济框架是否合理。

### 模拟粒度

| 粒度 | 说明 | 适用场景 |
|---|---|---|
| 天级 | 每天模拟一次 | 标准分析，7/14/30/60天 |
| 周级 | 每周汇总 | 长期趋势，90天+ |
| 事件级 | 按活动周期 | 活动收益评估 |

### 玩家画像模型

| 画像 | 特征 | 日活时长 | 付费 |
|---|---|---|---|
| F2P 休闲 | 只做每日，不刷满体力 | 15-30min | 0 |
| F2P 活跃 | 用完体力，完成所有每日/每周 | 45-60min | 0 |
| 小R（月卡） | F2P活跃 + 月卡 | 45-60min | 月卡 |
| 中R（通行证+月卡） | 全勤 + 月卡 + 通行证 | 60min+ | 月卡+通行证 |
| 大R | 全勤 + 额外付费抽卡/购买 | 60min+ | 月卡+通行证+额外 |

### 模拟核心假设

1. **体力假设**
   - 日基础体力恢复：待确认（可能120-240）
   - 付费体力购买：价格递增
   - 体力溢出：超过上限不计入

2. **行为假设**
   - F2P活跃：每日必做 = 每日任务 + 资源副本满次 + 主线推进
   - 体力分配：主线30% / 资源副本50% / 装备副本20%（可调）
   - 抽卡策略：攒到保底一次性抽 vs 来一个抽一个

3. **时间假设**
   - 主线推进速度：每天3-5关（受战力限制）
   - 卡关时转为刷资源副本
   - 活动期间资源收入 +20-50%

## 模拟输出

### 标准输出项

```
Day 7 Summary:
  Gold: +350,000 / -200,000 = Net 150,000 (Total: 520,000)
  Diamond: +340 / -0 = Net 340 (Total: 1,200)
  Character EXP: +96,000 / -80,000 = Net 16,000
  Main Character Level: 25 → 32
  Mainline Progress: Ch1-Stage 8 / 60
  Gacha Pulls Saved: 8
  Bottleneck: 角色经验中 (demand > supply)
  Next Milestone: Lv35 in ~3 days
```

### 里程碑检查点

| 时间点 | 期望达成 | 验证指标 |
|---|---|---|
| Day 3 | 完成序章，角色Lv15+ | 主线0-5通关 |
| Day 7 | Ch1完成，角色Lv30+ | 主线1-10通关 |
| Day 14 | Ch2进行中，角色Lv40+ | 资源副本全开放 |
| Day 30 | Ch3完成，角色Lv50+ | 开始验证玩法 |
| Day 60 | Ch4完成，角色接近Lv60 | 进入后期循环 |

## 模型局限性

- 模型基于假设，实际玩家行为有方差
- 不考虑卡关失败重试消耗
- 活动收益波动大，用平均值估算
- 抽卡结果有随机性，用期望值
- 新功能上线可能改变收入结构
