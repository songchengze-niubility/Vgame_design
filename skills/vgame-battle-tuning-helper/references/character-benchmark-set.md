# Vgame 角色高置信标杆集

## 使用定位

当用户要求评价角色强弱、做全角色量化、解释偏强偏弱、或要求“给出具体怎么改”时，使用本标杆集作为项目级强度锚点。

本文件记录用户明确认可的高置信角色。它优先级高于“同组均值”，但不替代真实 runtime、战斗日志和场景实测。

## 使用规则

- 同职业同品质存在标杆时，优先计算“标杆偏差”。
- 同组均值仍保留，用来发现整体分布和离群情况。
- 如果同组均值和标杆结论冲突，优先向用户说明冲突，而不是直接下最终结论。
- R 卡不设独立标杆；只检查是否超过同职业 SR 标杆。低于 SR 标杆属于符合品质预期，不视为低于风险。
- 跨品质比较只作为上限检查，不用于判定“低于标杆风险”；同品质比较才判断高于/低于标杆。
- Support SSR 当前缺少高置信标杆，评价时必须标注“待补标杆”。

## 标杆偏差

```text
标杆偏差 = (角色综合分 - 同职业同品质标杆综合分) / 标杆综合分
```

解释：

- `接近标杆`：同品质标杆偏差在 ±15% 内。
- `显著高于标杆`：同品质标杆偏差 >= +15%。
- `显著低于标杆`：同品质标杆偏差 <= -15%。
- `标杆本体`：该角色就是当前口径标杆。
- `低于SR标杆（符合品质预期）`：R 卡或其他低品质角色低于高品质标杆，不视为风险。
- `低品质越级风险`：低品质角色综合分高于同职业高品质标杆。

## 输出要求

评价角色时同时写：

- 对照标杆是谁。
- 标杆偏差是多少。
- 同组偏差是多少。
- 两个结论是否一致。
- 该结论是否仍受 SkillData、Buff 覆盖、召唤继承、Boss 免疫或实测缺失影响。

## 标杆校准检查

运行 `character_fullmax_quant.py` 时会生成 `标杆校准检查` sheet。优先查看：

- `标杆本体`：标杆角色当前最高子项是否符合定位。
- `高于标杆`：是否存在无明显成本却超过标杆的角色。
- `低品质越级风险`：低品质角色是否超过同职业高品质标杆。
- `待补标杆`：哪些分组仍缺高置信锚点。

这页用于校准模型权重和发现下一轮复核对象，不直接替代实战结论。
