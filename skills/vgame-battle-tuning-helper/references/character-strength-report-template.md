# Vgame 角色强度报告模板

## 使用定位

当用户要求完整审查一个角色、技能套件、专武或装备 build 的强度时，使用本模板。模板用于统一输出结构，方便策划审核、数值调参和后续配置验证。

## 结论写法

优先使用这些可审查结论：

- `结构健康，待数值验证`
- `结构健康，待横向对照验证`
- `单角色体检存在偏强风险，缺少同定位对照`
- `机制成立，但缺少配置证据`
- `核心循环偏弱，需要确认技能系数和覆盖率`
- `Boss 场景偏强，清怪场景成本明确`
- `多场景泛用偏强，需要削弱覆盖率或乘区叠加`
- `生存贡献偏强，需要检查护盾/治疗/减伤循环`
- `投入后强度跳变过大，需要拆分升星或专武收益`
- `证据不足，不能给最终强度结论`

## 必须避免

- 不要只写“强/弱”，必须说明在哪个场景、相对哪个定位、基于哪些证据。
- 不要在没有同定位、同投入、同场景对照组时输出最终强弱结论；此时只能写单角色风险判断。
- 不要把设计文档里的技能文本当作真实配置事实。
- 不要把 `battle_tuning.py` 的简化单体 TTK 当作完整战斗仿真。
- 不要把未读配置、未读 runtime、未确认公式写成确定结论。

## 最小报告版本

如果用户只要快速判断，至少输出：

```text
结论：
证据：
定位：
核心机制：
测算/待测：
风险：
下一步：
未修改配置：
```
