# Vgame 全角色全满量化口径

## 使用定位

当用户要求“所有角色强度量化”“给我一版可视化表”“全满横评”“告诉我具体怎么改角色数值”时，使用本口径，并优先运行 `scripts/character_fullmax_quant.py` 生成派生分析表。

本口径用于配置可见层的横向审查，不等于最终 runtime 实战 DPS。它把角色、升星、能效图、专武、技能文本和配置字段转成可比较的指数，帮助策划先定位“谁可能偏强/偏弱、该改哪个乘区”。

## 默认对照条件

- 角色等级：60 级。
- 升星：满星。
- 能效图：全节点点满。
- 技能：取各技能槽最高档。
- 专武：仅纳入 `ExclWeapon.xlsx` 中可通过说明列角色名明确绑定的具名专武；通用专武不强绑到单个角色。
- 对照组：同 `HeroClass` + 同 `Quality` 横向比较。
- 高置信标杆：读取 `character-benchmark-set.md`；当前标杆为输出 SSR 陈单单、输出 SR 克莉丝汀、坦克 SSR 塔拉夏、坦克 SR 大暑、奶妈 SR 陈双双。
- 低品质规则：低品质角色对高品质标杆只做上限检查；低于高品质标杆属于符合品质预期，不视为低于风险。
- 首领控制规则：Vgame 首领不吃控制；`BossScore` 不计控制关键词，`ControlScore` 只代表小怪、波次、PVPVE 等非首领场景价值。
- 多段倍率识别：只把伤害百分比之前、且占位符后紧跟枚/发/段/次等数量单位的参数当作段数；冷却秒数、持续秒数、概率次数不能放大伤害倍率。
- 普攻每秒倍率：技能描述中“每秒造成 X%，Y 发/秒”按每秒总倍率 X% 理解；“枚/发/霰弹/发射频率”优先视为弹体表现、命中频率或拆分 hit，不直接放大每秒总伤害。只有 `SkillData` 明确每枚独立结算时，才把枚数折成伤害倍率。
- 普攻循环折算：攻速持续窗口、每 N 次普攻额外子弹、普攻概率无视防御、普攻回能会折成普攻循环收益；主要进入持续输出，部分进入首领单体和爆发。
- 百分比上下文归因：治疗、护盾、团队增益、敌方易伤/降防必须按单个技能描述的 `%s` 占位符前后文归类；不能把整套技能文本里的所有百分比整包累加。自增伤、自攻速、普通伤害倍率、持续秒数、冷却秒数不计入团队增益或敌方减益。
- 配置安全：只读真实配置，输出到派生目录，不回写 Excel。

## 输出文件

脚本生成：

- `vgame_character_fullmax_quant_v2.xlsx`
  - `ScoresSnapshot`：静态分数、排名、同组偏差、改动建议，打开即可看。
  - `RawFeatures`：从配置抽出的面板、能效、升星、专武、技能关键词和风险字段。
  - `Assumptions`：权重、阈值、标杆、限制。
  - `标杆校准检查`：检查标杆本体子项分布、同品质相对标杆离群、低品质越级风险和待补标杆分组。
  - `Suggestions`：偏强/偏弱角色的主风险项和修改建议。
  - 所有 sheet 的表头必须使用中文显示名，并设置为 `黑体` 加粗；内部计算字段可以继续使用英文 key。
  - 所有出现 `同组偏差` 的 sheet 都必须用颜色标注偏差区间，方便快速扫离群角色。
- `vgame_character_fullmax_quant_v2_summary.md`
  - 轻量中文摘要，方便直接贴给用户。

## 分数解释

所有分数都是启发式指数：

- 100 附近代表同职业同品质的配置均线。
- `PeerDeltaPct >= +22%` 标记偏强。
- `PeerDeltaPct <= -18%` 标记偏弱。
- `标杆偏差` 表示角色相对高置信标杆的偏差；只有同职业同品质比较才用来判断高/低风险，低品质对高品质标杆只看是否越级。
- `配置风险` 表示发现明显表内矛盾，例如升星描述写物攻但实际加魔攻系数。

同组偏差颜色：

- 红色：`>= +22%`，偏强。
- 橙色：`+10%` 到 `+22%`，偏高观察。
- 绿色：`-10%` 到 `+10%`，接近均值。
- 浅蓝：`-18%` 到 `-10%`，偏低观察。
- 蓝色：`<= -18%`，偏弱。

评分维度：

- `BossScore`：单体/Boss 贡献。
- `AoeScore`：群体/资源副本/PVPVE 清场贡献。
- `BurstScore`：短时间爆发、必杀、开局能量和高倍率窗口。
- `SustainScore`：普攻、攻速窗口、普攻额外子弹、普攻触发、冷却、长期叠层、持续输出。
- `SurvivalScore`：生命、防御、护盾、治疗、减伤和低血收益。
- `SupportScore`：团队增益、团队治疗/护盾、团队能量或敌方伤害相关减益。注意 v2.1 已避免把 DPS 自增伤、自攻速、伤害倍率错误计成辅助。
- `ControlScore`：非首领控制价值；冰冻、沉默、眩晕、击退、减速、打断等只按小怪、波次、PVPVE 等首领以外场景估值。
- `OverallScore`：按职业使用不同权重合成。

默认权重：

- DPS：Boss 30%, AOE 20%, Burst 17%, Sustain 25%, Survival 5%, Control 3%。
- Tank：Survival 52%, Control 15%, Support 13%, Boss 10%, AOE 10%。
- Support：Support 52%, Survival 15%, Control 13%, Boss 10%, AOE 10%。

注意：以上 `Control` 权重是综合玩法口径，不代表 Boss 有效强度。做 Boss 结论时优先看 `BossScore`、TTK 和实测，不使用控制分抬高评价。

## 标杆校准检查页

`标杆校准检查` sheet 用于把标杆从“评价参照”推进为“参数校准线索”：

- `标杆本体`：检查陈单单、克莉丝汀、塔拉夏、大暑、陈双双自身的最高子项是否符合定位。
- `高于标杆`：同职业同品质角色显著高于标杆，优先检查其最高子项和定位成本。
- `低于标杆`：只用于同职业同品质角色显著低于标杆；低品质低于高品质标杆不进入此类。
- `低品质越级风险`：低品质角色高于同职业高品质标杆，优先检查面板、技能倍率或权重。
- `待补标杆`：当前分组缺少高置信标杆，只能先看同组均值。

这页只提出校准线索，不直接证明权重正确或角色必须调整；最终仍需要 SkillData/runtime 和实测数据。

## 改动建议口径

偏强时不要先砍基础攻击，优先改最高风险项：

- Boss 偏强：先压单体乘区、无视防御、易伤、高血锁定收益，建议 8%-12%。
- AOE 偏强：先压范围覆盖、命中数、范围技能/必杀系数，建议 8%-12%。
- Burst 偏强：先压必杀系数、开局能量、回能、短窗口暴击/暴伤，建议 8%-12%。
- Sustain 偏强：先压普攻增伤、攻速、冷却、长期叠层，建议 8%-12%。
- DPS 生存偏高：先压护盾、治疗、低血收益，建议 10%-15%。
- Support 偏强：先压团队增益覆盖率或幅度，建议 8%-12%。

偏弱时只补主定位，不要一口气补所有场景：

- DPS 偏弱：补主定位乘区 8%-12%，不要同时补单体和群体。
- Tank 偏弱：补有效护盾/减伤覆盖或防御转化，不优先补输出。
- Support 偏弱：补核心辅助价值或覆盖率，不让辅助输出追上 DPS。

## 必须说明的限制

输出结论时必须注明：

- 未解析 `SkillData .bytes`，所以不是最终实战 DPS。
- 多段命中、Buff 覆盖率、召唤/幻象继承、Boss 免疫、控制生效范围需要 runtime 或实测复核。
- 持续伤害、每秒伤害、频率翻倍、系数降低、最高叠层等复杂描述仍是启发式估算；发现离群角色时优先回查 `Skill.xlsx` 的单技能描述和 `SkillData`。
- 分数是横评雷达和改动优先级，不是直接配置目标值。

## 脚本调用

```bash
python scripts/character_fullmax_quant.py ^
  --config-root ${VGAME_ROOT}\Config\GameConfig\Datas ^
  --output-dir ${VGAME_ROOT}\codex_output
```

如果用户只要求结论，可以先运行脚本，再读取生成的 `summary.md`；如果用户要“怎么改”，同时查看 `Suggestions` 和 `RawFeatures`。
