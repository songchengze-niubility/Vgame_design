# Vgame 技能机制价值折算规则

## 使用定位

`battle_tuning.py` 当前偏单体 TTK 和生存测算。角色技能经常包含护盾、治疗、召唤、DOT、控制、能量、概率触发等非直接 DPS 机制。本文件提供首版折算规则，用来把机制先转成可比较的贡献假设。

这些折算只用于审查和测算前置假设，不等于最终战斗仿真。

## 通用折算步骤

1. 找到机制来源：技能、天赋、升星、专武、旅券套装、映射装备词条。
2. 确认触发条件：时间、命中、暴击、低血、击杀、释放必杀、获得护盾等。
3. 估算覆盖率：持续时间 / 周期，或触发概率 * 可触发次数。
4. 计算有效值：只算真正改变战斗结果的部分，不计溢出。
5. 标记风险：是否有上限、冷却、目标限制、玩法免疫或 runtime 待确认。

## 伤害类

### 直接伤害

```text
单次期望伤害 = 攻击属性 * 技能系数 * 防御穿透修正 * 暴击期望 * 增伤期望
DPS贡献 = 单次期望伤害 * 释放频率
```

注意：

- 多段伤害要看是否每段独立触发暴击、命中、Buff。
- 写有“本次伤害不触发任何效果”的伤害，不应继续触发额外伤害链。
- AOE 伤害要区分理论命中数和实际命中覆盖。

### DOT

```text
DOT贡献 = 每跳伤害 * 跳数 * 有效覆盖率
DPS贡献 = DOT贡献 / 周期
```

风险：

- 是否可叠层。
- 刷新是延长、覆盖、独立还是叠加。
- Boss 是否免疫或缩短持续。

### 额外伤害和概率触发

```text
期望额外伤害 = 触发次数 * 触发概率 * 单次额外伤害
```

风险：

- 高频普攻会放大概率触发。
- 触发效果是否有内置冷却。
- 是否会被幻象、召唤物、DOT 额外触发。

## 召唤和幻象

```text
召唤DPS = 本体DPS或攻击属性 * 继承比例 * 行为效率 * 覆盖率
覆盖率 = 持续时间 / 召唤周期
```

需要确认：

- 继承哪些属性：攻击、暴击、暴伤、增伤、攻速、技能。
- 是否能释放本体全部技能。
- 是否吃专武、旅券、映射装备、Buff。
- 多个召唤物是否可并存。

风险：

- 幻象复制技能可能造成触发递归。
- 召唤持续时间被暴击、击杀或技能延长时，需要上限。

## 护盾

```text
有效护盾 = min(护盾量, 预期受击伤害窗口)
护盾DPS等价 = 有效护盾 / 保护窗口时长
团队护盾价值 = 有效护盾 * 受保护目标数 * 实际承伤概率
```

注意：

- 溢出护盾不计入有效值。
- 可叠加护盾必须看上限。
- 护盾死亡、自然消散、被顶替触发效果要单独计。

风险：

- 护盾 + 治疗 + 减伤形成无限续航。
- 护盾强度同时吃生命、防御、治疗率、专武，可能多乘区膨胀。

## 治疗

```text
有效治疗 = min(治疗量, 目标已损失生命)
治疗HPS = 有效治疗 / 周期
团队治疗价值 = 有效治疗 * 有效目标数
```

注意：

- 溢出治疗不算有效治疗，除非有明确“溢出转护盾”机制。
- 低血触发治疗需要看冷却和每场次数。
- 治疗率、受治疗提升、护盾转化要分乘区确认。

## 减伤和防御

```text
有效生命提升 = 原始承伤 / (1 - 减伤后比例) - 原始承伤
```

简化判断：

- 持续减伤看覆盖率。
- 短窗口减伤看是否覆盖 Boss 爆发或弹幕高压段。
- 多个减伤来源要确认是相加、相乘还是取最大。

风险：

- Vgame 已有减伤上限概念，测算时不能无限叠。
- 防御收益受 `DEF/(DEF+24000)` 曲线影响，越高越需要看边际收益。

## 控制和打断

```text
控制价值 = 被阻止的伤害 + 被创造的输出窗口收益
```

需要确认：

- Vgame Boss 不吃控制；Boss 场景中控制价值按 0 处理。
- 控制持续、冷却、命中目标数。
- 是否打断技能、弹幕或移动。

风险：

- 控制在小怪场极强，但在 Boss 场无效，不能用控制抬高 Boss 强度结论。
- PVPVE 中控制可能变成竞速核心。

## 能量和冷却

```text
能量收益 = 额外必杀次数 * 必杀期望伤害或团队收益
冷却收益 = 额外技能次数 * 技能期望伤害或团队收益
```

需要确认：

- 能量上限和初始能量。
- 每秒能量回复和额外回复来源。
- 必杀释放是否受玩家操作或自动逻辑影响。
- 冷却缩减是否有上限。

风险：

- 能量辅助让必杀循环过密。
- “释放技能有概率不耗能”在高频技能角色上收益异常。

## 增益和减益

```text
团队增伤贡献 = 队伍基础DPS * 增伤比例 * 覆盖率 * 有效目标比例
减敌输出贡献 = 敌方DPS * 降攻或减伤比例 * 覆盖率
```

注意：

- 增伤对谁生效：自身、单体、全队、召唤物。
- 目标条件：高血、低血、受控、DOT、Boss、护盾中。
- 增伤是否进入同一乘区。

## 概率和覆盖率

基础公式：

```text
至少触发一次概率 = 1 - (1 - p)^n
期望触发次数 = n * p
```

其中 `p` 是单次触发概率，`n` 是窗口内可触发次数。

风险：

- 高频普攻角色会显著放大低概率效果。
- 多段技能是否每段独立判定必须确认。
- 召唤物和幻象是否参与判定必须确认。

## 输出要求

机制折算必须写：

- 原始机制描述。
- 折算公式。
- 关键假设。
- 已确认和待确认项。
- 是否需要 runtime 配置证据。
- 该折算用于哪个验证场景。
