# Vgame 角色强度基准

## 使用定位

本文件用于把“角色强不强”拆成可审查的贡献维度。首版是审查基准，不是最终数值红线；所有具体阈值都需要用真实角色、真实配置和实战数据继续校准。

使用场景：

- 比较同定位角色强弱。
- 判断一个新角色是否定位清晰。
- 把角色设计审查结果交给战斗调参。
- 说明某个角色是“数值弱”“机制弱”“场景窄”还是“投入后过强”。

## 基本原则

- 角色强弱必须通过横向对比得出。没有同定位、同投入、同场景对照时，只能输出单角色体检或风险判断，不能输出最终强弱结论。
- 先按角色定位比贡献，不要直接把攻击、防御、辅助放在同一 DPS 排行里。
- 分离基础形态和投入形态：0 星/无专武、关键星级、满星/专武/装备成型要分开判断。
- 分离单体、群体、长线、爆发、环境适配和团队贡献。
- 角色强度必须绑定验证玩法；一个角色可以在 Boss 强，但在无限挑战或 PVPVE 普通。
- 未读真实配置和技能 runtime 前，只能给设计风险，不能给最终强度结论。

## 角色贡献维度

| 维度 | 说明 | 常用证据 |
|---|---|---|
| 单体输出 | 对 Boss 或高血量目标的有效伤害 | Boss TTK、技能系数、暴击/增伤/无视防御 |
| 群体效率 | 清小怪、弹幕密集场、资源副本效率 | AOE 范围、命中覆盖、技能频率、小怪 TTK |
| 爆发窗口 | 短时间压血线或抢时间能力 | 必杀倍率、能量循环、爆发持续时间 |
| 持续输出 | 长战斗稳定贡献 | 技能冷却、召唤/DOT 覆盖率、普攻占比 |
| 生存贡献 | 自保或团队承伤 | 护盾、治疗、减伤、嘲讽、弹幕阻挡 |
| 团队增益 | 提高队友输出或稳定性 | 增伤、攻速、能量、暴击、治疗、控制 |
| 机制解法 | 解决特殊战斗问题 | 打断、控制、弹幕处理、站位保护、环境适配 |
| 成长放大 | 投入资源后的收益曲线 | 升星、专武、旅券套装、映射装备、星盘 |

## 攻击角色基准

攻击角色优先看：

- 单体 Boss TTK 是否达到目标场景的预期。
- AOE 或多目标价值是否会额外抬高总输出。
- 爆发是否需要玩家择机释放，还是自动循环已经过强。
- 输出轴是普攻、技能、必杀、召唤、DOT，还是多轴叠加。
- 是否同时在 Boss、清怪、PVPVE 竞速和长线玩法中都没有短板。

首版审查口径：

- 单体主 C：Boss 场景应有明显优势，但清怪/长线/生存至少有一个成本。
- AOE 主 C：资源副本和小怪波次应强，Boss 单体不应天然压过单体主 C。
- 爆发主 C：短窗口强，但能量、冷却或持续期应形成节奏成本。
- 召唤/DOT 主 C：长线应强，爆发和目标切换通常应有成本。

风险信号：

- 单体、AOE、长线、爆发全部强。
- 自带高生存且不牺牲输出。
- 多个乘区同时叠加：增伤、暴击、无视防御、攻速、额外伤害、必杀增幅全部集中。

## 防御角色基准

防御角色优先看：

- 自身有效生命：HP、DEF、减伤、闪避、护盾。
- 团队保护：护盾、阻挡弹幕、分摊、嘲讽、减敌伤害。
- 保护覆盖率：持续时间、冷却、触发条件、低血保护频率。
- 是否牺牲输出或节奏。
- 在无限挑战、PVPVE、Boss、高压弹幕场的价值差异。

首版审查口径：

- 主坦：团队生存收益应明显，但个人输出不应接近同投入主 C。
- 护盾坦：看护盾量、覆盖率、破盾收益和是否可叠加。
- 减伤坦：看减伤上限、持续时间、作用对象和是否与全队减伤叠乘。
- 保护型副坦：应有关键场景价值，而不是全场无脑最优。

风险信号：

- 护盾、治疗、减伤能循环到接近无限续航。
- 自保和团队保护都强，同时输出也高。
- 触发条件过宽，例如低血自动触发但冷却很短。

## 辅助角色基准

辅助角色优先看：

- 输出增益：增伤、暴击、攻速、无视防御、技能/必杀增强。
- 资源增益：能量回复、冷却缩减、技能不耗能。
- 稳定性：治疗、护盾、净化、非 Boss 控制、打断。
- 覆盖率和对象：单体、全队、前后 N 人、最低血量、最高输出。
- 个人输出是否过高。

首版审查口径：

- 输出辅助：团队 DPS 提升应可量化，但自身输出应有明确上限。
- 生存辅助：有效治疗和护盾应按有效值计算，不计入溢出。
- 能量辅助：必须评估必杀循环是否被压缩到失控。
- 控制辅助：要区分小怪价值、Boss 免疫、PVPVE 价值；Boss 场景控制贡献按 0 处理。

风险信号：

- 全队增益覆盖率接近 100%，且没有目标/站位/冷却成本。
- 能量、冷却、增伤叠加导致必杀循环过密。
- 辅助自身输出接近攻击角色。

## 混合角色基准

混合角色必须明确主身份和副身份：

- 攻击+辅助：输出低于纯主 C，团队增益成为价值来源。
- 防御+辅助：个人输出较低，团队保护和稳定性强。
- 攻击+防御：需要明显条件限制，否则容易变成泛用最优。

审查时使用“主贡献 + 副贡献折算”，不要把所有贡献简单相加后判定强度。

## 强度结论分级

| 结论 | 含义 |
|---|---|
| 结构健康 | 定位清晰，强项和成本都明确，后续可进入数值测算 |
| 机制强但数值待测 | 设计闭环强，但需要用配置和脚本确认强度 |
| 数值可能偏强 | 一个或多个核心场景预计超过同定位基准 |
| 数值可能偏弱 | 设计有亮点，但核心场景贡献不足或成本过高 |
| 场景过窄 | 只有极少数玩法有效，需确认是否符合投放目标 |
| 泛用过强 | 多场景同时强，且没有投入、操作、冷却或环境成本 |
| 证据不足 | 缺真实配置、技能实现、倍率、覆盖率或验证场景 |

## 输出要求

角色强度判断必须同时写：

- 角色定位。
- 横向对照组和对照条件。
- 对标维度。
- 已观察证据。
- 未确认数据。
- 适用玩法。
- 风险点。
- 需要转交的下游 skill。
