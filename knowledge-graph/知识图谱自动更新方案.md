# Vgame 知识图谱自动更新方案

> 目标：让客户端代码、配置表结构和策划文档保持在同一张可查询图谱中，同时确保 `${VGAME_ROOT}` 始终只读。

## 三个来源

| 来源 | 读取位置 | 产物 |
|---|---|---|
| 客户端代码 | `${VGAME_CLIENT_GRAPH}` | client 节点与代码关系 |
| 配置表 | `${VGAME_CONFIG_DATAS}` | `config-schema.json` |
| 策划文档 | `Vgame_design/design`、`proposals`、`tasks`、`plans`、`harness` | `design-docs.json` |

统一结果发布到 `knowledge-graph/.understand-anything/`，供本地 Dashboard 和 AI 查询。

## 更新通道

1. 双击 `更新知识图谱.bat`：全量读取并重建。
2. 双击 `启动监控.bat`：每 30 秒检查来源变化，按变化类型增量触发。
3. 双击 `检查图谱状态.bat`：只读验证 JSON、节点、边和来源状态。
4. 双击 `启动UA图谱.bat`：启动本地 Dashboard；首次运行安装前端依赖。

## 更新流水线

```text
配置表 -> extract_vgame_config_schema.py ----+
策划文档 -> extract_vgame_design_docs.py ----+-> merge_vgame_graph.py
客户端图谱 -------------------------------+        |
                                                     v
                              knowledge-graph.json + lite + meta
```

## 维护规则

- 新增配置表后，必须先确认已注册到 `__tables__.xlsx`，再全量更新图谱。
- 新增设计文档后无需手工登记，提取器会扫描约定目录。
- 监控状态保存在 `watchdog-state.json`，它是本地缓存，不提交 Git。
- 图谱更新失败时保留旧图和旧指纹，下次检查会继续重试。
- Dashboard 源码固定在 `understand-anything-new/`，项目事实 JSON 不写入其源码目录。

## 当前边界

- Vgame 暂未接入独立服务端/Proto 图谱，因此不会照搬 TKL 的 Go 服务和 Proto 流程。
- 配置外键仍包含启发式推断；真实修改前必须回源确认。
- 文档图谱只提取标题、摘要、类型和同名 Proposal/Design 关系，不替代内容理解。
