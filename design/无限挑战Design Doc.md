## 背景与目标
### 背景
无限挑战是 Vgame 中的无限循环跑酷挑战玩法。玩家在 3 种波次类型（普通 → 精英 → Boss）之间无限循环，每轮怪物等级按非线性公式递增（上限 500 级）。该玩法以 "最佳深度" 作为核心指标，每周结算奖励，验证玩家的续航和极限输出能力。

### 目标
- 提供无限递增难度的战斗验证
- 玩家追求 "最佳深度" 记录，形成长期追求目标
- 通过非线性等级增长曲线，在前期提供快速成长感，后期提供极端挑战
- 每周奖励激励持续参与

### 非目标
- 非排行榜竞速（每周结算不包含排名）
- 非多人 / 异步 PVPVE
- 非无尽之外的独立终点

## 核心规则
### R-01: 三阶段循环波次
- **触发条件**：玩家进入无限挑战
- **前置限制**：解锁条件满足（主线 2-10 通关）
- **操作流程**：战斗以波次为单位推进 —— 第 1 波次：Normal（普通怪群），第 2 波次：Elite（精英怪），第 3 波次：Boss；第 4 波次回到 Normal，以此无限循环
- **状态变化**：波次类型按 `Normal → Elite → Boss → Normal → ...` 循环
- **资源变化**：每波次怪物生成消耗服务器怪物池
- **UI 反馈**：HUD 显示当前波次编号和类型标签
- **异常边界**：波次计数器溢出（> 2^31）理论上不会在实际游戏中出现（500 级封顶前已不可战胜）
- **落地点**：战斗逻辑 → 波次生成器

### R-02: 怪物等级非线性增长
- **触发条件**：每次新波次开始
- **前置限制**：波次序号 n 从 1 开始，level 公式从 `无限挑战难度线` sheet 读取
- **操作流程**：怪物等级 = `27 + 1000 * (n - 1) / ((n - 1) + 38)`，结果取整；当 n=1 时 level=27，n→∞ 时 level→1027 但公式有 cap=500
- **状态变化**：怪物数据按计算出的 level 缩放属性
- **资源变化**：无
- **UI 反馈**：HUD 可显示当前怪物等级（可选）
- **异常边界**：n 极小（1-3）时等级 27-72，快速攀升；n 极大时增长趋缓；实际 cap 500 由配置限定
- **落地点**：怪物生成模块

### R-03: 最佳深度记录
- **触发条件**：玩家在挑战中被击败或主动退出
- **前置限制**：服务器记录历史最佳深度（最大波次数）
- **操作流程**：每次结束时的波次数与服务器已存储的最佳深度比较；若本次 > 历史最佳，更新记录并持久化
- **状态变化**：`best_depth` 字段可能更新
- **资源变化**：无直接资源，但最佳深度决定每周奖励档位
- **UI 反馈**：结算界面显示本次深度、历史最佳深度；若刷新纪录则播放特殊动画
- **异常边界**：中途断线时深度以服务器最后记录的波次为准；主动退出不视为断线，正常记录深度
- **落地点**：玩家属性 `player_infinite_best_depth`

### R-04: 每周奖励结算
- **触发条件**：每周结算时间到达（如周一 5:00）
- **前置限制**：服务器基于玩家本周内最佳深度作为结算依据
- **操作流程**：查询玩家本周最佳深度 → 匹配奖励档位表 → 通过邮件发放对应档位奖励
- **状态变化**：结算完成后本周深度冲榜缓存可清理
- **资源变化**：发放货币、道具等到玩家邮件
- **UI 反馈**：结算邮件通知；无限挑战入口显示当前奖励档位预览
- **异常边界**：结算时玩家在线则即时发放并通知；离线则静默发放邮件；同一玩家同一周不可重复结算
- **落地点**：定时任务模块 → 邮件系统

### R-05: 解锁条件
- **触发条件**：玩家查看无限挑战入口
- **前置限制**：主线关卡 2-10 已通关
- **操作流程**：系统检查主线通关记录，若 2-10 已通关则解锁入口；否则入口灰态
- **状态变化**：入口 `Locked → Unlocked`
- **资源变化**：无
- **UI 反馈**：解锁提示弹窗；入口 ICON 变化
- **异常边界**：如果玩家通过特殊渠道进入（非正常入口），服务器仍校验解锁条件，未解锁则拒绝并返回错误码
- **落地点**：入口 UI 面板

### R-06: 跑酷续航机制
- **触发条件**：波次切换间隙
- **前置限制**：角色 HP 在波次间不自动恢复（或有限恢复，视配置而定）
- **操作流程**：每完成一波，进入极短准备阶段（3-5s），期间角色可微调位置 / 技能冷却；HP/MP 保留不重置
- **状态变化**：波次间无恢复
- **资源变化**：无
- **UI 反馈**：波次间倒计时条；下一波类型预告（Normal / Elite / Boss）
- **异常边界**：准备阶段掉线，重新连接回到同波次准备阶段
- **落地点**：波次切换控制器

### R-07: 入场与退出
- **触发条件**：玩家点击进入 / 战斗中退出
- **前置限制**：无每日次数限制（无限入场）
- **操作流程**：入场无消耗；战斗中可主动放弃退出；退出时以当前波次作为本次深度
- **状态变化**：进入 → `InChallenge`，退出 → 结算深度
- **资源变化**：无入场消耗
- **UI 反馈**：确认退出弹窗（"当前深度将作为本次记录"）
- **异常边界**：频繁进退无惩罚；无利用价值（深度奖励只看最佳，非按次数累计）
- **落地点**：入口 → 战斗 → 退出逻辑

## 技术方案
### 架构影响
- 战斗层新增 `LevelType_InfiniteChallenge` 关卡类型
- 新增波次循环生成器模块
- 新增深度记录字段 `player_infinite_best_depth`

### 数据变更
| 表 | 变更类型 | 说明 |
|----|----------|------|
| 无限挑战难度线 sheet（Vgame基本战斗框架.xlsx） | 已有 | 等级增长公式参数 |
| 无限挑战周奖励配置 | 新增 | 深度档位 → 奖励内容映射 |
| player_infinite_best_depth | 新增 | 玩家最佳深度字段 |

### 接口变更
- 入场接口：返回当前可战波次及怪物配置
- 深度上报接口：波次结束时上报当前深度
- 每周结算接口：定时任务调用，批量发放奖励

## 风险评估
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 等级增长曲线导致后期不具备可玩性（过难或过易） | 低 | 低 | 公式参数可在配置表微调 |
| 波次生成怪物池枯竭 | 低 | 低 | 循环复用怪物模板 |
| 最佳深度记录丢失 | 极低 | 中 | 数据库持久化 + 每波次实时写入 |
