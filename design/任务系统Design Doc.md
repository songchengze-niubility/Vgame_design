## 背景与目标
### 背景
任务系统是驱动玩家日常行为与长线留存的核心管道，覆盖每日、每周、章节、系统引导四类任务。通过活跃点（Active Point）阈值机制串联日常活跃奖励与周常奖励，形成"做任务→积活跃点→领奖励"的闭环。

### 目标
- 支持每日、每周、章节、系统引导四类任务类型
- 每日任务通过权重池随机分配
- 活跃点达阈值触发 DailyActiveReward / WeeklyReward
- 任务链：完成任务后可解锁下一任务（NextTaskList）
- 系统引导任务轻量嵌入（好友、战斗、装备）

### 非目标
- 不处理成就系统（独立模块）
- 不处理赛季任务（由赛季活动配置）
- 不设计实时任务推送

## 核心规则
### R-01: 每日任务分配
- **触发条件**：每日 5:00 刷新
- **前置限制**：玩家已登录
- **操作流程**：从 DailyTask.xlsx 任务池按权重随机抽取 N 个任务分配给玩家 → 初始化进度为 0
- **状态变化**：玩家当日任务列表生成，进度均为 0/目标值
- **资源变化**：无
- **UI 反馈**：任务面板更新当日任务列表 → 进度条置零
- **异常边界**：权重总和为 0 或配置缺失 → 使用默认最小任务集
- **落地点**：DailyTask.xlsx, Task.xlsx

### R-02: 任务进度累积与完成
- **触发条件**：玩家行为匹配任务目标类型
- **前置限制**：任务处于进行中状态
- **操作流程**：行为触发 → 匹配活跃任务列表 → 对应任务进度 +1 → 达到 TaskTarget 则标记完成 → 可领取奖励
- **状态变化**：进度值递增；进度达目标后状态变为"已完成"
- **资源变化**：无（领取时发放）
- **UI 反馈**：任务进度条实时更新 → 完成时"领取"按钮高亮 → 红点提示
- **异常边界**：溢出进度不结转；断线恢复后重新计算进度
- **落地点**：Task.xlsx

### R-03: 任务奖励领取
- **触发条件**：玩家点击已完成任务的"领取"按钮
- **前置限制**：任务状态 = 已完成（未领取）
- **操作流程**：查询 Task.xlsx 中 DropId → 发放对应 Drop 奖励 → 标记已领取 → 检查 NextTaskList 是否有后续任务
- **状态变化**：任务状态变为"已领取"；活跃点增加
- **资源变化**：+Drop 奖励内容；+每日/每周活跃点
- **UI 反馈**：奖励飞入动画 → 任务标记 ✓ → 活跃点进度条推进
- **异常边界**：Drop 配置缺失 → 跳过奖励但标记已领取；重复领取拒绝
- **落地点**：Task.xlsx, DailyActiveReward.xlsx, WeeklyReward.xlsx

### R-04: 活跃点阈值奖励
- **触发条件**：每日/每周活跃点达到 DailyActiveReward.xlsx / WeeklyReward.xlsx 中配置的阈值
- **前置限制**：无
- **操作流程**：活跃点累加时遍历奖励阈值表 → 若当前活跃点 ≥ 阈值且该档未领取 → 可领取
- **状态变化**：该档奖励可领取
- **资源变化**：领取后 +RewardId 对应 Drop 奖励
- **UI 反馈**：活跃进度条标注阈值节点 → 达到时节点亮起 → 点击领取
- **异常边界**：阈值配置跳跃过大或重复 → 校验脚本拦截
- **落地点**：DailyActiveReward.xlsx, WeeklyReward.xlsx

### R-05: 任务链推进（NextTaskList）
- **触发条件**：当前任务完成并领取奖励
- **前置限制**：Task.xlsx 中该任务 NextTaskList 字段非空
- **操作流程**：读取 NextTaskList（任务 ID 列表）→ 将后续任务解锁并加入玩家任务列表
- **状态变化**：新任务状态 = 进行中
- **资源变化**：无
- **UI 反馈**：任务面板新增任务条目 → 提示"新任务解锁"
- **异常边界**：NextTaskList 中的任务 ID 不存在 → 跳过并日志告警
- **落地点**：Task.xlsx

### R-06: 系统引导任务
- **触发条件**：玩家首次进入对应功能场景（好友 / 战斗 / 装备）
- **前置限制**：SystemGuideTask.xlsx 中配置了相关引导
- **操作流程**：读取引导任务配置 → 展示轻量引导提示 → 完成任务（通常一步到位）→ 发奖励
- **状态变化**：引导标记完成
- **资源变化**：+引导奖励
- **UI 反馈**：半屏引导弹窗 → 完成任务后消失
- **异常边界**：同一引导不重复触发
- **落地点**：SystemGuideTask.xlsx

### R-07: 章节任务
- **触发条件**：玩家推进主线章节
- **前置限制**：达到对应章节
- **操作流程**：ChapterTasks.xlsx 读取当前章节任务列表 → 随主线推进自动触发 → 完成条件为通关章节节点
- **状态变化**：章任务进度跟随通关进度
- **资源变化**：领取后获得章节奖励
- **UI 反馈**：章节界面右侧任务面板
- **异常边界**：章节跳转（账号回溯）需重置任务状态
- **落地点**：ChapterTasks.xlsx

## 技术方案
### 架构影响
- 任务进度由事件驱动：战斗结算、资源变更等事件推送给任务模块匹配
- 每日/每周刷新定时器驱动，需处理跨天边界
- 活跃点系统独立于任务系统，仅通过事件耦合

### 数据变更
| 表 | 变更类型 | 说明 |
|----|----------|------|
| Task.xlsx | 配置 | 任务定义、目标、奖励 DropId、后续任务链 |
| DailyTask.xlsx | 配置 | 每日任务池 + 权重 |
| WeeklyTask.xlsx | 配置 | 每周任务集合 |
| DailyActiveReward.xlsx | 配置 | 每日活跃点阈值 → 奖励 |
| WeeklyReward.xlsx | 配置 | 每周活跃点阈值 → 奖励 |
| SystemGuideTask.xlsx | 配置 | 引导任务轻量配置 |
| ChapterTasks.xlsx | 配置 | 章节进度任务 |

### 接口变更
- `TaskGetList`：入参 TaskType；出参 TaskList[] (含进度、状态、可领标识)
- `TaskClaimReward`：入参 TaskId；出参 RewardItems[] + UpdatedActivePoints
- `ActiveRewardClaim`：入参 ThresholdId + Type(1=Daily/2=Weekly)；出参 RewardItems[]
- 事件推送：`OnTaskProgressUpdated`(TaskId, NewProgress)

## 风险评估
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 每日任务权重池全部可完成任务数为零 | 低 | 中 | 保证至少一个无条件任务常驻 |
| 活跃点计算误差导致阈值奖励不可领 | 低 | 中 | 服务端权威累加，客户端仅展示 |
| 跨天刷新时玩家在线导致任务未刷新 | 中 | 中 | 每日 5:00 强制推送刷新事件 |
| NextTaskList 配置循环引用 | 低 | 中 | 配置校验脚本检测循环链 |
| 引导任务在低端机卡死 | 低 | 低 | 引导含超时机制，允许跳过 |
