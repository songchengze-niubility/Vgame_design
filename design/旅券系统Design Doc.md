## 背景与目标
### 背景
旅券是角色养成核心的6槽位装备系统，采用4+2套装结构驱动配装策略深度。每个角色可装备6件旅券，按槽位分为前槽（基础属性）与后槽（百分比/等级/伤害标签），4件套提供核心套装效果，2件套提供属性加成。系统对标"圣遗物"式随机追深模型，品质4档、主属性32%权重、副属性68%权重，预期6件总战斗提升约30%。

### 目标
- 构建以套装组合为核心的深度配装策略，4+2结构支撑33套技能差异化
- 通过品质梯度（蓝/紫/橙/红）与随机副属性驱动长线养成追深
- 三级合成体系（基础/高级/特级）作为随机装备的核心消耗出口与回收闭环
- 控制6件总战力提升在~30%区间，避免数值膨胀

### 非目标
- 不支持装备锁定/已装备旅券的直接交易
- 不引入超越旅券品质的新稀有度层级（当前版本上限为红）
- 合成不产出非红品质旅券

## 核心规则
### R-01: 旅券装备与槽位绑定
- **触发条件**：角色打开旅券装备界面
- **前置限制**：角色已解锁旅券系统（主线进度或等级解锁）；无正在进行的配置操作
- **操作流程**：1) 选择角色进入旅券页签；2) 点击6个槽位中的目标槽位（前槽×?/后槽×?）；3) 从背包或旅券池中选择目标旅券；4) 确认装备
- **状态变化**：装备状态由 `Unequipped → Equipped`；旧旅券（如有）退回背包；当前角色套件计数 +1；2件套/4件套激活状态重新判定
- **资源变化**：无直接货币消耗（装备操作免费）；装备绑定生效
- **UI 反馈**：槽位图标刷新为所选旅券；套装激活高亮；套装效果面板左侧点亮；战力面板实时预测（对比现有装备）
- **异常边界**：同一旅券不可同时装备于多个角色；相同套装同名旅券同部位规则冲突时以最后装备为准；断网时回退至上次有效状态
- **落地点**：`EquipManager` 或等效装备系统（`StrategyCraft` 表驱动），发送 `EquipResult` 协议，战力重算

### R-02: 套装效果激活判定（4+2结构）
- **触发条件**：每次旅券装备/卸下操作完成
- **前置限制**：至少1件该套装旅券已装备
- **操作流程**：1) 统计该角色当前装备的各套装件数；2) 达到2件：激活套装2件套效果；3) 达到4件：激活套装4件套效果；4) 2件套与4件套效果独立判定；5) 同套装同时满足2件和4件时，两种效果均生效
- **状态变化**：套装激活状态 `Inactive → Active(2pc)` 或 `Inactive → Active(4pc)`；角色战斗属性实时更新；`System → SetEffectActive(SetId, 2 or 4)`
- **资源变化**：无
- **UI 反馈**：套装图标发光/边框变化；套件计数显示 `2/4`、`4/4`；套装效果文字高亮；激活弹窗提示（仅首次）
- **异常边界**：多个2件套可同时存在（如2+2+2），但4件套仅可同时1个（6槽位约束）；0件对应套装不触发任何效果；处理套装变更时先卸除效果再加成
- **落地点**：`SetEffectManager` 统一管理，`英雄设计.xlsx` Sheet `旅券套装技能` 中33套配置驱动

### R-03: 旅券品质与属性权重体系
- **触发条件**：旅券生成（掉落/合成）时
- **前置限制**：已确定旅券所属套装和槽位
- **操作流程**：1) 按掉落/合成规则决定品质（蓝=0.4/紫=0.6/橙=0.8/红=1）；2) 根据槽位类型决定主属性池（前槽→基础属性池/后槽→百分比/等级/伤害标签池）；3) 主属性值 = 基础值 × 品质系数 × 等级系数；4) 副属性从公共池随机抽取，值 = 基础值 × 品质系数 × 随机系数；5) 主属性权重32%，副属性权重68%，总战力值据此计算
- **状态变化**：旅券创建完毕；品质/属性值落实；战力估值计算；ID入库
- **资源变化**：新旅券占用背包格子 / 存储记录
- **UI 反馈**：品质边框色（蓝/紫/橙/红）；属性条数值显示；战力估值对比标记（高于当前→绿色箭头，低于→红色箭头）
- **异常边界**：品质系数超出[0.4,1]时强制钳位；主属性与槽位不匹配时按槽位规则重新分配（不抛错）；等级超过上限时钳位至5
- **落地点**：`TicketGenerator` 创建流程，`StrategyCraft.xlsx` 主副属性池配置

### R-04: 三级合成体系（基础/高级/特级）
- **触发条件**：玩家在合成界面提交所需材料旅券
- **前置限制**：基础合成→提交任意3个同品质或更高品质旅券；高级合成→除基础条件外额外消耗指定合成材料；特级合成→除高级条件外额外消耗锁定道具；背包空位≥1；被消耗旅券未装备
- **操作流程**：1) 选择合成等级入口（基础/高级/特级）；2) 放入消耗旅券（基础：任意3件；高级：3件+合成石；特级：3件+合成石+锁定符）；3) 预览产出规则（基础→套装随机；高级→套装+槽位确定；特级→套装+槽位+主属性确定）；4) 点击合成；5) 抵扣消耗；6) 生成新旅券
- **状态变化**：消耗旅券 `Active → Consumed` 从背包移除；生成新旅券 `Born → Active` 进入背包；特级：主属性锁定验证通过
- **资源变化**：消耗3件旅券；消耗合成石×1（高级/特级）；消耗锁定符×1（特级）；产出1件红旅券（品质固定=1，等级固定=0，副属性随机）
- **UI 反馈**：合成材料区域显示消耗预览；产出预览区显示概率范围（基础显示"随机套装"，高级显示"XX套装+XX槽位"，特级显示"XX套装+XX槽位+XX主属性"）；成功弹窗展示旅券详情；战力提升预估
- **异常边界**：材料不足时合成按钮灰化；空格不足时提示"背包空间不足"；合成石或锁定符不足时弹出快捷购买入口；服务器断连时不扣材料但暂停流程；过期/非法旅券ID过滤
- **落地点**：`TicketSynthesisManager`，三级合成配置在 `StrategyCraft.xlsx` Sheet `Synthesis`

### R-05: 旅券等级成长
- **触发条件**：玩家对指定旅券执行"升级"操作
- **前置限制**：旅券当前等级 < 最大等级（5）；有足够升级材料（旅券经验券/fodder）；旅券未处于出战且未锁定的异常状态
- **操作流程**：1) 选择旅券→点击升级；2) 投入升级材料；3) 预览升级后属性变化（主属性按等级系数成长）；4) 确认升级；5) 扣除材料；6) 等级+1，属性更新
- **状态变化**：`Level N → Level N+1`；主属性值增长（基础×品质系数×新等级系数）；副属性不变（等级不影响副属性）
- **资源变化**：扣除升级材料若干；金币消耗（按等级递增）；旅券经验累积
- **UI 反馈**：等级标签更新；属性值绿色"+XX"增长提示；升级成功动效；满级时升级按钮替换为"MAX"
- **异常边界**：超过最大等级时拒绝操作并提示"已达最大等级"；材料不足时显示缺口；升级过程中断网回退至升级前状态
- **落地点**：`TicketLevelUp` 流程，等级系数表 `StrategyCraft.xlsx`

### R-06: 旅券掉落与产出
- **触发条件**：击败指定怪物/完成指定关卡/开启奖励宝箱
- **前置限制**：该掉落池已配置旅券产出；玩家等级或关卡进度满足掉落解禁条件；背包空位≥1
- **操作流程**：1) 结算时判定是否触发旅券掉落（按配置权重/概率）；2) 确定品质（按掉落池配置的品质概率分布）；3) 确定套装/槽位/主副属性（按上述R-03流程）；4) 生成旅券实例；5) 进入背包；6) 记录掉落日志
- **状态变化**：战役结算→掉落判定→`DropResult(Ticket)`→背包入库
- **资源变化**：新旅券×1进入背包；背包计数+1
- **UI 反馈**：掉落弹窗展示旅券品质光效；与当前同槽位对比提示（"优于当前装备"/"不如当前"）；一键查看详情入口
- **异常边界**：背包满时发送到邮件/临时仓库；断线未领取的掉落进入补偿邮箱；掉落池为空时服务器告警并停止判定
- **落地点**：`DropManager → TicketDropProcessor`，掉落配置 `StrategyCraft.xlsx` DropSheet

### R-07: 旅券经济回收闭环
- **触发条件**：玩家将旅券标记为"分解"或合成消耗
- **前置限制**：旅券未装备且未锁定/未收藏；非唯一/任务不可丢弃品；背包非满状态（用于接收回收产出）
- **操作流程**：1) 选择旅券→分解/或合成入口自动选中；2) 预览回收产出（分解：货币+材料碎片；合成：新旅券）；3) 确认操作；4) 扣除旅券→产出结算
- **状态变化**：旅券 `Active → Destroyed`；产出材料/货币/新旅券进入背包
- **资源变化**：原旅券移除；奖励生成：分解产出 = 品质系数×基础货币 + 品质碎片；回收率达合成消耗的60-70%（防止无限循环）
- **UI 反馈**：分解预览窗显示预期获得；可批量选择（一键筛选蓝/紫品质）；分解动画（旅券碎裂→材料填充）
- **异常边界**：锁定旅券不可选中；批量分解上限限制（单次≤20）；逆合成（将低品质材料合成高品质需大量堆叠，不会造成通货紧缩）
- **落地点**：`TicketRecyclingManager`，回收系数表 `StrategyCraft.xlsx`

## 技术方案
### 架构影响
- 新增 `TicketManager`、`TicketSynthesisManager`、`SetEffectManager` 三个核心模块
- 装备系统需扩展对6槽位+多套装叠加的支持
- 战力计算模块需新增32%/68%权重的旅券战力估值算法
- 背包系统需支持旅券类型的专属过滤与批量操作

### 数据变更
| 表 | 变更类型 | 说明 |
|----|----------|------|
| StrategyCraft.xlsx | 新增Sheet | Synthesis/Slots/MainAttr/SubAttr/DropPool 等配置 |
| 英雄设计.xlsx | 扩展Sheet | 旅券套装技能（33套2/4件套效果+关键字） |
| PlayerCharacter | 新增字段 | TicketSlots[6], ActiveTicketSetEffects |
| Ticket | 新增表 | TicketId, SetId, SlotId, Quality, Level, MainAttr, SubAttrs[], EquippedCharId, Locked, Fodder |
| TicketMaterial | 新增表 | 合成石、锁定符、经验券等消耗品 |

### 接口变更
- `POST /ticket/equip` — 装备旅券
- `POST /ticket/unequip` — 卸下旅券
- `POST /ticket/synthesize` — 合成旅券（含等级参数）
- `POST /ticket/upgrade` — 升级旅券
- `POST /ticket/recycle` — 分解旅券
- `GET /ticket/list?charId=` — 获取角色旅券状态
- `Push: TicketDropNotify` — 掉落通知

## 风险评估
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 套装平衡性失控（某套过强导致全服统一配装） | 中 | 高 | 33套需逐一数值模拟；2/4件套效果单独调参；上线后预留调整窗口 |
| 随机追深导致玩家挫败感（副属性无法定向） | 高 | 高 | 特级合成可锁定主属性；未来版本开放副属性定向道具 |
| 三级合成经济闭环被破坏（回收率过高/过低） | 中 | 高 | 回收系数按品质加权+衰减；QA测试所有回收路径验证产出膨胀 |
| 等级升级主属性膨胀超预期 | 低 | 中 | 等级系数采用递减曲线（1→5级收益逐级降低） |
