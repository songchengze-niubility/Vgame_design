## 背景与目标
### 背景
专武在 Vgame 中不是普通数值装备，而是角色机制的延伸。专武与角色深度绑定（RelatedHero），承载一部分技能机制、基础属性和偏向属性。专武成长分为升级、突破、升星三条子线，各自承担不同职责：升级投基础属性，突破投偏向属性，升星投技能机制成长。专武有独立抽卡池（150 钻石/抽，60 保底），是与角色抽卡并列的另一商业化主轴。

### 目标
反推并记录专武系统的完整规则：武器定义与角色绑定、升级消耗曲线（SSR/SR）、突破门控、升星消耗（重复专武 + 万能道具）、每星解锁技能槽、属性成长公式。

### 非目标
- 不涉及专武抽卡池的具体概率和保底机制（另见抽卡系统文档）
- 不涉及旅券和映射装备（另见独立文档）
- 不涉及专武外观/特效展示

## 核心规则

### R-01: 专武定义与角色绑定
- **触发条件**：专武配置定义；运行时专武抽到或装备时读取
- **前置限制**：不适用
- **操作流程**：
  - 每个专武 ID 范围 51001+
  - `RelatedHero` 字段指定绑定角色（HeroId）
  - `HeroClass` 字段表示适配职业（DPS/Tank/Support）
  - `Quailty` = SSR/SR（R 品质专武待确认）
  - `Star` 字段表示初始星级
  - `SkillList`：长度为 6 的数组，按星级（对应专武自身的星级）解锁不同技能槽
  - `AttrNameList`：每次突破（ascension）增加的属性名和值（如 HpBonus, 0.2）
- **状态变化**：不适用
- **资源变化**：不适用
- **UI 反馈**：专武详情页展示绑定角色头像、适配职业图标、品质框
- **异常边界**：RelatedHero 指向不存在的 HeroId 时记录配置错误
- **落地点**：`ExclWeapon.xlsx`（Id/Name/Star/Quailty/HeroClass/RelatedHero/SkillList/AttrNameList/Hp/HpGrow/PhysicsAttack/PhysicsAttackGrow/MagicAttack/MagicAttackGrow/Armor/ArmorGrow）

### R-02: 专武升级消耗
- **触发条件**：玩家在专武详情页点击"升级"
- **前置限制**：专武当前等级 < 当前突破门控的 LvUpperLimit；武器经验道具充足；金币充足
- **操作流程**：选择升级次数 → 扣除武器经验道具 + 金币 → 专武等级提升 → 属性随等级增长
- **状态变化**：专武等级 +1
- **资源变化**：
  - SSR 专武 Lv1→60 累计 EXP：**2,634,500**，累计金币：**263,450**
  - 每级属性成长约 **1.42%**
  - SR 专武消耗约为 SSR × 0.8
  - 消耗曲线：前期平缓 → 中后期加速 → Lv50-60 占总量约 50%+
- **UI 反馈**：升级动画；攻血防增量提示；等级条更新
- **异常边界**：经验/金币不足提示获取途径；已达 LvUpperLimit 提示需要突破（进入 R-03）
- **落地点**：`ExclWeaponLv.xlsx`（等级 EXP 和金币需求）→ `ExclWeapon.xlsx`（属性成长字段：HpGrow/PhysicsAttackGrow/MagicAttackGrow/ArmorGrow 等）

### R-03: 专武突破门控
- **触发条件**：专武达到当前 LvLimit（20/30/40/50），玩家点击"突破"
- **前置限制**：玩家等级达标（PlayerLv 要求）；突破材料充足；金币充足
- **操作流程**：检查材料 + 金币 → 扣除 → LvUpperLimit 放开 → 可继续升级 → 同时获得 AttrNameList 中该 AscendId 对应的额外属性加成
- **状态变化**：AscendId +1；LvLimit 从 20→30→40→50→60 逐步放开
- **资源变化**：
  - 突破金幣消耗递增：20000 / 50000 / 100000 / 200000
  - 突破材料：按专武品质（SSR/SR）+ AscendId 从 ExclWeaponAscendMaterial.xlsx 取值
  - 额外属性加成：突破后获得对应 AscendId 的 AttrNameList 效果（如 HpBonus +20%）
- **UI 反馈**：突破动画；等级上限显示更新；新增属性显示
- **异常边界**：材料不足时引导到产出关卡；突破不成功不回滚已有状态
- **落地点**：`ExclWeaponAscendLv.xlsx`（AscendId/LvLimit/LvUpperLimit/PlayerLv）→ `ExclWeaponAscendMaterial.xlsx`（Quality/ItemClass/MoneyCost）→ `ExclWeapon.xlsx`（AttrNameList 各 Ascend 对应属性）

### R-04: 专武升星
- **触发条件**：玩家在专武升星界面操作
- **前置限制**：拥有同款专武重复（dupes）或万能专武材料；金币充足；当前星数 < 上限
- **操作流程**：消耗重复专武 + 万能材料 + 金币 → 专武星数 +1 → SkillList 对应槽位技能解锁/升级
- **状态变化**：专武星数 +1；SkillList[星数-1] 的技能效激活
- **资源变化**：
  - SSR 升星消耗：1星→10000金 / 2星→20000金 / 3星→50000金 / 4星→80000金
  - SR 升星消耗：1星→5000金 / 2星→10000金 / 3星→25000金 / 4星→40000金
  - 重复专武数量按星极递增，具体见 ExclWeaponStar.xlsx
  - 万能专武材料可替代重复专武
- **UI 反馈**：升星动画；星数更新；新技能槽点亮效果
- **异常边界**：重复专武和万能材料数量同时不足时提示获取途径；已达满星时不可继续升星
- **落地点**：`ExclWeaponStar.xlsx`（星数/消耗重复数/万能材料/金币）→ `ExclWeapon.xlsx`（SkillList 数组 6 技能槽）

### R-05: 专武属性计算与占比
- **触发条件**：任何专武养成模块变更（升级/突破/升星）后重新计算
- **前置限制**：不适用
- **操作流程**：
  - 基础属性（HP/ATK/DEF）= 专武初始属性 + 等级 × 各属性 Grow 字段值
  - 突破属性（HP% / ATK% / DEF% / 二级属性）= AttrNameList 中对应 AscendId 的加成值
  - 专武基础属性约占角色总属性的 **~15%**
  - 专武属性也受品质系数（SSR=1, SR=0.8）影响
  - 注：该品质系数与角色本体的品质系数是同一机制（仅影响裸体模块 + 专武属性）
- **状态变化**：角色面板属性中专武贡献部分更新
- **资源变化**：不适用
- **UI 反馈**：面板属性值变化展示
- **异常边界**：不适用
- **落地点**：`ExclWeapon.xlsx`（Hp/Grow 等字段）→ 运行时战斗属性计算

### R-06: 专武抽卡与获得
- **触发条件**：玩家进入专武抽卡池界面并点击抽卡
- **前置限制**：专武系统已开放（玩家等级 ≥ 27，通过 SystemOpen.UnlockId=4016）；钻石数量 ≥ 150
- **操作流程**：单抽消耗 150 钻石 → 从专武卡池中随机 → 保底 60 抽必出 SSR 专武 → 预期约 35 抽出 SSR
- **状态变化**：抽卡计数更新；保底计数器更新
- **资源变化**：扣除钻石 150/抽；获得专武（新武器进入仓库，重复武器自动转化为升星材料或分解为 ExclWeaponResolve 定义的道具）
- **UI 反馈**：抽卡动画；专武展示；保底进度条更新
- **异常边界**：钻石不足时提示充值/获取途径；卡池关闭/更换时保底是否继承（待确认）
- **落地点**：`DrawCard.xlsx`（专武卡池，OnePrice=150 钻石）→ `ExclWeaponResolve.xlsx`（专武分解返还）

### R-07: 专武系统开放
- **触发条件**：玩家等级达到 27 级
- **前置限制**：SystemOpen.UnlockId=4016 的条件（玩家等级 ≥ 27）已满足；系统未关闭（CloseSys=false）
- **操作流程**：Unlock 条件满足 → SystemOpen 广播 OnSystemUnlock → 专武入口按钮显示 → 可能需要新手引导（Sequence.UnlockId 匹配）
- **状态变化**：专武系统从不可用变为可用
- **资源变化**：不适用
- **UI 反馈**：主界面/角色页面显示专武入口按钮
- **异常边界**：CloseSys=true 时即使等级达标也不开放
- **落地点**：`SystemOpen.xlsx`（Id 对应专武系统，UnlockId=4016）→ `Unlock.xlsx`（Unlock_UUT_player_Grade, Params=27）

## 技术方案
### 架构影响
专武模块为已有功能，无需新模块。武器经验计算、突破门控、升星消耗散落在武器养成管理器中。

### 数据变更
| 表 | 变更类型 | 说明 |
|---|---|---|
| `ExclWeapon.xlsx` | 已有 | 武器核心定义：Id/Name/Star/Quailty/HeroClass/RelatedHero + Hp/HpGrow/PhysicsAttack/Grow/MagicAttack/Grow/Armor/Grow + SkillList(6) + AttrNameList |
| `ExclWeaponLv.xlsx` | 已有 | 每级升级消耗（EXP + 金币） |
| `ExclWeaponAscendLv.xlsx` | 已有 | 突破阶段门控（AscendId/LvLimit/LvUpperLimit/PlayerLv） |
| `ExclWeaponAscendMaterial.xlsx` | 已有 | 突破材料消耗（按 Quality × AscendId） |
| `ExclWeaponStar.xlsx` | 已有 | 升星消耗（重复武器数量/万能材料数量/金币） |
| `ExclWeaponResolve.xlsx` | 已有 | 武器分解返还道具（重复武器转化为升星材料的兜底） |

### 接口变更
无

## 风险评估
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 专武升星需要大量重复武器（SSR 满星需多少 dupes），F2P 几乎不可能满星导致付费墙感知 | 高 | 高 | 确认万能专武材料投放节奏；验证 F2P 30d/60d 至少能升到几星 |
| 专武 150 钻石/抽 + 角色 150 钻石/抽的双抽卡压力过大，F2P 两者不可兼得 | 高 | 中 | 验证钻石月收入能否支撑双抽卡基本体验 |
| 专武 SkillList 的 6 个技能仅靠升星激活，低星专武在机制上相比高星缺失太多 | 中 | 中 | 确保 0 星专武至少激活 1-2 个技能槽，核心机制不锁死在高星 |
| 专武 Lv50-60 的升级消耗占比过高（类似角色的 67%），非重氪玩家长期卡在 Lv40-50 | 中 | 中 | 确认此设计意图是否为中长期追求线；若是，需在 UI 上明确其高投入性质 |
| 专武 60 保底但预期 35 抽，实际期望与保底差较大，可能导致个别非酋体验差 | 低 | 低 | 保底机制已足够兜底 |
