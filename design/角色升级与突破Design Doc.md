## 背景与目标

### 背景

角色升级是 Vgame 最基础的养成线。角色从 Lv1 升至 Lv60，消耗角色经验道具 + 金币。等级上限受突破次数控制（0→4 次突破，逐次放开 10 级上限）。属性成长采用裸体公式：各来源（初始+等级+星盘+突破）加总后 × 品阶系数。

### 目标

反推并记录角色升级与突破的完整规则：消耗曲线、突破门控、属性成长公式、与关联系统的引用关系。

### 非目标

- 不涉及角色升星（另见独立文档）
- 不涉及角色星盘/能效（另见独立文档）
- 不涉及专武成长（另见独立文档）

## 核心规则

### R-01: 角色升级消耗

- **触发条件**：玩家在角色详情页点击"升级"
- **前置限制**：角色经验道具数量充足；金币数量充足；角色当前等级 < LvLimit（突破限制）
- **操作流程**：选择升级次数 → 扣除经验道具+金币 → 角色等级提升 → 属性随等级增长
- **状态变化**：角色等级 +1；累计经验消耗更新
- **资源变化**：
  - Lv1→60 累计 EXP：**1,922,500**（SSR）
  - Lv1→60 累计金币：**961,250**（SSR）
  - SR 品质消耗约为 SSR × 0.8
  - R 品质消耗约为 SSR × 0.7
  - 经验道具：低(500EXP/个)、中(2000EXP/个)、高(10000EXP/个)
- **UI 反馈**：升级动画；属性变化提示；升级后等级显示
- **异常边界**：经验/金币不足时提示获取途径（资源副本/主线/商店）；已达 LvLimit 时提示需要突破
- **落地点**：`HeroLv.xlsx` (Lv/UpgradeCost) + `HeroAttribute.xlsx` (每级属性) + `Vgame经济框架.xlsx` 角色升级 sheet

### R-02: 等级属性成长

- **触发条件**：R-01 升级成功触发
- **前置限制**：不适用
- **操作流程**：升级后重新计算裸体属性
- **状态变化**：角色 HP/ATK/DEF 及其他战斗属性按 HeroAttribute.xlsx 中对应等级取值增长
- **资源变化**：不适用（已在 R-01 消耗）
- **UI 反馈**：面板属性数值变化展示
- **异常边界**：HeroAttribute 中找不到对应等级时记录错误日志
- **落地点**：`HeroAttribute.xlsx`（ID=HeroId×1000+Lv）→ 运行时 `AttributeComponent`

### R-03: 角色突破门控

- **触发条件**：角色达到当前 LvLimit，玩家点击"突破"
- **前置限制**：满足 HeroAscendMaterial 中的材料数量要求；满足 PlayerLv 要求（玩家等级达标）
- **操作流程**：检查材料+金币 → 扣除 → LvUpperLimit 放开 → 可继续升级
- **状态变化**：AscendId +1；LvLimit 从 20→30→40→50→60 逐步放开
- **资源变化**：消耗对应品质（SSR/SR）的升阶材料（攻/防/辅）+ 金币。具体材料数量见 HeroAscendMaterial.xlsx
- **UI 反馈**：突破动画；突破后等级上限显示更新
- **异常边界**：材料不足时提示获取途径；注意突破本身不直接加属性（当前设计，只做门槛控制）
- **落地点**：`HeroAscendLv.xlsx` (AscendId/LvLimit/LvUpperLimit/PlayerLv) + `HeroAscendMaterial.xlsx` (AscendId/Quality/ItemClass/MoneyCost)

### R-04: 裸体属性计算

- **触发条件**：任何养成模块变更（升级/突破/星盘/升星）后重新计算
- **前置限制**：不适用
- **操作流程**：
  - HP = [初始HP + 等级HP + 星盘HP + 突破HP] × 当前品阶系数
  - ATK = [初始ATK + 等级ATK + 星盘ATK + 突破ATK] × 当前品阶系数
  - DEF = [初始DEF + 等级DEF + 星盘DEF + 突破DEF] × 当前品阶系数
- **状态变化**：裸体属性值更新，影响后续面板计算
- **资源变化**：不适用
- **UI 反馈**：不直接展示裸体属性（展示面板属性）
- **异常边界**：品阶系数异常时取 1.0
- **落地点**：`Vgame养成数值.xlsx` 属性规划 sheet + 运行时裸体→面板计算逻辑

### R-05: 品质系数

- **触发条件**：角色创建时确定，所有养成模块应用
- **前置限制**：不适用
- **操作流程**：SSR=1, SR=0.8, R=0.7。品质系数只影响裸体模块 + 专武属性，不影响旅券/映射装备
- **状态变化**：不适用
- **资源变化**：不同品质角色升级/突破消耗不同（HeroAscendMaterial 按 Quality 区分）
- **UI 反馈**：不适用
- **异常边界**：品质异常时按 R 处理
- **落地点**：`Vgame养成数值.xlsx` → `Hero.xlsx` (Quality 字段) → `HeroAttribute.xlsx` (按 HeroId 索引)

### R-06: 升级消耗曲线

- **触发条件**：配置定义，非运行时触发
- **前置限制**：不适用
- **操作流程**：EXP 和金币消耗随等级递增，后期加速：
  - 前期 (Lv1-20): 消耗平缓，30,000 EXP
  - 中期 (Lv20-40): 梯度上升，250,000 EXP
  - 中后 (Lv40-50): 加速，350,000 EXP
  - 后期 (Lv50-60): 陡峭，1,292,500 EXP（占总量的 67%）
- **状态变化**：不适用
- **资源变化**：不适用
- **UI 反馈**：不适用
- **异常边界**：不适用
- **落地点**：`Vgame经济框架.xlsx` 角色升级 sheet (Col D=每级EXP, Col F=每级金币)

### R-07: 经验道具

- **触发条件**：玩家获得经验道具时（主线首通/资源副本/每日任务等）
- **前置限制**：不适用
- **操作流程**：道具进入背包 → 升级时消耗
- **状态变化**：背包道具数量变化
- **资源变化**：3 档道具：低(100号, 500EXP, 0.5钻等价)、中(101号, 2000EXP, 2钻等价)、高(102号, 10000EXP, 10钻等价)
- **UI 反馈**：背包道具图标+数量显示
- **异常边界**：不适用
- **落地点**：`Items.xlsx` (Id=100/101/102) → 来源渠道见 `vgame-economy-source-map`

## 技术方案

### 架构影响

角色升级和突破为已有功能，无需新模块。

### 数据变更

| 表 | 变更类型 | 说明 |
|---|---|---|
| `Hero.xlsx` | 已有 | HeroId/Name/Quality/HeroClass/基础属性 |
| `HeroAttribute.xlsx` | 已有 | 每角色每级完整属性（ID=HeroId×1000+Lv） |
| `HeroLv.xlsx` | 已有 | 每级升级消耗（UpgradeCost） |
| `HeroAscendLv.xlsx` | 已有 | 突破阶段门控（AscendId/LvLimit/LvUpperLimit/PlayerLv） |
| `HeroAscendMaterial.xlsx` | 已有 | 突破材料消耗（按 Quality×AscendId） |

### 接口变更

无

## 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| Lv50→60 经验消耗占总比 67%，F2P 卡点可能过长 | 高 | 中 | 用 `vgame-player-progression-simulator` 模拟 F2P 30d/60d 进度 |
| 多角色同时升级金币压力显著（5角色满级=480万金币） | 高 | 中 | 验证金币日常收入是否能支撑预期角色培养数量 |
| 突破材料按攻/防/辅三分支，某职业供需失衡 | 中 | 低 | 监控各分支材料产出渠道 |
| 突破材料消耗数值不明确（HeroAscendMaterial 未完整反推） | 中 | 中 | 补读 Excel 获取精确值 |
