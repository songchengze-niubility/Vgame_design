## 背景与目标
### 背景
竞技场是 Vgame 中唯一的异步 PVPVE 玩法。两名玩家同时在同一关卡中竞速，击杀自己一侧的敌人会将其同步刷到对手一侧的敌人队列中。双方各有一个炮台（基地），未清理的敌人会攻击炮台减少基地 HP，先被击破的一方判负。该玩法兼具 PVE 输出效率和 PVP 对抗策略，是游戏社交和竞争的核心载体。

### 目标
- 提供 PVE → PVP 转化的竞争对抗体验
- 通过 "击杀即增敌" 机制创造攻防博弈（压输出 vs 守节奏）
- 通过日/周排名和奖励激励持续参与
- 通过炮台系统提供额外战略维度

### 非目标
- 非实时 PVP（异步匹配、同步战斗，但非帧同步格斗）
- 非多人组队竞技
- 非跨服竞技（首期仅本服）

## 核心规则
### R-01: 匹配与入场
- **触发条件**：玩家点击竞技场入口
- **前置限制**：玩家等级 >= 20；当前无进行中的竞技场战斗
- **操作流程**：点击 "匹配" → 服务器根据分段分数查找近分对手（优先在线玩家，超时后降级到机器人）；匹配成功 → 双方进入同一关卡开始战斗
- **状态变化**：`Idle → Matching → Matched → InBattle`
- **资源变化**：扣除入场券 / 体力（具体由经济表定义）；每日免费次数耗完后消耗道具补充
- **UI 反馈**：匹配等待动画（倒计时 / 转圈）；匹配成功震动提示；对手头像和分段展示
- **异常边界**：匹配超时（如 30s）自动匹配机器人（ArenaRobot.xlsx）；匹配中退出取消匹配；对手断线时服务器模拟对手操作直到战斗结束
- **落地点**：匹配服务器 → 战斗房间初始化

### R-02: PVPVE 核心机制 — 杀敌送敌
- **触发条件**：任一方击杀其一侧的敌人
- **前置限制**：双方共享同一关卡配置，敌人波次由双方击杀进度驱动
- **操作流程**：玩家 A 击杀一个敌人 → 该敌人立即加入玩家 B 的待刷出队列；B 侧的敌人会按队列顺序持续刷出；此机制双向生效
- **状态变化**：敌人从 A 侧移除 → 加入 B 侧队列 → B 侧生成
- **资源变化**：无直接资源变化
- **UI 反馈**：B 侧屏幕边缘出现 "敌人 +1" 预警提示（来源 A 玩家）；队列压力指示器显示当前背负的待清理敌人数量
- **异常边界**：若双方几乎同时清空一侧，以服务器时间戳为准决定击杀顺序；队列上限由配置定义，超出后暂停向对手输送
- **落地点**：PVPVE 战斗逻辑核心模块

### R-03: 基地 HP 与伤害系统
- **触发条件**：敌人未被及时清理、持续移动至基地位置
- **前置限制**：基地 HP 初始值由炮台等级决定；每个敌人对基地造成的伤害基础值固定
- **操作流程**：敌人到达基地攻击范围后自动攻击基地；基地 HP 持续扣减；任一方基地 HP 降为 0 则战斗结束，该方判负
- **状态变化**：基地 HP 从初始值递减 → 某一方 HP==0 → `BattleOver`
- **资源变化**：无（战斗内）
- **UI 反馈**：双方基地 HP 条在界面对应位置实时显示；HP 低于阈值时红色闪烁警告
- **异常边界**：双方基地几乎同时归零 —— 服务器以先 / 后时间戳判定胜负；一个玩家断线时其基地继续被攻击，正常扣 HP
- **落地点**：战斗场景 → 基地实体

### R-04: 炮台（Turret）系统
- **触发条件**：基地旁炮台自动攻击或玩家升级
- **前置限制**：炮台属性独立于角色；初始/升级数值从专有配置表读取
- **操作流程**：炮台自动攻击范围内的敌人；炮台可被玩家消耗资源升级，提升攻击力和基地 HP 上限；炮台被敌人攻击时会受损（HP 降低）
- **状态变化**：炮台等级提升 / 炮台 HP 变化
- **资源变化**：升级消耗战斗内金币（每局独立经济）
- **UI 反馈**：炮台模型 + 血条；升级按钮 + 消耗提示；炮台攻击特效
- **异常边界**：炮台被击毁后需等待自动修复或消耗资源修复；炮台修复期间敌人直接攻击基地
- **落地点**：炮台实体 → 升级 / 修复逻辑

### R-05: 战斗结算与胜负判定
- **触发条件**：任一方基地 HP==0 或一方主动投降
- **前置限制**：一方负方判定规则 —— 基地 HP 先到 0 输；投降直接判输
- **操作流程**：战斗结束 → 双方进入结算界面；胜者加分、败者扣分（或保底不扣，视分段）；新增分段分写入排行榜
- **状态变化**：`InBattle → Settling → Done`
- **资源变化**：胜者获得积分 + 奖励，败者获得参与奖励
- **UI 反馈**：结算界面展示胜负判定、段位分变化、奖励详情
- **异常边界**：如果对手系机器人（ArenaRobot.xlsx），胜负对分段影响调降；对局中一方向服务器异常，保留原始战斗数据以便 GM 回溯裁决
- **落地点**：结算服务

### R-06: 每日与每周排名
- **触发条件**：每日 / 每周结算时刻
- **前置限制**：排名分由 ArenaRankReward.xlsx 定义档位；ArenaWinReward.xlsx 定义单场胜利奖励
- **操作流程**：每日结算：基于日排名发邮件奖励（日清）；每周结算：基于周排名发邮件奖励（周清）；结算时间与服务器维护窗口对齐
- **状态变化**：排名快照生成 → 发放奖励 → 新周期排名重置
- **资源变化**：发放排名奖励（货币、荣誉道具等）
- **UI 反馈**：结算邮件；竞技场入口显示当前排名和预计奖励
- **异常边界**：结算时断线不影响发放（离线静默发邮件）；同周期排名数据不可篡改（只读快照）
- **落地点**：排名系统 + 定时任务

### R-07: 机器人对手
- **触发条件**：匹配超时或玩家分段无合适真实对手
- **前置限制**：机器人数据从 ArenaRobot.xlsx 读取，包括角色配置、AI 行为策略、难度档位
- **操作流程**：AI 模拟玩家行为 —— 清怪速率、炮台升级频率、失误概率按难度档位动态调节；真人无感（匹配界面不标注是否机器人）
- **状态变化**：机器人在对局中充当对手方，逻辑与 P2 完全一致
- **资源变化**：服务器负担（AI 实例）
- **UI 反馈**：结算界面不揭示对手为机器人（体验一致性）
- **异常边界**：机器人 AI 崩溃 —— 自动切换为挂机模式（敌人正常刷出但不升级炮台），保证对局正常结束
- **落地点**：ArenaRobot.xlsx → AI 行为执行模块

## 技术方案
### 架构影响
- 新增 `PVPVEMode` 运行模式，战斗服务器需支持双向消息同步和数据流
- 匹配服务器需处理玩家队列、分时段匹配策略、机器人降级逻辑
- 新增炮台实体 + 炮台升级 / 修复逻辑
- 排行榜需支持日/周双周期

### 数据变更
| 表 | 变更类型 | 说明 |
|----|----------|------|
| ArenaMonsterMultiplier.xlsx | 已有 | 怪物属性倍率配置 |
| ArenaWinReward.xlsx | 已有 | 单场胜利奖励 |
| ArenaRankReward.xlsx | 已有 | 排名奖励档位 |

### 接口变更
- 匹配请求 / 取消匹配 / 匹配结果通知
- 战斗消息同步（杀敌事件、炮台操作、基地 HP 变化）
- 结算请求 / 奖励发放
- 排名查询 / 奖励领取

## 风险评估
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| PVP 平衡失控（某角色/套路胜率过高） | 中 | 高 | 分段分隔离 + 持续数据监控 + 热更调整匹配系数 |
| 战斗同步延迟导致双方体验不一致 | 中 | 高 | 服务器权威判定 + 客户端预测 + 弱网容错 |
| 机器人 AI 被玩家识别出造成体验下降 | 中 | 低 | 持续优化 AI 行为多样性；高分段减少机器人占比 |
| 排行榜恶意刷分 | 低 | 中 | 对战历史审计 + 异常检测 + 匹配冷却机制 |
