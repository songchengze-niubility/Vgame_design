## 背景与目标
### 背景
邮件系统承担游戏内消息推送与补偿发放的职能。分两类邮件：纯文本模板邮件（Mail.xlsx）仅做消息通知；预发送邮件（PresendMail.xlsx）可附带 Drop 奖励，用于运营补偿、活动奖励下发。补偿类一次性大量投放不应计入日常收入核算。

### 目标
- 文本邮件基于模板 Id 下发，含标题、正文、过期时间
- PresendMail 绑定 Unlock 条件 + Drop 奖励，预配置后到达触发条件自动下发
- 邮件到期自动清理（ExpirationTime）
- 附件领取后邮件可删除保留
- 补偿投放与日常收入核算解耦

### 非目标
- 不设计玩家间邮件/私信
- 不处理富文本/图文混排邮件
- 不设计邮件收藏/归档

## 核心规则
### R-01: 系统邮件下发
- **触发条件**：运营后台/活动系统/补偿系统发起邮件下发
- **前置限制**：邮件模板 Id 在 Mail.xlsx 中存在
- **操作流程**：读取 Mail.xlsx 模板（Title, Text, ExpirationTime）→ 创建玩家邮件实例 → 写入玩家邮箱
- **状态变化**：邮件状态 = 未读
- **资源变化**：无（纯文本无附件）
- **UI 反馈**：邮箱图标红点 → 邮件列表新增条目
- **异常边界**：模板 Id 不存在 → 下发失败告警
- **落地点**：Mail.xlsx

### R-02: 预发送邮件触发
- **触发条件**：玩家达到 PresendMail.xlsx 中配置的 Unlock 条件（如特定等级/任务/活动进度）
- **前置限制**：PresendMail 条目中 Unlock 条件匹配
- **操作流程**：匹配条件 → 读取 MailId 获取邮件模板 → 读取 Drop 字段获取奖励附件 → 创建邮件实例并下发
- **状态变化**：邮件状态 = 未读 + 含附件
- **资源变化**：无（领取前不发放，资源仅记录在邮件附件中）
- **UI 反馈**：邮箱红点 → 邮件条目含附件图标
- **异常边界**：同一 PresendMail 重复触发 → 去重（按 PresendId + 玩家 ID 判定已发）
- **落地点**：PresendMail.xlsx

### R-03: 邮件附件领取
- **触发条件**：玩家打开含附件的邮件并点击"领取附件"按钮
- **前置限制**：邮件未过期；附件未领取
- **操作流程**：验证附件 DropId 有效性 → 发放 Drop 奖励至玩家背包 → 标记附件已领取 → 标记已读
- **状态变化**：附件状态变为已领取；邮件变为已读
- **资源变化**：+Drop 奖励内容
- **UI 反馈**："领取"按钮变为"已领取"灰色 → 奖励飞入
- **异常边界**：附件已领取重复点击 → 拒绝并提示；背包满 → 部分发放或转为临时存储
- **落地点**：PresendMail.xlsx

### R-04: 邮件过期自动清理
- **触发条件**：邮件创建时间 + ExpirationTime < 当前时间
- **前置限制**：无
- **操作流程**：定时任务扫描玩家邮箱 → 对比 ExpirationTime → 删除过期邮件 → 若含未领取附件则补发 Drop 至背包（或直接丢弃，按配置）
- **状态变化**：邮件从邮箱移除
- **资源变化**：补发未领附件（若配置补发）; 否则附件消失
- **UI 反馈**：邮箱列表自动刷新移除过期邮件
- **异常边界**：大量邮件同时过期 → 分批处理防卡顿；补发失败需日志记录
- **落地点**：Mail.xlsx (ExpirationTime)

### R-05: 邮件一键领取
- **触发条件**：玩家点击邮箱"一键领取"按钮
- **前置限制**：邮箱中存在 ≥ 1 封含未领取附件的邮件
- **操作流程**：遍历所有含未领取附件的邮件 → 批量汇总 Drop → 一次性发放 → 批量标记已领已读
- **状态变化**：全部含附件邮件附件标记已领取
- **资源变化**：汇总全部附件奖励一次性到账
- **UI 反馈**：批量飞入汇总奖励 → 所有邮件标注已领
- **异常边界**：某封邮件附件 Drop 无效 → 跳过并记录，不阻塞其余领取
- **落地点**：PresendMail.xlsx

### R-06: 邮件已读/删除
- **触发条件**：玩家点击单封邮件打开 或 点击删除
- **前置限制**：打开 → 无；删除 → 附件已领取或邮件无附件
- **操作流程**：
  - 打开：标记已读 + 展示内容
  - 删除：从玩家邮箱移除
- **状态变化**：标记已读 / 邮件删除
- **资源变化**：无
- **UI 反馈**：打开 → 邮件详情展开；删除 → 邮件从列表移除
- **异常边界**：删除含未领取附件邮件 → 弹二次确认；拒绝删除
- **落地点**：Mail.xlsx

### R-07: 补偿投放经济隔离
- **触发条件**：运营通过 PresendMail 发起大规模补偿投放
- **前置限制**：补偿类 PresendMail 需标记为"补偿"类型
- **操作流程**：发放时记录来源类型 = 补偿 → 数据统计时区分"日常收入"与"补偿收入" → 不做日常收入核算
- **状态变化**：无
- **资源变化**：同 R-02/R-03
- **UI 反馈**：无特殊反馈
- **异常边界**：补偿标记缺失 → 统计部门按默认规则处理
- **落地点**：PresendMail.xlsx

## 技术方案
### 架构影响
- 邮箱为玩家附属数据，设计容量上限（如 100 封），超上限自动删除最旧已读邮件
- PresendMail 去重机制：玩家维度记录已触发的 PresendId 集合
- 过期清理为定时任务，建议每日 5:00 执行

### 数据变更
| 表 | 变更类型 | 说明 |
|----|----------|------|
| Mail.xlsx | 配置 | 邮件模板（Id, Title, Text, ExpirationTime） |
| PresendMail.xlsx | 配置 | 预发送邮件（Unlock, MailId, Drop） |

### 接口变更
- `MailGetList`：出参 MailList[] (Id, Title, SendTime, IsRead, HasAttachment, IsClaimed, ExpireTime)
- `MailRead`：入参 MailId；出参 Content + AttachmentInfo
- `MailClaimAttachment`：入参 MailId；出参 RewardItems[]
- `MailClaimAll`：出参 TotalRewardItems[]
- `MailDelete`：入参 MailId (可批量)；出参 Success

## 风险评估
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 补偿投放计入日常收入导致数据偏差 | 中 | 中 | 标记来源类型，统计侧区分 |
| 过期邮件附件补发遗漏 | 低 | 中 | 补发日志 + 对账脚本 |
| 全服邮件下发性能瓶颈 | 中 | 高 | 分批写入，延迟可接受 |
| 邮箱容量爆满 | 低 | 低 | 自动清理最旧已读无附件邮件 |
| 重复 PresendMail 触发 | 低 | 中 | 玩家维度去重 |
