## 背景与目标
### 背景
商店系统承担游戏内多货币经济循环的终端出口角色，覆盖钻石、金币、竞技币、公会币、商店币等多元货币的消费场景。通过分页、刷新周期、限购机制控制资源流出节奏，同时接入 PayProduct 实现直购付费。

### 目标
- 统一管理多货币货架，支持分页、刷新、限购
- 日/周/月刷新周期驱动商品重置
- 支持 PayProduct RMB 直购商品
- 售罄隐藏、限购类型/数量控制
- 解锁条件集成 Unlock.xlsx 系统

### 非目标
- 不处理实时拍卖行/玩家间交易
- 不设计动态定价引擎
- 不处理活动限时商店（由活动系统接管）

## 核心规则
### R-01: 商品购买
- **触发条件**：玩家在商店页面点击商品"购买"按钮
- **前置限制**：商品未被售罄；未达到限购上限；持有足够货币/PayProduct；商店已解锁
- **操作流程**：服务端校验资源 → 扣除货币 → 发放商品 → 更新购买计数 → 回传结果
- **状态变化**：商品剩余可购次数 -1（若有限购）；玩家货币减少
- **资源变化**：-币种×数量（PropType "币种ID,金额"）；+商品货物
- **UI 反馈**：货币更新动画 → 商品计数刷新 → 若售罄则置灰/隐藏
- **异常边界**：货币不足弹提示；并发购买需加锁防超购
- **落地点**：Shop.xlsx, ShopGoods.xlsx

### R-02: 商店刷新（自动周期刷新）
- **触发条件**：到达刷新时间点（每日 5:00 / 每周一 5:00 / 每月 1 日 5:00）
- **前置限制**：商店 Tab 配置了刷新周期
- **操作流程**：遍历所有商店 Tab → 匹配当前刷新周期 → 重置购买计数 → 若为随机商品则重新随机货架
- **状态变化**：所有限购商品的可购次数恢复为上限
- **资源变化**：无
- **UI 反馈**：商店标签角标"已刷新"提示；货架内容变更动画
- **异常边界**：服务器重启时需对照上次刷新时间判断是否执行刷新
- **落地点**：Shop.xlsx

### R-03: 手动刷新商店
- **触发条件**：玩家点击"手动刷新"按钮
- **前置限制**：该商店 Tab 支持手动刷新；玩家持有刷新消耗资源
- **操作流程**：扣除刷新费用 → 重置购买计数 → 重新随机货架物品
- **状态变化**：刷新次数 +1；购买计数重置
- **资源变化**：-刷新费用
- **UI 反馈**：刷新动画 → 货架重展示
- **异常边界**：每日手动刷新次数上限到达时隐藏按钮；费用不足提示
- **落地点**：Shop.xlsx, ShopGoods.xlsx

### R-04: 限购控制
- **触发条件**：每次购买请求时校验
- **前置限制**：ShopGoods 行配置了 LimitType（1=终身/2=每日/3=每周/4=每月）与 LimitNum
- **操作流程**：读取玩家当日/当周/当月/总计已购次数 → 对比 LimitNum → 通过则允许购买
- **状态变化**：购买成功后对应周期计数器 +1
- **资源变化**：无额外
- **UI 反馈**：商品卡片显示"剩余 X/X"
- **异常边界**：LimitType 非法值按终身处理
- **落地点**：ShopGoods.xlsx

### R-05: PayProduct 直购
- **触发条件**：玩家点击标有 RMB 价格的商品
- **前置限制**：商品配置了 PayProduct 字段
- **操作流程**：客户端拉起支付 SDK → 支付成功回调服务端 → 发放商品 → 若失败不扣不发
- **状态变化**：购买成功后更新计数器
- **资源变化**：玩家支付 RMB；+商品货物
- **UI 反馈**：支付中 loading → 成功/失败提示 → 货架更新
- **异常边界**：支付回调延迟需异步补发；掉单需对账补发
- **落地点**：ShopGoods.xlsx

### R-06: 售罄隐藏
- **触发条件**：商品可购次数降至 0 且 HiddenWhenSoldOut = true
- **前置限制**：无
- **操作流程**：客户端拉取商品列表时过滤隐藏已售罄商品；服务端同样拒绝购买
- **状态变化**：商品对玩家不可见
- **资源变化**：无
- **UI 反馈**：货架位置自动补位（下方商品上移）
- **异常边界**：商品隐藏后需在刷新周期到时重新可见
- **落地点**：ShopGoods.xlsx

### R-07: 多货币价签展示
- **触发条件**：商店界面打开/刷新
- **前置限制**：无
- **操作流程**：读取 Shop.xlsx 中 PriceShow 字段（货币 ID 列表），在界面上方展示玩家当前各货币余额
- **状态变化**：无
- **资源变化**：无
- **UI 反馈**：货币栏水平排列展示余额
- **异常边界**：货币类型过多时滑动展示
- **落地点**：Shop.xlsx

## 技术方案
### 架构影响
- 商店服务模块需与货币系统、解锁系统、支付系统解耦，通过事件/接口调用
- 限购计数器存储于玩家存档，刷新时批量重置
- 随机货架种子由服务端生成，防客户端作弊

### 数据变更
| 表 | 变更类型 | 说明 |
|----|----------|------|
| Shop.xlsx | 配置 | 商店页签、刷新周期、货币展示 |
| ShopGoods.xlsx | 配置 | 商品 Item、价格（PropType）、限购、售罄隐藏 |
| Unlock.xlsx | 配置 | 商店解锁条件 |

### 接口变更
- `ShopGetList`：入参 TabId；出参 GoodsList[] (含剩余次数)
- `ShopBuy`：入参 GoodsId + Count；出参 RewardItems[] + UpdatedCurrency
- `ShopRefresh`：入参 TabId；出参 NewGoodsList[] + RefreshCost
- 复用通用支付回调接口

## 风险评估
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 并发购买绕限购 | 低 | 高 | 服务端加锁/事务保护 |
| 刷新时间配置错误导致不刷新/频繁刷新 | 低 | 中 | 配置检查脚本 + 上线前验证 |
| PayProduct 掉单 | 中 | 高 | 补发机制 + 对账脚本 |
| 价格配置错误（货币 ID 写错） | 中 | 高 | Excel 提交时跑校验脚本 |
| 货币价值失衡导致通货膨胀 | 中 | 高 | 与数值规划对齐，监控日报 |
