## 背景与目标
### 背景
主角等级（玩家账号等级）是 Vgame 整个进度节奏体系的"基础横轴"。所有系统功能开放（SystemOpen）、养成模块解锁、玩法接入最终都通过 Unlock 条件挂载到主角等级或主线推进进度上。主角等级表定义经验成长曲线、每次升级返还的体力（储能）、升级奖励道具，以及等级与主线章节/关卡的对应关系。

### 目标
反推并记录主角等级系统的完整规则：经验曲线、体力返还机制、升级道具奖励、等级-章节-关卡映射关系、以及挂载在本系统上的 SystemOpen 功能开放节点。

### 非目标
- 不涉及主线关卡的具体首通/重复奖励（另见主线关卡系统文档）
- 不涉及养成进度预估（另见养成节奏文档）
- 不涉及体力获取和消耗（储能）的完整产消链（另见经济框架文档）

## 核心规则

### R-01: 升级经验消耗
- **触发条件**：玩家获得玩家经验值（通关关卡/完成任务等），累积 EXP 达到升级门槛
- **前置限制**：无（经验溢出可连续升级）
- **操作流程**：战斗/任务产出玩家 EXP → 累加到 currentExp → currentExp ≥ UpgradeCostExp 时自动升级 → currentExp 溢出部分结转到下级
- **状态变化**：主角等级 +1；升级计数器更新
- **资源变化**：EXP 来源主要为关卡通关奖励（FallReward 中的玩家经验道具）、任务奖励、每日活跃奖励
- **UI 反馈**：主界面等级进度条更新；升级动画（弹出升级提示、返还内容展示）
- **异常边界**：一次获得大量 EXP 导致连升多级时，每级触发一次 UR-02（体力返还）和 UR-03（升级道具），动画需串联播放
- **落地点**：`PlayerLv.xlsx`（UpgradeCostExp 字段）→ `Vgame新手引导以及开启节奏.xlsx` 主角等级 sheet

### R-02: 升级体力返还
- **触发条件**：每次主角等级提升时自动触发
- **前置限制**：不适用
- **操作流程**：读取 PlayerLv.UpgradeGetPhy → 增加对应数量的体力（ItemId=3，储能）到玩家体力槽
- **状态变化**：玩家体力值增加
- **资源变化**：每次升级返还体力数量：初期较少（如 Lv1→2 返 10），中后期递增。体力上限 = 基础上限（如 120）+ 月卡加成
- **UI 反馈**：升级弹窗中显示「回复储能 ×N」；体力槽数字跳动增加
- **异常边界**：体力超过上限时的处理：溢出体力保留（当前观察可能无上限或上限足够高）；月卡体力上限增加后是否允许溢出部分回收（待确认）
- **落地点**：`PlayerLv.xlsx`（UpgradeGetPhy 字段）

### R-03: 升级道具奖励
- **触发条件**：每次主角等级提升时自动触发
- **前置限制**：不适用
- **操作流程**：读取 PlayerLv.UpgradeGetItem → 发放对应道具到背包（如钻石、抽卡券、角色经验等）
- **状态变化**：背包道具数量增加
- **资源变化**：升级奖励内容按等级阶段不同：
  - 前期（Lv1-15）：少量钻石 + 经验道具
  - 中期（Lv15-30）：钻石 + 抽卡券 + 养成材料
  - 后期（Lv30+）：较丰厚的钻石/抽卡券（作为升级激励）
- **UI 反馈**：升级弹窗中显示获得道具图标和数量
- **异常边界**：UpgradeGetItem 为空时不放道具；道具 ItemId 不存在时记录日志（不发）
- **落地点**：`PlayerLv.xlsx`（UpgradeGetItem 字段，格式为 PropType 数组：ItemId,Qty;ItemId,Qty）

### R-04: 等级-章节-关卡映射
- **触发条件**：配置定义，运行时用于展示主线可用范围和推荐等级
- **前置限制**：不适用
- **操作流程**：
  - Lv1-39 映射到 8 个章节，共 105 个关卡
  - 具体映射关系定义在 "主角等级" sheet 中
  - 每个主角等级对应一个"推荐推进到的关卡编号"
- **状态变化**：不适用
- **资源变化**：不适用
- **UI 反馈**：主线界面显示当前推荐推进的关卡（按主角等级匹配）；推荐等级显示在关卡面板上（RecommendLevel 字段）
- **异常边界**：等级超出映射表范围（>39）时不显示推荐关卡
- **落地点**：`Vgame新手引导以及开启节奏.xlsx` 主角等级 sheet → `UIlevel.xlsx`（RecommendLevel 字段）

### R-05: 系统功能开放（SystemOpen）
- **触发条件**：玩家等级达到 SystemOpen 关联的 Unlock 条件时自动触发
- **前置限制**：系统未被关闭（SystemOpen.CloseSys=false）；系统的 IsValid=true
- **操作流程**：等级提升 → Unlock 条件检查（UnLock_UUT_player_Grade）→ 条件满足 → OnSystemUnlock 事件 → 新系统入口开放 + 可选触发新手引导
- **状态变化**：对应系统从不可用变为可用
- **资源变化**：不适用
- **UI 反馈**：新功能入口按钮出现（EntranceResident 决定是否常驻）；部分系统开放时有全屏引导或气泡提示
- **异常边界**：CloseSys=true 时永不开放；IsValid=false 时功能隐藏
- **落地点**：下表 — 主角等级驱动的关键系统开放节点：

| 主角等级 | SystemOpen 系统 | UnlockId | 说明 |
|------|------|------|------|
| Lv1 | 背包/角色/主线 | 4004/4005/4008 | 立即开放 |
| Lv1 | 邮件 | 4001 | 通过 0-4 后 |
| Lv1 | 任务/商店/招募 | 4006/4012/4017 | 通过 1-1 后 |
| Lv1 | 挑战(资源本) | 4009 | 通过 1-6 后 |
| Lv1 | 好友/聊天 | 4002/4003 | 通过 1-10 后 |
| Lv20 | 竞技场 | 4112 | |
| Lv25 | 爬塔 | 4101 | |
| Lv27 | 专武 | 4016 | |
| Lv32 | 首领挑战 | 4104 | |
| Lv34 | 旅券副本 | — | |
| Lv37 | 大秘境 | 4102 | |

- 注：主线推进到达指定关卡也可单独触发系统开放（UnLock_UUT_Level），与等级开放为 OR 关系还是 AND 关系待确认

### R-06: 经验曲线特征
- **触发条件**：配置定义，非运行时触发
- **前置限制**：不适用
- **操作流程**：
  - 前期（Lv1-10）：升级较快，曲线平缓 — 作为新手期持续正反馈
  - 中前期（Lv10-20）：梯度上升，日活跃可推进 2-3 级
  - 中后期（Lv20-30）：稳步递增，日推进 1 级
  - 后期（Lv30-39）：明显放缓，需多日才能升 1 级
  - Lv39 后是否继续开放等级取决于版本节奏
- **状态变化**：不适用
- **资源变化**：不适用
- **UI 反馈**：不适用
- **异常边界**：不适用
- **落地点**：`PlayerLv.xlsx`（UpgradeCostExp）→ `Vgame新手引导以及开启节奏.xlsx` 养成进度预估 sheet

### R-07: 等级开放节奏与养成联动
- **触发条件**：配置定义，运行时通过 SystemOpen + Unlock 生效
- **前置限制**：不适用
- **操作流程**：主角等级 → 解锁新玩法 / 新养成系统 → 驱动玩家进入新的养成深度
- **状态变化**：不适用
- **资源变化**：不适用
- **UI 反馈**：不适用
- **异常边界**：不适用
- **落地点**：`Vgame新手引导以及开启节奏.xlsx` 开启节奏和引导 sheet → `SystemOpen.xlsx` + `Unlock.xlsx`

各养成系统随时间接入的大致节奏：
- 前期（Lv1-15）：角色升级、突破基础
- 中期（Lv15-27）：星盘解锁、专武接入、资源副本、竞技场
- 中后期（Lv27-37）：爬塔、旅券、首领挑战、大秘境
- 后期（Lv37+）：映射装备、装备深化（宝石/铭文）、高星高突破

## 技术方案
### 架构影响
主角等级模块为已有基础模块，无需新系统。

### 数据变更
| 表 | 变更类型 | 说明 |
|---|---|---|
| `PlayerLv.xlsx` | 已有 | 升级经验（UpgradeCostExp）、返还体力（UpgradeGetPhy）、升级道具（UpgradeGetItem） |
| `SystemOpen.xlsx` | 已有 | 系统开放定义（Id/Name/UnlockId/CloseSys/EntranceResident） |
| `Unlock.xlsx` | 已有 | 解锁条件（Id/UnLockType/UnLockParams），主角等级条件为 UnLock_UUT_player_Grade |
| `Vgame新手引导以及开启节奏.xlsx` | 已有 | 节奏母表：主角等级 sheet（等级-章节映射）+ 开启节奏和引导 sheet（功能开放排序）+ 养成进度预估 sheet（养成节奏） |

### 接口变更
无

## 风险评估
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 升级经验曲线设置不当导致前期升级太快（内容消耗过快）或后期太慢（卡点过长） | 中 | 高 | 用 `vgame-player-progression-simulator` 验证各阶段每日升级速度是否匹配内容供给 |
| 体力返还不合理：前期返太少（新手卡体力）、后期返太多（溢出浪费） | 低 | 中 | 模拟每日体力收支，确保升级返还在日常体力预算中占比合理 |
| 功能开放密度不均：某几级连开多个系统（理解成本过载）或某段长时间不开新系统（无聊） | 中 | 中 | 绘制 SystemOpen 密度热力图，每 5 级至少开放 1 个新功能 |
| SystemOpen 条件从主角等级改为关卡推进时，已有玩家可能因等级更高但未推关而错过开放 | 低 | 中 | 条件修改时评估现有玩家状态，必要时批量补偿开放 |
| 升级道具奖励与关卡首通奖励重复发放同一类道具（前期经济意外宽松） | 低 | 低 | 交叉核对 PlayerLv.UpgradeGetItem 与 level.FirstReward 的道具交集 |
