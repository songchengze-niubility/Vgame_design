## 背景与目标
### 背景
角色等级是玩家核心成长线，直接关联属性上限、技能解锁、装备槽位等。除主线首通经验外，日常角色经验缺口主要由经验副本填补。经验副本定位为高效率刷怪场：大量低威胁小怪、短战斗时长、稳定经验产出，区别于金币副本的"拾取驱动"体验。

### 目标
1. 提供每日角色经验道具（EXP 书）稳定产出，支撑玩家日常升级节奏
2. 9 档难度阶梯匹配玩家练度，低难度保底、高难度高效
3. 单局 60-90 秒短平快，适配"扫荡/速刷"体验
4. 经验道具产出分低/中/高三级品质，难度越高高品质掉率越高

### 非目标
- 不提供角色直接升级功能（产出的是经验道具，由玩家自行使用）
- 不替代主线首通经验奖励
- 不引入随机经验暴击（critical exp）等变数设计

## 核心规则
### R-01: 副本开放与解锁
- **触发条件**：通关主线 1-6
- **前置限制**：SystemOpen 表中经验副本条目开启；与金币副本同时解锁
- **操作流程**：
  1. 通关主线 1-6 → SystemOpen 解锁经验副本入口
  2. 玩家在主界面选择角色经验副本
  3. 进入 UIlevel 难度选择界面
- **状态变化**：`LevelType_Exp` → Opened
- **资源变化**：无
- **UI 反馈**：副本入口图标解锁；解锁动画/红点引导
- **异常边界**：与金币副本规则一致（SystemOpen 未配置则隐藏）
- **落地点**：SystemOpen 表、UIlevel.xlsx

### R-02: 体力消耗与每日次数
- **触发条件**：玩家确认进入指定难度
- **前置限制**：
  - 体力 ≥ 15（item_id=3）
  - 当日经验副本挑战次数 < 每日上限（3-5 次）
  - 玩家等级满足该难度最低要求
- **操作流程**：
  1. 校验体力 + 次数 + 等级
  2. 扣减体力 15，增量计数
  3. 加载 EquipmentLevel.xlsx 难度配置（怪物组、波次、经验掉落组）
- **状态变化**：体力-15；经验副本日次数+1；进入战斗态
- **资源变化**：`PlayerVitality(item_id=3) -= 15`
- **UI 反馈**：体力条更新；次数文本更新；加载战斗
- **异常边界**：
  - 体力/次数不足：标准弹窗提示
  - 网络超时扣体力未进战：补偿逻辑同金币副本
- **落地点**：UIlevel.xlsx、EquipmentLevel.xlsx、每日计数存储

### R-03: 效率刷怪战斗模式
- **触发条件**：进入经验副本关卡
- **前置限制**：NormalMode 或专用 ExpMode 激活
- **操作流程**：
  1. 场景加载：小怪大量生成（与金币副本不同，怪数量更多、个体更弱、无金币掉落）
  2. 战斗循环：Loop Zone 结构 — 小怪波 → 清场 → 下一波 → 循环 → 精英/Boss 收尾
  3. 每只怪击杀时按概率掉落经验道具（EXP_LOW / EXP_MID / EXP_HIGH），即时拾取
  4. 单局时长控制在 60-90s（高密度怪保证效率体验）
- **状态变化**：进入战斗；击杀计数递增；掉落物品列表累积
- **资源变化**：战斗中为临时数据，结算后写入背包
- **UI 反馈**：
  - 顶部 HUD：倒计时、击杀数、已获经验道具预览
  - 掉落物品拾取提示（图标 + 名称弹出）
  - 波次进度条（当前波/总波）
- **异常边界**：
  - 掉线：保留已拾取道具列表快照，重连恢复
  - 全清但时间未到：提前进入结算（不浪费玩家时间）
- **落地点**：EquipmentLevel.xlsx、战斗状态机、掉落表

### R-04: 经验道具结算
- **触发条件**：战斗结束（时间到 / 全清 / 手动退出）
- **前置限制**：服务端校验战斗数据合法性（击杀数、时长、伤害量）
- **操作流程**：
  1. 客户端上报：击杀数、获取的经验道具列表
  2. 服务端校验：击杀数 vs 怪物配置上限；时长合理性（不能短于理论最低清怪时间）
  3. 服务端确认掉落列表有效后，写入玩家背包
  4. 结算面板展示明细（各品质经验书数量、总量经验值预览）
- **状态变化**：战斗态 → 结算态 → 退出
- **资源变化**：`PlayerBag += [EXP_LOW * N1, EXP_MID * N2, EXP_HIGH * N3]`
- **UI 反馈**：结算面板 — 各品质道具图标+数量、总经验预览、品质高亮（HIGH 品质特殊动效）
- **异常边界**：
  - 校验不通过（可疑数据）：不发放奖励，记录日志 + 标记账号待人工审核
  - 背包满：道具发邮件，邮件有效期 30 天
- **落地点**：EquipmentLevel.xlsx 掉落组、背包系统、邮件系统

### R-05: 9 档难度阶梯
- **触发条件**：玩家在 UIlevel 面板选择难度
- **前置限制**：高难度逐级解锁（依赖玩家等级/主线进度）
- **操作流程**：
  1. 难度 1-9：怪物等级 10 → 130，怪物数量递增，EXP_HIGH 掉率递增
  2. 每档难度配置独立的怪物组、波次数、掉落概率组
  3. 产出曲线：难度 N+1 总经验产出 ≈ 难度 N × 1.15 ~ 1.3 倍
- **状态变化**：无持久变化
- **资源变化**：高难度 = 同体力换更多经验（性价比更高）
- **UI 反馈**：难度选择面板展示预览 — 推荐战力、怪物等级、经验产出范围（Min-Max）
- **异常边界**：难度配置缺失 → 该档位灰态不可选，不影响其他难度
- **落地点**：EquipmentLevel.xlsx、UIlevel.xlsx（LevelType=LevelType_Exp）

### R-06: 每日重置与产出上限
- **触发条件**：服务器每日 5:00 AM
- **前置限制**：无
- **操作流程**：
  1. 每日 5:00 AM 重置经验副本挑战次数
  2. 每日产出的经验道具量通过"次数上限 × 单次产出区间"自然控制
  3. 无额外全局产出上限（仅靠次数控制）
- **状态变化**：日次数归零
- **资源变化**：无
- **UI 反馈**：次数刷新为满；剩余次数倒计时提示
- **异常边界**：重置异常同金币副本（补偿全额次数）
- **落地点**：定时任务调度、Redis 计数器

## 技术方案
### 架构影响
- 战斗层：经验副本可复用 NormalMode，与金币副本的 CoinMode 不同（无特殊金币收集逻辑）
- 掉落系统：需支持按概率组的多品质掉落（LOW/MID/HIGH），此逻辑可复用至武器经验副本
- 反作弊：服务端校验击杀数 vs 理论上限（最大清怪速率 × 时长）

### 数据变更
| 表 | 变更类型 | 说明 |
|----|----------|------|
| EquipmentLevel.xlsx | 已有 | 难度配置：怪物组、波次、掉落概率组 |
| level.xlsx | 已有 | LevelType=LevelType_Exp |
| UIlevel.xlsx | 已有 | Vitality, 每日上限, LevelId 关联 |
| SystemOpen.xlsx | 已有 | 解锁条件配置 |
| Bag/Item | 已有 | EXP_LOW, EXP_MID, EXP_HIGH 物品定义 |
| PlayerDailyCounter | 已有 | exp_daily_count |

### 接口变更
- `EnterExpDungeon(difficulty_id: int) -> EnterResult`
- `SettleExpDungeon(session_id, kills, loot_list) -> SettleResult { items[], total_exp_preview }`
- `GetExpDungeonDailyStatus() -> { remaining, total, next_reset_time }`

## 风险评估
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 经验产出过高导致满级过快 | 中 | 高 — 缩短游戏生命周期 | 经济模拟覆盖 90 天升级曲线；观察期内产出可热更 |
| 外挂加速刷经验 | 高 | 高 — 破坏公平 | 服务端最小通关时间校验；击杀速率异常检测；次数上限兜底 |
| 品质掉率感知差（玩家觉得不掉 HIGH） | 中 | 低 — 舆情 | 结算面板展示概率明细；保底机制（N 局必出 HIGH 可选） |
| 背包邮件溢出 | 低 | 低 | 邮件 30 天过期自动清理；背包满提示优化（引导清理） |

| 无特殊 GameMode，复用 NormalMode | 当前规则 | 与金币副本 CoinMode 不同 |
| 单局时长控制在 60-90s | 待确认 | 需实测验证怪物密度是否达标 |
| 无单独全局产出上限，仅依靠次数控制 | 待确认 | 是否需加全服产出上限待定 |
