## 背景与目标

### 背景

主线是 Vgame 的核心推进轴。所有功能开放（SystemOpen）、养成节奏、前期经济供给都挂载在主线推进进度上。当前已实装 5 章 60 关（Ch0 序章 5关 + Ch1-4 各 10/15/15/15关），采用横版跑酷 + Loop 波次战斗的关卡结构。

### 目标

反推并记录当前主线关卡的完整规则：推进逻辑、首通/重复奖励结构、难度线、关卡结构（Loop机制）、与其他系统的引用关系。

### 非目标

- 不涉及关卡编辑器操作说明
- 不涉及新章节设计提案
- 不涉及高难主线（困难模式，另见独立文档）

## 核心规则

### R-01: 关卡推进

- **触发条件**：玩家在 MainLevel 地图节点上选择下一关
- **前置限制**：前一关已通关；满足 ChapterLevel.Unlock 条件
- **操作流程**：选择关卡 → 检查体力 → 加载场景 → 战斗 → 通关/失败
- **状态变化**：通关后关卡状态变为"已通关"；首次通关触发首通奖励；三星达成触发星级奖励
- **资源变化**：扣除体力 10（道具ID=3）；通关后获得奖励
- **UI 反馈**：MainLevel 地图显示关卡连通状态（已解锁/未解锁/已通关）；关卡面板显示推荐等级、奖励预览
- **异常边界**：体力不足时弹窗提示购买/等待恢复；关卡加载失败时回退到地图界面
- **落地点**：`level.xlsx` + `UIlevel.xlsx` + `MainLevel.xlsx`，运行时 `LLevelMgr.cs`

### R-02: 首通奖励

- **触发条件**：关卡首次通关
- **前置限制**：关卡状态为"未通关"
- **操作流程**：通关结算 → 读取 level.FirstRewardDrop（DropId） → 查阅 Drop 逻辑表 → 发放奖励包
- **状态变化**：关卡标记为"已通关"；首通奖励变为不可领取
- **资源变化**：
  - Ch0 (5关): 钻石20, 金币10000(Boss 20000), 经验低×6(Boss ×10), 星盘/升阶材料
  - Ch1 (10关): 钻石20, 金币20000(Boss 40000), 经验中×6(Boss ×10), 星盘/升阶材料
  - Ch2 (15关): 钻石20, 金币30000(Boss 60000), 经验中×8(Boss ×12), 星盘中+映射装备
  - Ch3 (15关): 钻石20, 金币40000(Boss 80000), 经验中×12(Boss ×20), 星盘高+映射装备
  - Ch4 (15关): 钻石20, 金币50000(Boss 100000), 经验中×16(Boss ×24), 星盘高+映射装备
- **UI 反馈**：结算界面显示首通奖励列表；关卡面板上首通标记
- **异常边界**：首通奖励 DropId 不存在时降级为空奖励（记录日志）
- **落地点**：`level.xlsx` (FirstRewardDrop 字段) → `Drop` 逻辑表（8 文件合并） → `UIlevel.xlsx`（FirstRewardDrop + LevelFPDrop 展示）

### R-03: 重复通关（常规奖励）

- **触发条件**：关卡已通关，再次进入
- **前置限制**：同上体力消耗；无次数限制（主线刷无限）
- **操作流程**：同 R-01 流程
- **状态变化**：不改变通关状态
- **资源变化**：获得 FallRewardDrop 中的玩家经验+掉落材料（不含钻石和首通额外奖励）
- **UI 反馈**：结算界面展示常规奖励；支持多倍结算（MultiSettlementTimes）
- **异常边界**：同 R-01
- **落地点**：`level.xlsx` (FallRewardDrop) → `Drop` 逻辑表 → `UIlevel.xlsx` (FallRewardDrop + LevelPPDrop)

### R-04: 三星评价

- **触发条件**：关卡通关
- **前置限制**：无
- **操作流程**：结算时比对 LevelTask 三星条件 → 达成则升星 → 累计所有关卡星级
- **状态变化**：关卡星级从 0→1→2→3；章节累计星级更新
- **资源变化**：累计星级达标时触发 ChapterStarReward（一次性，钻石+抽卡券+装备）
- **UI 反馈**：关卡面板显示当前星级；章节界面显示累计星级和奖励进度
- **异常边界**：三星条件（LevelCondition）不存在时不做评星
- **落地点**：`level.xlsx` (LevelTask) → `LevelCondition.xlsx` → `ChapterStarReward.xlsx`

### R-05: 章节进度奖励

- **触发条件**：章节内关卡通关进度达到 20%/40%/60%/80%/100%
- **前置限制**：无
- **操作流程**：通关后检查进度 → 达标时弹窗发放
- **资源变化**：每档进度发放角色抽卡券普通 ×1
- **UI 反馈**：章节界面显示进度条和阶段奖励预览
- **异常边界**：进度计算取已通关关卡数/章节总关卡数
- **落地点**：`ChapterTasks.xlsx` (ChapterId + DropId + TaskIdList)

### R-06: 关卡难度线

- **触发条件**：不触发，为配置定义
- **前置限制**：不适用
- **操作流程**：每关通过怪物等级 + HP系数 + ATK系数 映射难度
- **状态变化**：不适用
- **资源变化**：不适用
- **UI 反馈**：不适用
- **异常边界**：不适用
- **落地点**：`Vgame基本战斗框架.xlsx` — 玩法难度线 sheet。怪物等级映射表 + `level.xlsx` (HpCoefficient / AttackCoefficient / ArmorCoefficient)

## 技术方案

### 架构影响

主线关卡已实现，无需新模块。关卡运行时由 `LLevelMgr.cs` 管理（地形流式加载、Loop 波次、实体生命周期）。

### 数据变更

| 表 | 变更类型 | 说明 |
|---|---|---|
| `level.xlsx` | 已有 | 关卡基础定义（LevelId/LevelDataPath/Vitality/系数/奖励引用） |
| `UIlevel.xlsx` | 已有 | UI 层（LevelType/ChapterId/奖励 DropId/推荐等级/怪物展示） |
| `MainLevel.xlsx` | 已有 | 章节地图节点布局和 IsBoss 标记 |
| `ChapterLevel.xlsx` | 已有 | 章节信息和解锁条件 |
| `ChapterStarReward.xlsx` | 已有 | 星级累计奖励 |
| `ChapterTasks.xlsx` | 已有 | 章节进度任务 |
| `LevelCondition.xlsx` | 已有 | 三星条件定义 |

### 接口变更

无

## 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 首通奖励数值修改导致前期经济失衡 | 中 | 高 | 修改前跑 `vgame-player-progression-simulator` 模拟后果 |
| 关卡怪物等级映射改偏导致难度跳跃 | 中 | 中 | 修改后用 `vgame-battle-tuning-helper` 验 TTK |
| Drop 表公式列被误写 | 高 | 高 | 写入前查 `config-schema.json` 确认 `formula_columns` |
| 新增关卡时 MainLevel 节点链路断 | 中 | 中 | 验证 NextLevel 链完整 |
