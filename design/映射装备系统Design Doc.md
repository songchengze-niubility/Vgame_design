## 背景与目标
### 背景
映射装备是游戏终局（Endgame）的核心追深系统，定位为"高品质追求+极致随机深度"的长期养成载体。8个独立槽位（武器/头部/护甲/手套/腰带/鞋子/装置/饰品）对应不同的主属性固定分配，副属性通过重度随机机制生成。系统引入孔位（Socket）概念，可镶嵌宝石与铭文，与宝石/铭文体系形成装备维度的立体养成回路。通过洗练（Refinement）机制允许有限定向优化，超限（Transcend）机制在产出层提供极小概率的数值突破，构成终局玩家的核心动力。

品质按等级档位映射：蓝lv30、紫lv40、金lv50、红lv60，品质系数分别为0.3/0.5/0.75/1。红色装备固定4条属性类型 + 1条技能词缀（来自模板）。190条词缀涵盖机制型与属性型，覆盖所有伤害标签。

### 目标
- 提供终局玩家持续数月的高随机深度追逐目标
- 通过洗练降低部分随机挫败感，在随机与可控间取得最佳平衡
- 超限机制作为锦上添花的"欧皇时刻"，提供强烈的情绪峰值
- 词缀系统跨伤害标签全覆盖，支撑多流派构建
- 宝石/铭文嵌入形成二阶随机，将装备深度延长至2层以上

### 非目标
- 不支持装备交易/AH（维持绑定生态）
- 不支持主属性洗练（仅副属性4词条可以洗练，类型不变）
- 超限仅发生于掉落/生成阶段，不支持后天道具赋予超限
- 铭文不支持继续合成升级（仅限于1-3级初始赋予）

## 核心规则
### R-01: 8槽位装备装配与主属性固定
- **触发条件**：角色达到装备解锁等级，进入映射装备界面
- **前置限制**：玩家等级 ≥ 各槽位解锁等级阈值；角色处于非战斗状态；目标装备品质与槽位允许品质匹配
- **操作流程**：1) 进入映射装备页签（8个槽位呈环状或列表展示）；2) 点击目标槽位（武器/头部/护甲/手套/腰带/鞋子/装置/饰品）；3) 从背包映射装备列表中选择目标装备；4) 确认装配；5) 旧装备（如有）退回背包
- **状态变化**：槽位 `Empty → Occupied(EqpId)` 或 `Occupied(Old) → Occupied(New)`；主属性固定生效（不等同于旅券的自由选择）；角色战力更新
- **资源变化**：无直接货币消耗；装备绑定状态设为已装备
- **UI 反馈**：槽位显示装备图标+品质光效；主属性与副属性展开；战力变化预览（绿色+/红色-）与当前装备差值
- **异常边界**：跨品质等级限制（lv30角色不可装备lv60红装）；同槽位不能同时装备多件；装备被锁定/标记为材料时不可装配；交换操作（2件装备直接互换槽位）需前端做事务处理
- **落地点**：`MappingEquipManager`，`SpecialEquipment.xlsx` 装备基础配置，`EquipmentLevel.xlsx` 等级解锁表

### R-02: 品质/等级梯度与品质系数
- **触发条件**：映射装备生成（掉落/奖励发放/系统产出）
- **前置限制**：玩家等级或关卡等级决定产出品质上限；各品质掉率按掉落池配置分配；红装备需额外满足模板池不为空
- **操作流程**：1) 按品质权重roll出品质（蓝0.3/紫0.5/金0.75/红1）；2) 根据品质决定装备等级（蓝=lv30/紫=lv40/金=lv50/红=lv60）；3) 品质系数写入装备实例，贯穿后续属性计算
- **状态变化**：品质确认→属性基准值计算→实例创建→入库
- **资源变化**：消耗掉落机会×1；新装备入库
- **UI 反馈**：品质色边框（蓝/紫/金/红）；装备标签显示品质名+"Lv.XX"；战力估值按品质系数差异化表现（红色显著高于金色）
- **异常边界**：品质系数不符合[0.3, 0.5, 0.75, 1]任一档位时QA报警并回退至默认（蓝0.3）；红装备生成时模板池为空→降级为金品质替代产出
- **落地点**：`MappingEquipGenerator`，品质系数常量 `SpecialEquipment.xlsx`，`SpecialLevel.xlsx` 等级对照

### R-03: 副属性重度随机与4+1红装词缀模板
- **触发条件**：映射装备生成（品质已确定）后续属性填充阶段
- **前置限制**：主属性已按槽位确定；副属性池配置已加载（`SpecialEquipmentEffectDrop.xlsx`）；红装需额外加载词缀模板池（190行，`英雄设计.xlsx`）
- **操作流程**：1) 非红装备：从副属性池中随机抽取N条副属性（N依品质变化，如蓝2/紫3/金4），每条随机数值；2) 红装备：固定抽取4条副属性类型 + 从词缀模板池随机1条技能词缀；3) 各条副属性的数值 = 基础值 × 品质系数 × 部位系数 × 随机系数；4) 红装词缀：独立于副属性，不可被洗练影响
- **状态变化**：装备属性字段填充完毕；词缀字段赋值；战力预估计算
- **资源变化**：无额外消耗（生成过程的内部流程）
- **UI 反馈**：属性详情展示所有副属性条+数值；红装额外展示技能词缀栏（特殊光效/铭文框）；属性品质颜色（数值越高颜色越深）
- **异常边界**：抽取到重复副属性类型时跳过重新抽取（非红装不可有重复属性类型）；词缀池为空时红装降级产出金；副属性值超出预设上下限时钳位
- **落地点**：`SubAttrRoller`，`SpecialEquipmentEffectDrop.xlsx` 副属性池，`英雄设计.xlsx` Sheet `映射装备词缀` 190行词缀

### R-04: 孔位系统与镶嵌
- **触发条件**：装备生成阶段确定孔位数量；玩家主动镶嵌宝石/铭文
- **前置限制**：孔位数量 = 0~N随机（N按品质递增：蓝上限2/紫上限3/金上限4/红上限5）；装备自身未被锁定；背包中有可用宝石或铭文；宝石/铭文等级符合孔位兼容规则
- **操作流程**：1) 打开装备详情→孔位分页；2) 点击空闲孔位；3) 从背包选择宝石或铭文；4) 确认镶嵌；5) 属性加成即时生效
- **状态变化**：孔位 `Empty → Filled(GemId/InscriptionId)`；装备属性面板更新（加总宝石/铭文加成）；战力重算
- **资源变化**：消耗宝石/铭文×1；装备属性永久附加（可替换，替换时原宝石/铭文销毁）
- **UI 反馈**：孔位图标亮/暗（已镶嵌/空闲）；镶嵌物名称+等级显示；替换确认弹窗："原镶嵌物将销毁，确认替换？"；一键插入推荐宝石（按玩家流派）
- **异常边界**：孔位数随机为0时该装备无法镶嵌任何物；替换时需二次确认；已镶嵌宝石合成/洗练时不影响装备属性（装备独立存储）；铭文不可继续合成，替换后直接销毁不可退还
- **落地点**：`SocketManager`，孔位随机规则 `SpecialEquipment.xlsx`

### R-05: 洗练系统（Refinement）
- **触发条件**：玩家对红色/金色品质装备发起洗练操作
- **前置限制**：仅4条属性值槽位可洗练（非属性类型，非词缀槽）；装备品质 ≥ 金色；消耗洗练石×1；玩家勾选至少1条待洗练属性；支持锁定（额外消耗锁定符×1/每条锁定属性）
- **操作流程**：1) 选择装备→洗练入口；2) 勾选需要重新随机的属性条（可多选）；3) 勾选需要锁定的属性条（可选）；4) 预览消耗（洗练石 + 锁定符×N）；5) 点击洗练；6) 服务器生成新属性值（勾选属性条重新随机，锁定属性条保持原值）；7) 展示"洗练前/洗练后"对比界面；8) 玩家选择"确认替换"或"保留原属性"
- **状态变化**：若确认 → 装备属性数据更新为新值；若保留 → 属性不变但消耗品不返还；洗练次数计数+1
- **资源变化**：消耗洗练石×1（必扣）；消耗锁定符×N（可选）；洗练历史记录新增1条
- **UI 反馈**：洗练对比面板（左旧右新），变化条高亮绿色/红色；保留/确认双按钮；锁定图标（小锁）；洗练次数显示（部分统计：历史总次数+今日次数）
- **异常边界**：不勾选任何属性条时洗练按钮不可点击；锁定所有属性时等同于无效洗练（提示"至少需要一条未锁定属性"）；连续X次洗练无提升时触发安抚机制提示；洗练后属性值与当前完全相同时弹提示但不阻止确认
- **落地点**：`RefineManager`，洗练石/锁定符在 `SpecialEquipment.xlsx` 消耗品表

### R-06: 超限系统（Transcend）
- **触发条件**：映射装备生成（掉落/奖励/系统产出）时概率判定
- **前置限制**：仅生成阶段触发（不可后天道具赋予）；每件装备最多超限1次；超限概率极低（预设0.1%~1%区间，按品质分档：蓝0.1%/紫0.3%/金0.5%/红1%）
- **操作流程**：1) 装备生成→常规属性填充完毕→超限判定（独立概率roll）；2) 若命中→该装备标记 `Transcend=true`；3) 超限装备的所有属性值突破正常上限（额外×1.1~1.2系数）；4) 属性面板标星（☆标记）；5) 超限状态永久且不可继承/转移
- **状态变化**：`Normal → Transcended`；属性上限突破（比同类装备高10%~20%）；装备稀有度视觉升级（额外光效）
- **资源变化**：无额外消耗（纯概率事件）
- **UI 反馈**：装备图标超限特效（流光/辉光）；属性名旁"☆"标记；战力估值以超限系数加成显示；掉落公告（超限装备产出时全服/公会广播）
- **异常边界**：超限不可堆叠（`Transcend=true` 后不再判定）；伪随机保护：累计N次未超限后概率递增至触发后重置；超限系数超过预设范围（1.1~1.2）时钳位；超限装备不可作为合成/洗练材料进行二次修改
- **落地点**：`TranscendRoller`（生成流程内置），超限系数配置 `SpecialEquipment.xlsx`

### R-07: 洗练/掉落经济平衡
- **触发条件**：系统持续运营阶段的定量监控
- **前置限制**：洗练石产出速率和消耗速率数据积累 ≥ 1个月；超限产出比例统计周期 = 1周
- **操作流程**：1) 统计洗练石日均产出总量 / 日均消耗总量；2) 洗练石转化率（洗练→有效提升次数/总洗练次数）；3) 超限装备产出占比 / 活跃用户持有率；4) 设定阈值警告：洗练石堆积率>150%时考虑增加消耗出口；洗练提升率<5%时考虑提高洗练波动下限；超限持有率<0.1%时考虑微调概率
- **状态变化**：指标监控 → 触发阈值 → 运维工单 → 配表调整
- **资源变化**：配表热更（不涉及客户端更新）
- **UI 反馈**：仅在运维后台展示，玩家无感知
- **异常边界**：需区分新服/老服数据差异；季节性活动期间的短期波动不作为调整依据（需活动结束后判定）；调整时不得影响已有装备的超限/洗练属性存储格式
- **落地点**：运维指标面板 → `SpecialEquipment.xlsx` 配表参数

## 技术方案
### 架构影响
- 新增 `MappingEquipManager`（与旅券 `TicketManager` 平行）、`RefineManager`、`SocketManager` 模块
- 装备系统需抽象为通用装备框架，映射装备与旅券共享底层装备存储但逻辑分离
- 洗练需要"比较→选择"的两段式交互协议，需服务器保存洗练中间态
- 超限判定在装备生成管线末尾作为独立Pass，确保不影响其他生成流程

### 接口变更
- `POST /mapping/equip` — 装配装备
- `POST /mapping/unequip` — 卸下
- `POST /mapping/refine` — 发起洗练
- `POST /mapping/refine/confirm` — 确认/拒绝洗练结果
- `POST /mapping/socket/embed` — 镶嵌宝石/铭文
- `POST /mapping/socket/replace` — 替换镶嵌物
- `GET /mapping/detail?equipId=` — 获取装备详情
- `Push: TranscendBroadcast` — 超限全服广播

## 风险评估
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 随机深度过高导致玩家挫败退出（Endgame空洞） | 高 | 极高 | 洗练提供可控性出口；超限提供正向情绪峰值；宝石/铭文可成长作为确定性提升路径 |
| 洗练石经济崩溃（产出>消耗→边际效应归零） | 中 | 高 | 监控指标预警；洗练石消耗可随时间阶梯式增长（递增消耗）；定期配表调整 |
| 超限装备过于稀有/泛滥 → 终局动力失衡 | 低 | 极高 | 伪随机机制+概率微调能力；超限仅作用于数值突破不改变玩法机制 |
| 红装词缀平衡性失控（某词缀过强） | 高 | 高 | 190条词缀需逐一数值校验；上线后预留调整窗口；设定词缀使用率阈值告警 |
| 8槽位装配复杂度与UI/UX理解成本过高 | 中 | 中 | 新手指引逐步解锁槽位；推荐装配功能降低选择负担 |
