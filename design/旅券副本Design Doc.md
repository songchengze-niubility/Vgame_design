## 背景与目标
### 背景
旅券副本是 Vgame 中面向中期玩家的 Boss 挑战玩法。玩家到达 34 级后解锁 4 个独立 Boss 关卡，每个关卡配置独立 Boss、专属战斗节奏和旅券装备掉落池。该玩法是旅券装备的稳定产出渠道，承载玩家中期装备成长的核心动力。

### 目标
- 提供差异化 Boss 战体验（4 个 Boss 各具战斗节奏和机制）
- 提供旅券装备稳定掉落来源
- 通过时间限制（60-120s）验证玩家 DPS 和生存能力
- 作为玩家中期装备验证和提升的回馈闭环

### 非目标
- 非多人合作玩法（纯单人 Boss 挑战）
- 非赛季周期性刷新（暂不在本设计范围）
- 非排行榜 / 竞速玩法

## 核心规则
### R-01: 副本解锁
- **触发条件**：玩家查看旅券副本入口
- **前置限制**：玩家等级 >= 34
- **操作流程**：玩家到达 34 级时自动解锁入口；入口点亮后玩家可选择任一已解锁 Boss
- **状态变化**：入口状态由 `Locked` → `Unlocked`
- **资源变化**：无直接资源消耗
- **UI 反馈**：主界面旅券副本入口解锁动画；入口图标从灰态点亮
- **异常边界**：等级刚好 34 时需即时刷新解锁状态（无需重登）；跨版本回退等级不会重新锁定
- **落地点**：旅券副本入口 UI 面板

### R-02: Boss 关卡选择
- **触发条件**：玩家点击旅券副本入口
- **前置限制**：当前无正在进行的旅券副本战斗
- **操作流程**：界面展示 4 个 Boss 卡片（策略 I/II/III/IV），玩家选择其一进入；每个 Boss 标注推荐战力和掉落预览
- **状态变化**：选择关卡 → `EnteringCombat`；确认后锁定关卡配置
- **资源变化**：扣除入场体力 / 券（具体数值由经济表定义）
- **UI 反馈**：Boss 卡片展示头像、名称、推荐战力、掉落图标列表
- **异常边界**：入场资源不足时按钮灰态并提示；若结算时断线，服务器保证掉落写入邮件
- **落地点**：关卡选择 UI → 战斗场景加载

### R-03: 单人 Boss 战斗
- **触发条件**：关卡加载完成、Boss 生成
- **前置限制**：关卡配置已锁定，Boss 数据从 `BossBattleTime.xlsx` 读取
- **操作流程**：单一 Boss 战，玩家使用当前上阵角色进行战斗；Boss 可在 HP 阈值切换行为阶段（Phase）
- **状态变化**：`CombatStarted` → 战斗中（Boss 阶段切换） → `CombatEnded`
- **资源变化**：无实时资源（战斗中不掉落）；战斗数据上传结算服务器
- **UI 反馈**：Boss HP 条、阶段名称、倒计时、伤害数字
- **异常边界**：超时未击杀则判定失败；战斗中途客户端断线，服务器保留战斗快照允许重连；残留异常状态在结算时清理
- **落地点**：战斗场景 → 结算

### R-04: 战斗时间限制
- **触发条件**：Boss 战斗开始
- **前置限制**：关卡对应的限时配置从 `BossBattleTime.xlsx` 读取（60-120s）
- **操作流程**：倒计时从战斗开始运行；计时器到达 0 时若 Boss 未死亡则判定失败
- **状态变化**：`CombatStarted` → 倒计时结束且 Boss 存活 → `CombatFailed`
- **资源变化**：失败情况下入场资源不返还；不发掉落
- **UI 反馈**：屏幕顶部/中央倒计时显示；最后 10 秒红色闪烁警告
- **异常边界**：网络延迟导致的客户端与服务器时间差由服务器权威时间裁决；倒计时暂停无手段（不可作弊暂停）
- **落地点**：战斗 HUD

### R-05: 旅券装备掉落
- **触发条件**：Boss 被击杀
- **前置限制**：关卡 DropId 在 20001101-20001110（地下城 1）至 20004101-20004110（地下城 4）范围内；掉落规则表 `Drop_dungeon.xlsx`
- **操作流程**：服务器根据 DropId 随机掉落 —— 首次通关固定额外掉落（首通奖励），后续为常规掉落；掉落组包含旅券装备 + 辅材
- **状态变化**：掉落物写入玩家背包
- **资源变化**：`+ 掉落装备`、`+ 辅材`（货币/材料）
- **UI 反馈**：结算界面掉落卡片展示（首通标示特殊样式）；背包自动新增物品
- **异常边界**：掉落生成时背包已满则通过邮件补发；首通记录不可重复触发（服务器幂等判断）
- **落地点**：结算界面 → 背包 / 邮件

### R-06: 首通与复刷差异化
- **触发条件**：玩家首次完成某一 Boss 关卡
- **前置限制**：服务器记录每个 Boss 的首通状态（独立维度）
- **操作流程**：首次通关时额外发放首通宝箱；后续同关卡重复通关仅获得常规掉落（掉率/数量不同）
- **状态变化**：关卡首通标记 `FirstClear=True → False`（不可逆）
- **资源变化**：首通宝箱内含旅券装备（固定品阶）+ 额外货币；复刷无宝箱
- **UI 反馈**：首通关卡在列表显示 "已首通" 角标；结算时首通宝箱开启动画
- **异常边界**：首次通关时若背包已满导致宝箱无法领取，宝箱保留在待领取队列直到玩家手动领取或背包有空位
- **落地点**：结算 + 关卡列表 UI

### R-07: Boss 战斗节奏与阶段机制
- **触发条件**：Boss 进入特定 HP 阈值
- **前置限制**：Boss 阶段定义表（BossBattleTime.xlsx 或专用阶段配置表）
- **操作流程**：Boss 在不同 HP% 触发阶段切换 —— 例如满血阶段（100%-60%）：常规攻击模式；半血阶段（60%-30%）：增加 AoE 机制或怒气技能；残血阶段（30%-0%）：狂暴，攻速/伤害提升
- **状态变化**：Boss Phase `Phase_1 → Phase_2 → Phase_3`
- **资源变化**：无直接资源变化
- **UI 反馈**：阶段切换时 Boss 模型变化（颜色/特效）、屏幕文字提示阶段名称
- **异常边界**：若由于过量伤害一次跳过多个阶段（如 100%→0%），仍按顺序触发所有阶段切换事件但每个阶段仅 1 帧 —— 掉落正常发放
- **落地点**：Boss 战斗 AI/阶段控制器

## 技术方案
### 架构影响
- 战斗层新增 `LevelType_Strategy_First/Second/Three/Four` 关卡类型支持
- 掉落层复用 `Drop_dungeon.xlsx` 配置，扩展 DropId 20001101-20004110
- 首通记录需要数据库新增字段：`player_boss_firstclear`（player_id, boss_id, cleared, timestamp）

### 数据变更
| 表 | 变更类型 | 说明 |
|----|----------|------|
| BossBattleTime.xlsx | 已有 | 4 Boss 限时配置 |
| Drop_dungeon.xlsx | 已有 | DropId 20001101-20004110 掉落组 |
| player_boss_firstclear | 新增 | 记录玩家对每个 Boss 的首通状态 |

### 接口变更
- 战斗初始化：传入 LevelType + BossId，返回战斗配置
- 战斗结算：传入战斗数据（伤害、时长），返回掉落列表 + 首通标示

## 风险评估
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| Boss 机制平衡导致某些 Boss 过难/过易 | 中 | 中 | 通过 BossBattleTime 和阶段参数表热更调参 |
| 掉落概率过于稀疏导致装备卡进度 | 低 | 中 | 首通保底 + 保底计数器 |
| 首通标记被重复触发 | 低 | 高 | 服务器幂等校验，数据库唯一约束 |
