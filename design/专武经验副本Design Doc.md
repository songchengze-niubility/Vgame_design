## 背景与目标
### 背景
专武（专属武器）是角色的核心战力组件，独立于角色等级的养成线。专武等级直接影响武器技能倍率、基础属性加成等。专武经验道具产出于独立副本，与角色经验副本、金币副本形成三角式日常资源框架。由于专武养成的数值权重较高，该副本开放时间晚于前两者（玩家等级 27），作为中期养成内容解锁。

### 目标
1. 提供每日专武经验道具稳定产出，支撑中期玩家的专武养成节奏
2. 延后解锁（PLv.27）控制前期资源投放节奏，避免新手信息过载
3. 与角色经验副本相同结构（9 档难度、效率刷怪、品质掉落），降低学习成本
4. 单局 60-90 秒保持日常效率，不增加日活负担

### 非目标
- 不产出通用武器经验（仅产出专武经验道具，两者区分）
- 不替代武器强化材料的其他获取途径（如活动、商店）
- 不引入武器等级突破等关联系统

## 核心规则
### R-01: 延后解锁（PLv.27）
- **触发条件**：玩家账号等级达到 27 级
- **前置限制**：
  - 主线 1-6 已通关（SystemOpen 基础解锁已触发）
  - 玩家等级 ≥ 27
  - 金币副本和角色经验副本已解锁（系统前置链完整）
- **操作流程**：
  1. 玩家达到 27 级时，客户端向服务端查询解锁状态
  2. 服务端校验 `PlayerLevel >= 27`
  3. SystemOpen 对应条目置为 Opened，客户端显示专武经验副本入口
- **状态变化**：`LevelType_Weapon` → Opened
- **资源变化**：无
- **UI 反馈**：副本入口图标解锁 + 红点引导 + 首次解锁弹窗提示"专武经验副本已开放！"
- **异常边界**：
  - 玩家通过经验道具跃升跳过 27 级升级事件 → 升级回调中补检 SystemOpen 解锁
  - PLv.27 前通过漏洞进入副本 → 服务端二次校验玩家等级，不满足则拒绝进入
- **落地点**：SystemOpen 表（解锁条件=PLv.27）、UIlevel.xlsx

### R-02: 体力消耗与每日次数
- **触发条件**：玩家确认进入指定难度
- **前置限制**：
  - 体力 ≥ 15（item_id=3）
  - 当日专武经验副本挑战次数 < 每日上限（3-5 次）
  - 玩家等级满足该难度要求
- **操作流程**：
  1. 校验体力 + 次数 + 等级
  2. 扣减体力 15，增量专武副本日计数
  3. 加载 EquipmentLevel.xlsx 中 `LevelType_Weapon` 难度配置
- **状态变化**：体力-15；专武副本日次数+1；进入战斗
- **资源变化**：`PlayerVitality(item_id=3) -= 15`
- **UI 反馈**：体力条更新；次数文本更新
- **异常边界**：同金币/经验副本的体力不足、次数不足、网络超时补偿逻辑
- **落地点**：UIlevel.xlsx、EquipmentLevel.xlsx、每日计数存储

### R-03: 专武经验道具掉落
- **触发条件**：战斗中击杀怪物
- **前置限制**：怪物配置表已关联专武经验掉落组（Weapon EXP DropGroup）
- **操作流程**：
  1. 大量小怪波次循环，配合精英/Boss 收尾
  2. 每只怪死亡时按概率组掉落专武经验道具（WEAPON_EXP_LOW / MID / HIGH）
  3. 即时拾取，客户端预览累积数量
  4. 战斗结束时结算写入背包
- **状态变化**：战斗中掉落列表累积（临时）
- **资源变化**：结算后 `PlayerBag += [WEAPON_EXP_LOW, WEAPON_EXP_MID, WEAPON_EXP_HIGH]`
- **UI 反馈**：HUD 展示已获专武经验道具数量 + 品质分布；拾取时专武图标弹出提示
- **异常边界**：
  - 掉落组配置错误掉出角色经验道具 → 配置 CI 校验：`LevelType_Weapon` 只关联 Weapon EXP DropGroup
  - 背包满 → 邮件发放（30 天有效期）
- **落地点**：EquipmentLevel.xlsx 掉落组、背包系统

### R-04: 9 档难度阶梯
- **触发条件**：玩家在 UIlevel 面板选择难度
- **前置限制**：难度 1-9 逐级解锁，与金币/角色经验副本的解锁规则对齐
- **操作流程**：
  1. 难度 1 → 9：怪物等级 10 → 130
  2. 每档配置：怪物组、波次数、掉落概率组（HIGH 掉率随难度递增）
  3. 产出曲线：难度 N+1 总武器经验产出 ≈ 难度 N × 1.15 ~ 1.3
- **状态变化**：无
- **资源变化**：高难度同体力换更多经验
- **UI 反馈**：难度选择面板展示推荐战力、怪物等级、武器经验产出预览
- **异常边界**：同经验副本 — 配置缺失则灰态不可选
- **落地点**：EquipmentLevel.xlsx 中 Weapon EXP 难度行、UIlevel.xlsx（LevelType_Weapon）

### R-05: 结算与反作弊
- **触发条件**：战斗结束（时间到 / 全清 / 退出）
- **前置限制**：服务端校验
- **操作流程**：
  1. 上报击杀数、武器经验道具列表
  2. 服务端校验：击杀数 ≤ 地图怪物总数；战斗时长 ≥ 理论最短清怪时间（防加速）
  3. 通过后发放道具到背包；不通过则记录 + 标记
- **状态变化**：战斗 → 结算 → 退出
- **资源变化**：`PlayerBag += weapon_exp_items[]`
- **UI 反馈**：结算面板 — 各品质道具数量、总武器经验预览
- **异常边界**：校验失败不予发放，人工审核标记
- **落地点**：服务端结算服务、反作弊日志

### R-06: 每日重置与独立计数器
- **触发条件**：服务器每日 5:00 AM
- **前置限制**：无
- **操作流程**：
  1. 专武经验副本日计数独立于金币/角色经验副本（三者日上限互不干扰）
  2. 每日 5:00 AM 重置
- **状态变化**：专武副本日计数归零
- **资源变化**：无
- **UI 反馈**：次数刷新；倒计时提示
- **异常边界**：重置异常补偿全额次数
- **落地点**：定时任务、Redis 计数器（weapon_exp_daily_count）

## 技术方案
### 架构影响
- 复用角色经验副本的完整框架（NormalMode、掉落系统、结算系统）
- 配置层区分：`EquipmentLevel.xlsx` 中通过 LevelType 区分 Weapon vs Exp 副本
- 掉落组需新增专武经验道具 ID，确保不与角色经验道具混淆
- 解锁条件增加 PLv.27 二次校验（服务端强校验）

### 数据变更
| 表 | 变更类型 | 说明 |
|----|----------|------|
| EquipmentLevel.xlsx | 已有 | LevelType=LevelType_Weapon 的难度配置行 + 武器经验掉落组 |
| level.xlsx | 已有 | LevelType=LevelType_Weapon 条目 |
| UIlevel.xlsx | 已有 | Vitality=15, 每日上限, LevelType_Weapon |
| SystemOpen.xlsx | 已有 | 解锁条件配置（PLv.27） |
| Item.xlsx | 已有/新增 | WEAPON_EXP_LOW/MID/HIGH 道具定义 |
| PlayerDailyCounter | 已有 | weapon_exp_daily_count 独立计数 |

### 接口变更
- `EnterWeaponExpDungeon(difficulty_id: int) -> EnterResult`（内部路由到通用副本进入接口 + LevelType 参数）
- `SettleWeaponExpDungeon(session_id, kills, loot_list) -> SettleResult { items[], total_weapon_exp_preview }`
- `GetWeaponExpDungeonDailyStatus() -> { remaining, total, next_reset_time }`

## 风险评估
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| PLv.27 解锁过晚导致空窗期过长 | 中 | 中 — 中期玩家无武器经验来源 | 策划确认解锁节点合理性（与专武获取时间对齐）；若过晚，通过活动补充空窗期产出 |
| 武器经验产出 vs 角色经验产出混淆 | 低 | 中 — 配置错误导致掉错道具 | CI 强校验：LevelType_Weapon 只关联 WeaponEXP DropGroup；QA 冒烟用例覆盖 |
| 外挂速刷（同角色经验副本风险） | 高 | 高 — 武器快速满级破坏数值 | 服务端校验：击杀速率、最短时长、次数上限三重防控 |
| 与角色经验副本每日次数共用导致玩家选择困难 | 低 | 低 — 体验纠结 | 三者次数独立（当前方案），不冲突 |

## 工作假设
| 假设 | 状态 | 澄清结论 |
|------|------|----------|
| 解锁节点为玩家等级 27 | 待确认 | 需策划确认最终数值，是否改为主线某关解锁 |
| 武器经验道具独立于角色经验道具 | 当前规则 | 两类道具 item_id 不同，不互通 |
| 复用角色经验副本框架，无独立 GameMode | 当前规则 | 共享 NormalMode + 掉落系统 |
| 单局 60-90s 与角色经验副本一致 | 待确认 | 需实测调优怪物密度 |
| 专武经验副本与角色经验副本每日次数独立 | 当前规则 | 已确认，三者互不影响 |
