## 背景与目标
### 背景
公会系统构建游戏的核心社交循环，通过公会任务（Task_TTT_GuildTask）、公会商店（公会币消费）、公会战（Guild War）等模块聚合玩家协作与竞争，驱动社交粘性与长线留存。解锁门槛为玩家等级 22 级。

### 目标
- 公会创建与成员管理（加入/退出/踢出/职位）
- 公会任务：公会维度协作任务，产出公会经验与公会币
- 公会商店：使用公会币兑换专属商品
- 公会战：公会级 PvP/PvE 竞争
- 捐献机制：成员消耗资源捐献为公会贡献

### 非目标
- 不设计公会科技树/建筑升级（首版不做）
- 不处理跨服公会战（仅本服）
- 不设计公会合并/转让的复杂流程

## 核心规则
### R-01: 公会创建与解散
- **触发条件**：创建：玩家等级 ≥ 22 且未加入公会，点击"创建公会"；解散：会长操作且公会无其他成员
- **前置限制**：创建：消耗钻石或金币；公会名合法（不重复、不违禁）；解散：仅会长；公会成员数 = 1（仅会长）
- **操作流程**：
  - 创建：校验资源 → 校验名称 → 扣除费用 → 创建公会实体 → 会长为创建者
  - 解散：二次确认 → 删除公会实体
- **状态变化**：公会创建 → 新公会 ID 生成，会长归属；公会解散 → 实体删除
- **资源变化**：创建：-创建费用；解散：无退费
- **UI 反馈**：创建：公会面板展开 → 公会 ID/名称展示；解散：确认弹窗 → 返回无公会状态
- **异常边界**：公会名含敏感词 → 过滤提示；创建并发 → 服务端控制
- **落地点**：公会服务端逻辑，无专属 Excel 表

### R-02: 加入/退出公会
- **触发条件**：加入：玩家等级 ≥ 22，搜索/推荐公会，点击"申请"或"一键加入"；退出：公会成员点击"退出公会"
- **前置限制**：加入：未加入其他公会；目标公会未满员；审批模式（自由加入 / 需审批）；退出：非会长（会长需先转让）
- **操作流程**：
  - 加入：发送申请 → 审批通过（或直接加入）→ 成员列表新增
  - 退出：确认 → 从成员列表移除 → 清除公会相关数据（贡献值可保留）
- **状态变化**：成员加入/退出 → 公会人数变更
- **资源变化**：退出：部分公会币/贡献可能清零或保留
- **UI 反馈**：公会大厅成员列表刷新 → 公会公告/聊天频道可见
- **异常边界**：退出冷却期（退出后 N 小时内不可加新公会）
- **落地点**：Unlock.xlsx (解锁条件 22 级)

### R-03: 公会捐献
- **触发条件**：公会成员在捐献界面选择捐献档位并确认
- **前置限制**：玩家当天捐献次数未达上限；持有足够的捐献资源
- **操作流程**：扣除资源 → 公会经验增加 → 个人贡献值增加 → 公会币发放到成员
- **状态变化**：公会当日捐献总次数 +1；个人贡献值增加；公会等级经验累积
- **资源变化**：-捐献资源（金币/钻石）；+公会币（个人）；+公会经验（公会级）
- **UI 反馈**：捐献成功动画 → 贡献值/公会币更新 → 公会经验条推进
- **异常边界**：多次捐献上限由配置控制；捐献档位独立限制
- **落地点**：公会捐献配置表（未在已知 Excel 中明确，可能需要新建）

### R-04: 公会任务
- **触发条件**：公会在任务周期（如周常）内激活
- **前置限制**：玩家已加入公会
- **操作流程**：任务类型为 Task_TTT_GuildTask 的任务激活 → 公会成员各自/共同推进进度 → 达到公会总进度目标后全员可领奖励
- **状态变化**：公会任务进度累积；达成后标记完成
- **资源变化**：达成后全员获得公会币/公会经验
- **UI 反馈**：公会任务面板 → 总进度条 → 个人贡献占比 → 完成奖励领取
- **异常边界**：成员在任务中途退出 → 贡献进度是否保留按配置
- **落地点**：Task.xlsx (Task_TTT_GuildTask 类型)

### R-05: 公会商店
- **触发条件**：公会成员打开公会商店界面
- **前置限制**：公会已解锁商店功能（可能需公会等级）
- **操作流程**：同商店系统 R-01，但货币为公会币 → 查询 ShopGoods.xlsx 中公会商店 Tab 的商品
- **状态变化**：购买计数更新；公会币减少
- **资源变化**：-公会币；+商品货物
- **UI 反馈**：商店货架展示（公会币符号标注）→ 购买后刷新
- **异常边界**：同商店系统限购/售罄逻辑
- **落地点**：Shop.xlsx (公会商店 Tab), ShopGoods.xlsx

### R-06: 公会战
- **触发条件**：公会战活动开启（按赛季/周期）
- **前置限制**：公会在报名期内；公会满足报名条件（如人数、活跃度）
- **操作流程**：
  - 报名期：会长/官员确认报名 →
  - 匹配期：系统匹配对手公会 →
  - 战斗期：成员对敌方公会据点/成员发起战斗 →
  - 结算期：汇总双方积分 → 判定胜负 → 发放排名奖励
- **状态变化**：公会战阶段流转（报名 → 匹配 → 战斗 → 结算）
- **资源变化**：成员获得参战奖励 + 胜负奖励（公会币/稀有道具）
- **UI 反馈**：公会战入口 → 敌方信息 → 战斗界面 → 实时积分榜 → 结算面板
- **异常边界**：成员离线 → 其防守阵容仍生效但自动战斗；掉线重连恢复战场
- **落地点**：公会战配置表（未在已知 Excel 中明确，可能需要新建）

### R-07: 成员管理与职位
- **触发条件**：会长/官员操作成员列表
- **前置限制**：操作者职位权限足够（会长 > 官员 > 普通成员）
- **操作流程**：
  - 任命/撤销官员：会长操作
  - 踢出成员：会长/官员操作 → 目标被移出公会
  - 转让会长：会长操作 → 选择目标 → 确认移交
- **状态变化**：成员职位变更；被踢/退出成员清除公会归属
- **资源变化**：无
- **UI 反馈**：成员列表刷新 → 职位标识更新 → 被操作者收到通知
- **异常边界**：会长转让需二次确认 + 冷却期；不能转让给自己
- **落地点**：无专属 Excel，纯服务端逻辑

## 技术方案
### 架构影响
- 公会为独立服务/模块，管理公会实体 CRUD、成员管理、捐献、公会战
- 公会币为独立货币类型，接入货币系统与商店系统
- 公会任务对接任务系统的事件驱动架构
- 公会战需匹配系统与战斗结算模块支持

### 数据变更
| 表 | 变更类型 | 说明 |
|----|----------|------|
| Shop.xlsx (公会 Tab) | 配置 | 公会商店页签 |
| ShopGoods.xlsx | 配置 | 公会商店商品（公会币定价） |
| Task.xlsx | 配置 | Task_TTT_GuildTask 任务 |
| Unlock.xlsx | 配置 | 公会解锁条件（22 级） |

### 接口变更
- `GuildCreate`：入参 GuildName + IconId；出参 GuildId + GuildInfo
- `GuildJoin`：入参 GuildId；出参 Success/FailReason
- `GuildLeave`：无入参（退出当前公会）；出参 Success
- `GuildDonate`：入参 DonateIndex；出参 PersonalContribution + GuildExp + GuildCoin
- `GuildTaskGetList`：入参 TaskType=GuildTask；出参 TaskList[] + GuildProgress
- `GuildShopGetList`：复用 ShopGetList(TabId=GuildTab)；出参 GoodsList[]
- `GuildWar*`：多协议（报名、匹配信息、战斗、结算）

## 风险评估
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 公会战匹配不均导致弱势公会体验差 | 中 | 中 | 按公会战力/活跃度分档匹配 |
| 会长流失 → 公会僵死 | 高 | 中 | 自动转让机制（会长离线 > N 天后转让给最高贡献官员） |
| 公会捐献刷取公会币 | 中 | 中 | 每日捐献上限控制 |
| 公会解散导致大量成员数据处理 | 低 | 中 | 成员数据保留贡献值、公会币，仅清除公会归属 |
| 公会名敏感词绕过 | 中 | 低 | 敏感词库持续更新 |
