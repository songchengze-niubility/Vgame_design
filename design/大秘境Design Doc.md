## 背景与目标
### 背景
大秘境是 Vgame 中面向后期硬核玩家的赛季制爬层玩法。以 28 天为一个赛季，玩家从 25 层难度中向上攀登（怪物等级 40-220）。每赛季引入环境词缀修改战斗规则，迫使玩家调整阵容和策略。该玩法是游戏的终局验证内容：考验玩家装备广度（Box Breadth）、环境适应能力和极限输出深度。

### 目标
- 提供赛季制长期目标（28 天 / 赛季）
- 通过 25 层递增难度验证装备深度和策略广度
- 通过词缀系统创造赛季差异化，防止单一阵容通吃
- 通过周奖励 + 赛季奖励双线激励持续参与

### 非目标
- 非永久性进度（赛季结束重置层数）
- 非纯竞速（通关为主，时间权重辅助）
- 非多人匹配（单人挑战）

## 核心规则
### R-01: 赛季结构（28 天）
- **触发条件**：赛季开始 / 结束
- **前置限制**：玩家等级 >= 37；赛季数据从配置表读取
- **操作流程**：赛季开始：所有玩家层数进度重置为 1 层，新词缀生效；赛季中：玩家推进层数、获取周奖励；赛季结束：发放赛季奖励、清空进度、等待下赛季开始
- **状态变化**：赛季状态 `OffSeason → Active → Settling → OffSeason`
- **资源变化**：赛季结束时层数重置，玩家需重新攀登
- **UI 反馈**：赛季倒计时面板；新赛季开场动画（展示新词缀）；赛季结算邮件
- **异常边界**：赛季交替时若玩家在线 —— 强制弹出结算界面，离线则静默结算；过渡期（结算中）禁止进入战斗
- **落地点**：赛季管理模块

### R-02: 25 层难度递增
- **触发条件**：玩家通关当前层或进入新层
- **前置限制**：25 层固定配置，怪物等级范围 40-220
- **操作流程**：每层怪物配置独立（怪物编队、属性倍率）；通关 N 层 → 解锁 N+1 层；必须顺序攀登，不可跳层
- **状态变化**：楼层解锁 `Layer_N unlocked`；最高通关层记录更新
- **资源变化**：无
- **UI 反馈**：层级选择面板；每层显示怪物等级范围和推荐战力；通关进度条
- **异常边界**：同一层可重复挑战（用于刷分或尝试不同策略），但周/赛季奖励只看最高层
- **落地点**：层级控制器

### R-03: 环境词缀（Affixes）
- **触发条件**：赛季开始时从词缀池选取；进入战斗时词缀生效
- **前置限制**：词缀配置表定义所有可用词缀及其效果；每赛季选取 1-3 个激活词缀
- **操作流程**：词缀在战斗全局生效，修改战斗规则 —— 例如 "敌人每 10 秒获得护盾"、"玩家治疗降低 30%"、"场地随机区域周期性造成伤害" 等；玩家需针对词缀调整阵容和策略
- **状态变化**：词缀在赛季中始终激活，不可关闭
- **资源变化**：无
- **UI 反馈**：赛季主界面展示当前激活词缀的图标和描述；战斗中词缀触发时有 UI 提示（如护盾条、区域预警圈）
- **异常边界**：词缀效果叠加时如有冲突 —— 按配置表优先级处理；词缀可能产生意料之外的组合效应，需设计阶段充分遍历测试
- **落地点**：词缀配置表 → 战斗规则引擎

### R-04: 每周最高层奖励
- **触发条件**：每周结算时刻
- **前置限制**：WeekMaxLayerReward.xlsx 定义 "最高通关层数 → 奖励内容" 映射；WeekScoreReward.xlsx 定义 "积分 → 奖励内容" 映射
- **操作流程**：每周结算时取玩家当周最高通关层数 → 匹配 WeekMaxLayerReward 发放奖励；同时取当周累计积分 → 匹配 WeekScoreReward 发放奖励（两线独立发放）
- **状态变化**：结算后当周数据进入历史快照
- **资源变化**：发放货币、赛季代币、升级材料等
- **UI 反馈**：周结算邮件（分别展示最高层奖励和积分奖励）；主界面倒计时至下次周结算
- **异常边界**：每周结算时刻玩家在线（发放并刷新 UI）、离线（静默邮件）；同一周同一玩家不可重复领取
- **落地点**：周结算定时任务

### R-05: 赛季结算奖励
- **触发条件**：28 天赛季结束
- **前置限制**：SeasonScoreReward.xlsx（积分档位奖励）、SeasonRankReward.xlsx（排名档位奖励）
- **操作流程**：赛季结束时生成两个维度的奖励 —— 赛季累计积分达标的档位奖励（全员达标发放）+ 赛季排名奖励（Top%）；排名基于赛季最高通关层 + 同层排名时间
- **状态变化**：赛季数据归档；玩家进度重置
- **资源变化**：发放赛季限定外观、称号、高价值装备等
- **UI 反馈**：赛季结算弹窗（展示成绩 + 奖励）；排行榜快照公示
- **异常边界**：赛季结算为批量操作，需分批复核；结算期间禁用大秘境入口；若结算异常则进入人工 GM 处理流程
- **落地点**：赛季结算模块

### R-06: 解锁条件
- **触发条件**：玩家查看大秘境入口
- **前置限制**：玩家等级 >= 37；当前赛季处于 Active 状态
- **操作流程**：系统校验等级 37 达到 → 首次解锁时播放引导动画 → 入口常驻点亮（赛季 OffSeason 期间入口可见但标注 "休赛期"）
- **状态变化**：入口 `Locked → Unlocked`
- **资源变化**：无
- **UI 反馈**：解锁引导弹窗；休赛期入口显示 "下一赛季 X 天后开始"
- **异常边界**：玩家在赛季 OffSeason 期间到达 37 级 —— 入口解锁但不允许进入战斗，直到新赛季开始
- **落地点**：入口 UI 控制

### R-07: 广度（Box Breadth）验证
- **触发条件**：高层难度 + 词缀组合对单一阵容构成克制
- **前置限制**：词缀设计刻意制造 "某一阵容被削弱" 的局面，需要玩家切换组建替补阵容
- **操作流程**：玩家无法用单一最优阵容打通全部 25 层 —— 需培养多个角色 / 多套装备以适应不同词缀；高级玩家通过 "装备广度" 取胜
- **状态变化**：不适用（设计意图，非运行时逻辑）
- **资源变化**：无
- **UI 反馈**：战前准备界面提示 "推荐调整阵容以应对当前词缀"
- **异常边界**：若某赛季词缀组合导致所有已知阵容均严重削弱 —— 需热更词缀效果或不启用该组合
- **落地点**：词缀设计原则（设计规则，非代码）

## 技术方案
### 架构影响
- 新增 `MysteryMode` 运行模式
- 新增赛季管理模块（赛季生命周期、进度重置）
- 新增词缀系统模块（规则引擎 → 战斗中动态规则注入）
- 双线奖励接入：周（WeekMaxLayerReward + WeekScoreReward）、赛季（SeasonScoreReward + SeasonRankReward）

### 数据变更
| 表 | 变更类型 | 说明 |
|----|----------|------|
| WeekMaxLayerReward.xlsx | 新增 | 周最高层奖励 |
| WeekScoreReward.xlsx | 新增 | 周积分奖励 |
| SeasonScoreReward.xlsx | 新增 | 赛季积分奖励 |
| SeasonRankReward.xlsx | 新增 | 赛季排名奖励 |
| 词缀配置表 | 新增 | 词缀效果、组合规则、赛季选取配置 |
| player_mythic_season | 新增 | 玩家赛季进度（当前层、最高层、积分、排名快照） |

### 接口变更
- 赛季状态查询（当前赛季、剩余时间、词缀）
- 层级列表 / 进入层级 / 层级结算
- 周奖励查询 / 领取
- 赛季奖励查询 / 领取
- 排名查询

## 风险评估
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 词缀组合设计不当导致某些赛季无法通关（dead end） | 中 | 高 | 上线前全词缀组合自动化测试；设 GM 热更开关可禁用单条词缀 |
| 赛季重置导致玩家断层流失（进度归零挫败感） | 中 | 中 | 赛季奖励力度 + 新词缀新鲜感补偿；展示新赛季预告内容 |
| 装备广度要求过高导致免费玩家无法参与高层 | 中 | 中 | 低层（1-10）词缀效果减弱；高层奖励偏荣誉性 |
| 赛季结算批量操作超时或部分失败 | 低 | 高 | 分批复核 + 失败补偿队列 + GM 手动补发工具 |
