## 背景与目标
### 背景
宝石与铭文是映射装备孔位系统的二级填充物，构成终局装备追深的三维随机结构（装备属性 × 孔位数量 × 镶嵌物品质）。宝石提供可合成的渐进式成长路径（1-5级），铭文提供不可继续合成但具有套装效果的深度选择（1-3级）。二者共同嵌入映射装备的孔位中，与190条词缀、洗练、超限形成完整的终局装备生态。

宝石定位为"确定性成长"——玩家通过合成稳定提升属性，降低纯随机体系带来的挫败感。铭文定位为"效果深度"——提供套装组合的策略选择，不可继续合成保证其稀缺性价值。

### 目标
- 宝石提供渐进可合成的确定性成长路径，平衡映射装备的高度随机性
- 铭文提供套装效果的策略深度，不可继续合成赋予长期稀缺价值
- 镶嵌/替换机制与映射装备孔位系统无缝对接
- 宝石合成消耗设计形成有效的资源回收闭环

### 非目标
- 铭文不支持继续合成升级（1-3级仅为产出分级，不可通过合成提升等级）
- 宝石不可降级/分解退还材料
- 已镶嵌的宝石/铭文不可跨装备转移（替换即销毁）
- 宝石/铭文不在旅券系统中使用（仅限映射装备孔位）

## 核心规则
### R-01: 宝石等级体系与属性成长
- **触发条件**：宝石生成（掉落/合成产出/奖励）时确定等级与属性
- **前置限制**：宝石类型（攻击/防御/辅助等大类）已确定；属性池按类型配置；等级范围1-5
- **操作流程**：1) 确定宝石类型与等级；2) 从该类型属性池中按权重抽取主属性（1条）；3) 属性值 = 基础值 × 等级系数（1级=1.0 / 2级=1.4 / 3级=1.9 / 4级=2.5 / 5级=3.2）；4) 实例化宝石→入库
- **状态变化**：宝石 `Born → Active` 进入背包
- **资源变化**：新宝石实例入库；消耗产出机会（掉落/合成）
- **UI 反馈**：宝石图标按类型+等级着色；等级罗马数字标注（I-V）；属性值数值显示；点击展示与当前镶嵌对比
- **异常边界**：等级超出[1,5]范围时钳位；属性为0或负值时视为异常数据，标记并重新roll；同一ID宝石不可重复生成（唯一实例ID保证）
- **落地点**：`GemGenerator`，`SpecialEquipment.xlsx` Sheet `Gem` 配置宝石类型与属性池

### R-02: 宝石合成系统
- **触发条件**：玩家在宝石合成界面放入足够数量的同类型同等级宝石
- **前置限制**：放入宝石数量 = 合成需求数（3合1 或 5合1，视配表）；所有放入宝石为同类型且同等级；目标等级 ≤ 5；背包空间 ≥ 1
- **操作流程**：1) 打开宝石合成界面；2) 选择合成配方（"3合1" 或 "5合1"，后者可能有额外属性提升/品质保底）；3) 放入宝石×N（自动筛选同类型同等级）；4) 预览产出（等级+1，类型不变）；5) 确认合成；6) 消耗宝石→产出高1级宝石
- **状态变化**：N件宝石 `Active → Consumed`；1件新宝石（等级+1）`Born → Active`
- **资源变化**：消耗N件同类型同等级宝石；消耗金币（按目标等级递增）；产出1件高1级宝石；净消耗N-1件宝石（资源回收）
- **UI 反馈**：合成材料区宝石堆叠展示；产出一侧显示目标等级宝石轮廓（点击可预览属性范围）；合成成功动效（宝石聚合→升级光效）；快捷"一键合成至最高"（自动循环合成直至材料不足）
- **异常边界**：放入不同类型/等级宝石时自动过滤，不可混入；最高等级（5级）时合成按钮替换为"MAX"；批量高于单次合成上限时自动分批处理；合成过程中断网回退至操作前状态，不损失宝石
- **落地点**：`GemSynthesisManager`，合成需求数/成本在 `SpecialEquipment.xlsx` Sheet `GemSynthesis`

### R-03: 铭文等级体系与不可合成规则
- **触发条件**：铭文通过掉落/奖励产出（仅产出阶段决定等级）
- **前置限制**：铭文类型已确定（套装归属）；等级范围1-3；产出稀有度按配置权重分配
- **操作流程**：1) 掉落判定→产出铭文；2) 确定类型与等级（1级基础/2级进阶/3级稀有）；3) 铭文套装效果固定（随等级提升效果强度）；4) 实例化→入库
- **状态变化**：铭文 `Born → Active` 进入背包；1-3级固定不可升级
- **资源变化**：新铭文入库；无合成路径（不可通过合成升到更高等级）
- **UI 反馈**：铭文等级标记（I-III）；套装名+"·"+等级；集齐套装提示（"集齐X件套可激活效果"）；不可合成的UI暗示（无合成入口，升级按钮灰化/隐藏）
- **异常边界**：伪铭文（等级超出1-3范围）拒绝入库；重复同套同等级铭文可持有但镶嵌时只计算最高等级效果（不可叠加）；铭文ID不冲突校验
- **落地点**：`InscriptionManager`，`SpecialEquipment.xlsx` Sheet `Inscription` 铭文配置

### R-04: 铭文套装效果
- **触发条件**：角色身上同一映射装备中已镶嵌的铭文进行套装判定；或每次镶嵌/替换铭文后
- **前置限制**：至少镶嵌了该套装2件铭文（可跨装备）；同套装的铭文等级可不同（最高等级计为套装效果等级）
- **操作流程**：1) 统计该角色所有映射装备孔位中镶嵌的铭文；2) 按套装ID分组计数；3) 件数达到2/3时激活对应套装效果（2件套/3件套）；4) 效果强度按套装中最高等级铭文计算（1级=100%基础 / 2级=130% / 3级=160%）；5) 同套装不同等级铭文时，计件数以高等级为准
- **状态变化**：套装效果 `Inactive → Active(Tier2/Tier3)`；角色属性面板实时刷新
- **资源变化**：无
- **UI 反馈**：铭文套装页展示当前激活状态；缺件高亮（暗色/灰色图标）；效果描述按当前激活等级动态显示数值；切换铭文预览套装效果变化
- **异常边界**：多个2件套可同时激活（不同套装）；3件套数受限于总孔位数（随机孔位上限）；同一装备内同套装铭文不可超过孔位总数；卸除非铭文相关装备时重新判定套装
- **落地点**：`InscriptionSetManager`，`SpecialEquipment.xlsx` Sheet `InscriptionSet` 套装效果配置

### R-05: 宝石/铭文镶嵌与替换
- **触发条件**：玩家对映射装备的孔位执行镶嵌/替换操作
- **前置限制**：目标装备孔位数量 ≥ 1；有空闲孔位（镶嵌）或已有镶嵌物（替换）；背包中有可用宝石/铭文；宝石/铭文未处于其他装备的镶嵌状态
- **操作流程**：1) 打开装备详情→孔位面板；2) 选择空闲孔位→从背包选宝石或铭文→确认镶嵌（首次）；3) 或选择已镶嵌孔位→点击替换→从背包选新宝石/铭文→确认替换；4) 镶嵌完成→属性更新
- **状态变化**：镶嵌：孔位 `Empty → Filled(ItemId)`；替换：旧镶嵌物 `Active → Destroyed`，新镶嵌物 `Active → Filled`；装备属性更新；若铭文替换可能影响套装效果→重新判定
- **资源变化**：镶嵌：消耗宝石/铭文×1；替换：消耗新宝石/铭文×1，旧镶嵌物销毁不可退还
- **UI 反馈**：镶嵌成功动效（宝石/铭文嵌入孔位动画）；替换二次确认弹窗红字提示"旧镶嵌物将永久销毁"；属性变化预览对比
- **异常边界**：孔位数为0的装备不可操作镶嵌（入口灰化）；替换确认窗口中可查看旧物详情；背包中无可用宝石/铭文时替换入口不可点击；替换操作断网回退，不销毁不消耗
- **落地点**：`SocketEmbedManager`，孔位操作协议复用 R-04（映射装备孔位系统）

### R-06: 宝石/铭文掉落与经济闭环
- **触发条件**：战斗/活动/关卡结算判定掉落
- **前置限制**：掉落池已配置宝石/铭文产出权重与等级分布；玩家满足关卡/活动解禁条件；背包空间 ≥ 1
- **操作流程**：1) 结算→掉落判定（先判类型：宝石/铭文/装备）；2) 命中宝石/铭文池→按配置权重确定类型；3) 按等级分布表 roll 出等级；4) 实例化→入库
- **状态变化**：`DropResult(Gem/Inscription)`→背包入库
- **资源变化**：新宝石/铭文×1入库；掉落计数+1（用于伪随机保底统计）
- **UI 反馈**：掉落弹窗展示宝石/铭文品质光效；点击查看详情；首次获得该铭文套装时提示套装效果预览
- **异常边界**：背包满→发送至邮箱；断线未领取→补偿邮件；掉落池为空→不产出也不报错；同活动期间防止同品质掉落池概率漂移
- **落地点**：`DropManager → GemDropProcessor / InscriptionDropProcessor`，`SpecialEquipment.xlsx` 掉落配置Sheet

### R-07: 宝石合成经济回收率控制
- **触发条件**：合成/掉落/活动的全链路定量监控
- **前置限制**：监控需持续 ≥ 2周数据确保统计有效性；需区分有机产出（体力→掉落）和活动投放
- **操作流程**：1) 统计各级宝石的人均持有量/流通量/消耗量；2) 3合1合成：净回收率 = 消耗3件-产出1件 = 净回收2件（66.7%）；5合1合成：净回收率 = 5-1 = 80%；3) 宝石等级增长率 = 高级宝石生成量 / 低级宝石消耗量；4) 设定预警：若玩家全服5级宝石饱和率超过30%，需增加消耗出口（如活动兑换/限时突破）
- **状态变化**：监控→分析→运维配表调整
- **资源变化**：无玩家感知变化
- **UI 反馈**：运维后台面板展示宝石经济曲线
- **异常边界**：区分5合1与3合1路径的独立回收率统计；活动赠送宝石不纳入自然回收率计算；铭文因不可合成，仅统计持有量分布
- **落地点**：运维监控面板，配表参数 `SpecialEquipment.xlsx` Sheet `GemEconomy`

## 技术方案
### 架构影响
- 新增 `GemManager`、`InscriptionManager`、`GemSynthesisManager`、`InscriptionSetManager` 四个模块
- 孔位操作模块（`SocketEmbedManager`）需与映射装备的 `SocketManager` 共享底层数据
- 铭文套装效果与旅券套装效果共享套装判定框架（`SetEffectManager` 抽象层）
- 宝石合成需支持批量合成与一键最高级合成（前端循环/后端批量事务）

### 数据变更
| 表 | 变更类型 | 说明 |
|----|----------|------|
| SpecialEquipment.xlsx | 新增Sheet | Gem / GemSynthesis / Inscription / InscriptionSet / GemEconomy |
| Gem | 新增表 | GemId, Type, Level(1-5), AttrType, AttrValue, SocketedEquipId |
| Inscription | 新增表 | InscriptionId, SetId, Level(1-3), EffectValue, SocketedEquipId |
| MappingEquip | 扩展字段 | Sockets[] → 存储 GemId/InscriptionId 引用 |
| InscriptionSet | 新增表 | SetId, Tier(2/3), EffectDesc, LevelMultiplier(1.0/1.3/1.6) |
| GemSynthesisLog | 新增表 | 合成记录：消耗宝石列表、产出宝石ID、合成类型（3合1/5合1）、时间戳 |

### 接口变更
- `POST /gem/synthesize` — 合成宝石（含3合1/5合1选择）
- `POST /gem/synthesize/all` — 一键合成至最高级
- `POST /socket/embed` — 镶嵌宝石/铭文（复用映射装备镶嵌接口）
- `POST /socket/replace` — 替换镶嵌物
- `GET /gem/list` — 获取背包宝石列表
- `GET /inscription/list` — 获取背包铭文列表
- `GET /inscription/sets?charId=` — 获取角色铭文套装激活状态
- `Push: GemSynthesisSuccess` — 宝石合成成功通知

## 风险评估
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 宝石合成经济崩溃（3合1→高级宝石泛滥→终局深度丧失） | 中 | 极高 | 回收率66.7%为设计基线；5合1可选路径设计为低效消耗；监控全服5级宝石占比不超过30% |
| 铭文"不可合成"导致高等级铭文获取过难→玩家挫败 | 中 | 高 | 活动投放与掉落池微调保障人均持有曲线；高等级铭文掉落概率设有保底计数器 |
| 铭文套装效果平衡（某套过强导致配装单一） | 高 | 高 | 套装效果定期参数Audit；上线后预留调参窗口 |
| 替换即销毁设计导致玩家"决策恐惧"不敢镶嵌 | 低 | 中 | 引导任务赠送低级试错宝石；替换前必须二次确认，提供2分钟反悔窗口（如有技术条件） |
| 宝石/铭文系统与旅券在经济上产生割裂（三个体系独立消耗出口） | 中 | 中 | 统一货币/体力驱动入口；各系统的付费点需全局统筹避免内部竞争 |
