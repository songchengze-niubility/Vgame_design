Vgame 驾驭工程 — 新人上手指南

本指南面向第一次接触 Vgame_design 仓库的策划。读完后你应该能独立完成：查配置表结构、写设计文档、跑 Proposal→Task 流程、让 AI 帮你改表。

第一步：克隆仓库

# 公司内网
git clone git@code.dobest.com:spark/Vgame_design.git

# 或者个人 GitHub
git clone https://github.com/songchengze-niubility/Vgame_design.git

第二步：配置本地路径（必做一次）

仓库不保存任何人的盘符。首次使用时：

copy local.env.example.bat local.env.bat

编辑 local.env.bat，填入你的本机路径：

set "VGAME_ROOT=<你的 Vgame SVN 工程路径>"
set "VGAME_DESIGN_ROOT=<你的 Vgame_design Git 仓库路径>"
set "VGAME_SOURCE_DOCS_ROOT=<你的 Vgame 原始策划文档路径>"
set "VGAME_OUTPUT_ROOT=<你的个人输出路径>"

local.env.bat 已在 .gitignore 中，不会上传到仓库。

使用 SVN 中的中文启动层时，直接双击 驾驭工程/配置本机路径.bat，按提示粘贴自己的目录即可。

核心概念

概念一句话解释
Design Doc"这个系统是什么、怎么运作" — 设计文档
Proposal"我要改什么、影响什么、怎么验证" — 落地方案
Task"拆成哪些步骤、每步怎么做" — 执行清单
SkillAI 的"策划知识库"，告诉 AI 配置表怎么查、规则是什么
知识图谱290 张配置表的结构数据（字段名、类型、公式列、外键）

第四步：日常怎么用

场景 A 查配置表结构

想查某张表有什么字段、哪些列有公式不能写：

# 方法 1：直接打开 Excel 看前 4 行
# Row 1 = 字段名, Row 2 = 类型, Row 3 = 分组, Row 4 = 中文说明

# 方法 2：用 Python 查（需要先安装 openpyxl）
python -c "
import json
schema = json.load(open('knowledge-graph/config-schema.json', 'r', encoding='utf-8'))
for node in schema['nodes']:
    if node['name'] == '你想查的表名':
        print('字段:', list(node['metadata']['fields'].keys()))
        print('公式列:', node['metadata']['formula_columns'])
        print('安全列:', node['metadata']['safe_write_columns'])
"

场景 D 让 AI 帮你做事

如果你用 OpenCode，AI 会自动加载 skills/ 下的 Skill。你只需要：

  1. 告诉 AI 你想做什么（比如"帮我查 Drop_mainlevel 的字段"）
  2. AI 自动读 AGENTS.md → 路由到对应 Skill → 读知识图谱 → 输出结果

不需要手动告诉 AI 去读哪个文件，AGENTS.md 已经定义好了路由。

第五步：AI 怎么配合

你想要AI 自动做什么
查一张表的字段读 config-schema.json 返回字段名/类型/公式列
改一个奖励配置查 config-schema.json 确认字段安全 → 读 Excel 当前值 → 改表
写一份设计文档读 design/TEMPLATE.md + 相关 Skill → 按模板输出
跑模拟验证调用 simulate_progression.py → 对比改前改后
调战斗数值读 vgame-battle-tuning-helper Skill → 计算 DPS/TTK/难度

AI 的知识来源是仓库里的 Skill 和知识图谱，不是它自己猜的。所以你问的问题越具体，AI 回答越准。

第六步：提交前检查

powershell -ExecutionPolicy Bypass -File scripts/check.ps1

这个脚本会检查：

  • 设计文档格式是否正确
  • 配置表结构是否有问题
  • 跨表引用是否完整

常见问题

Q：我改了配置表，需要更新知识图谱吗？

只有新增/删除表或字段时需要。只改数值不需要。

Q：知识图谱过期了怎么办？

双击 knowledge-graph/更新知识图谱.bat，它会重新读取所有配置表生成最新结构。

Q：AI 不知道我改了什么怎么办？

确保你改的配置表在 SVN 里，AI 会通过知识图谱读取最新结构。如果 AI 用的是旧数据，跑一次 更新知识图谱.bat。

Q：我想新增一个系统，从哪开始？

先写 design/新系统名Design Doc.md（用 TEMPLATE.md），再走 Proposal→Task 流程。

Q：Proposal 和 Task 有什么区别？

Proposal = "改什么 + 影响什么 + 风险"（给人看的）。Task = "拆成哪些步骤"（给执行用的）。

Q：我只想改一个数值，也要走完整流程吗？

不需要。小改动直接改表 + commit message 记录就行。只有跨系统、影响大的改动才需要 Proposal→Task。

Q：怎么知道哪些列能改、哪些不能改？

查 config-schema.json 里的 formula_columns（不能改）和 safe_write_columns（能改）。或者直接问 AI。

Q：我想给其他项目也用这套框架怎么办？

通用项目skill/ 目录下有通用模板，复制后填入新项目的具体信息即可。
