# Vgame 驾驭工程 — 新人上手指南

> 本指南面向第一次接触 `Vgame_design` 仓库的策划。读完后你应该能独立完成：查配置表结构、写设计文档、跑 Proposal→Task 流程、让 AI 帮你改表。

## 第二步：配置本地路径（可选但推荐）

仓库不保存任何人的盘符。首次使用时：

```bash
# 复制模板
copy local.env.example.bat local.env.bat
```

编辑 `local.env.bat`，填入你的本机路径：

```bat
set "VGAME_ROOT=D:\Vgame"
set "VGAME_CONFIG_DATAS=D:\Vgame\Config\GameConfig\Datas"
```

`local.env.bat` 已在 `.gitignore` 中，不会上传到仓库。

## 第三步：认识仓库结构（5 分钟）

```
Vgame_design/
├── AGENTS.md              ← AI 入口（AI 第一个读的文件）
├── design/                ← 32 篇系统设计文档
│   ├── TEMPLATE.md        ← 新设计文档的模板
│   └── 主线关卡系统Design Doc.md  ← 例子
├── proposals/             ← 落地方案（改表前写）
│   └── TEMPLATE.md
├── tasks/                 ← 执行任务（拆步骤）
│   └── TEMPLATE.md
├── skills/                ← 18 个策划 AI Skill
│   └── vgame-core-understanding/SKILL.md  ← 入口
├── knowledge-graph/       ← 知识图谱（配置表结构+代码结构）
│   ├── config-schema.json ← 290 张配置表的字段/类型/公式/外键
│   ├── knowledge-graph.json ← 代码图谱
│   └── 更新知识图谱.bat   ← 改表后跑这个
├── harness/               ← 约束规则（门禁、编码规范）
├── scripts/               ← 检查脚本
└── 通用项目skill/         ← 可复用到其他项目的 Skill 模板
```

### 核心概念

| 概念 | 一句话解释 |
|---|---|
| **Design Doc** | "这个系统是什么、怎么运作" — 设计文档 |
| **Proposal** | "我要改什么、影响什么、怎么验证" — 落地方案 |
| **Task** | "拆成哪些步骤、每步怎么做" — 执行清单 |
| **Skill** | AI 的"策划知识库"，告诉 AI 配置表怎么查、规则是什么 |
| **知识图谱** | 290 张配置表的结构数据（字段名、类型、公式列、外键） |

## 第四步：日常怎么用

### 场景 A：查配置表结构

想查某张表有什么字段、哪些列有公式不能写：

```bash
# 方法 1：用 Python 直接查
python -c "
import json
schema = json.load(open('knowledge-graph/config-schema.json', 'r', encoding='utf-8'))
for node in schema['nodes']:
    if node['name'] == '你想查的表名':
        print('字段:', list(node['metadata']['fields'].keys()))
        print('公式列:', node['metadata']['formula_columns'])
        print('安全列:', node['metadata']['safe_write_columns'])
"

# 方法 2：直接打开 Excel 看前 4 行
# Row 1 = 字段名, Row 2 = 类型, Row 3 = 分组, Row 4 = 中文说明
```

### 场景 B：写一份新系统设计

```bash
# 1. 复制模板
copy design/TEMPLATE.md design/新系统名Design Doc.md

# 2. 按模板填写
#    - 元信息（功能名/作者/状态/风险档位）
#    - 背景与目标（背景/目标/非目标）
#    - 核心规则（R-01 起，每条 8 个维度）
#    - 技术方案
#    - 风险评估
#    - 工作假设

# 3. 提交
git add design/新系统名Design Doc.md
git commit -m "Design: 新系统名"
git push
```

### 场景 C：改配置表（走 Proposal→Task 流程）

```bash
# 1. 写 Proposal
copy proposals/TEMPLATE.md proposals/改动名-YYYY-MM-DD.md
# 填写：背景、影响范围、落地方案、风险、验收标准

# 2. 写 Task
copy tasks/TEMPLATE.md tasks/改动名-YYYY-MM-DD.md
# 拆成子任务步骤

# 3. 改表（在 D:\Vgame\Config\GameConfig\Datas\ 下的 Excel）

# 4. 改完后更新知识图谱（如果新增/删除了表或字段）
双击 knowledge-graph/更新知识图谱.bat

# 5. 更新 Task 状态

# 6. 提交
git add proposals/ tasks/
git commit -m "Task: 改动名已完成"
git push
```

### 场景 D：让 AI 帮你做事

如果你用 OpenCode，AI 会自动加载 `skills/` 下的 Skill。你只需要：

1. 告诉 AI 你想做什么（比如"帮我查 Drop_mainlevel 的字段"或"帮我写一份新系统设计"）
2. AI 会自动读 `AGENTS.md` → 路由到对应 Skill → 读知识图谱 → 输出结果

**不需要**手动告诉 AI 去读哪个文件，`AGENTS.md` 已经定义好了路由。

## 常见问题

**Q：我改了配置表，需要更新知识图谱吗？**
A：只有新增/删除表或字段时需要。只改数值不需要。

**Q：知识图谱过期了怎么办？**
A：双击 `knowledge-graph/更新知识图谱.bat`，它会重新读取所有配置表生成最新结构。

**Q：AI 不知道我改了什么怎么办？**
A：确保你改的配置表在 SVN 里，AI 会通过知识图谱读取最新结构。如果 AI 用的是旧数据，跑一次 `更新知识图谱.bat`。

**Q：我想新增一个系统，从哪开始？**
A：先写 `design/新系统名Design Doc.md`（用 TEMPLATE.md），再走 Proposal→Task 流程。

**Q：Proposal 和 Task 有什么区别？**
A：Proposal = "改什么 + 影响什么 + 风险"（给人看的）。Task = "拆成哪些步骤"（给执行用的）。

**Q：我只想改一个数值，也要走完整流程吗？**
A：不需要。小改动直接改表 + commit message 记录就行。只有跨系统、影响大的改动才需要 Proposal→Task。

**Q：怎么知道哪些列能改、哪些不能改？**
A：查 `config-schema.json` 里的 `formula_columns`（不能改）和 `safe_write_columns`（能改）。或者直接问 AI。

**Q：我想给其他项目也用这套框架怎么办？**
A：`通用项目skill/` 目录下有通用模板，复制后填入新项目的具体信息即可。
