# Security And Safety

## 不提交的信息

- 私钥、账号、Token、Cookie、内部登录信息。
- 未脱敏的玩家数据或运营数据。
- 真实配置表的临时导出缓存。
- 本地分析过程中的大体积中间文件。

## 操作安全

- 修改 `${VGAME_ROOT}` 真实配置前，必须确认任务明确要求写入。
- 跨仓改动必须记录在 `CROSS-REPO-CHANGES.md`。
- 任何脚本默认只读，除非文件名、注释和使用说明明确写出会写入。
- 发现配置链不一致时，先输出审查结论，不直接修表。

## Agent 安全规则

- 不凭空补字段含义；字段含义来自表头、已有 skill、真实引用或人工确认。
- 不把经验权重说成事实。
- 不把历史草稿玩法当作正式线上规则。
