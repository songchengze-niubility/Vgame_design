# Proposal: 主线首通金币翻倍

**提案编号:** PROP-20260619-01
**日期:** 2026-06-19
**作者:** 反推整理
**状态:** 草案
**相关 Design:** [主线关卡系统Design Doc.md](../design/主线关卡系统Design Doc.md)

## 1. 背景与目标

当前主线首通金币量偏低（Ch0普通5000→Ch4普通25000），相比资源副本日常产出（日均30,000-50,000金币），主线一次性首通奖励的激励感不足。本次调整为所有主线章节首通金币翻倍，提升前期推关的正反馈。

## 2. 影响范围

| 模块/表/skill | 影响 | 说明 |
|---|---|---|
| `Drop_dungeon.xlsx` | 修改 | 主线首通 Drop 条目中金币数量翻倍 |
| `UIlevel.xlsx` | 不变 | 奖励预览字段不需要改，DropId不变，数值通过Drop表更新 |
| `level.xlsx` | 不变 | FirstRewardDrop 引用不变 |
| `Vgame经济框架.xlsx` | 需同步 | 主线副本 sheet 中的金币列需更新 |
| 玩家前期金币收入 | 大幅增加 | 60关首通金币总量翻倍，约 +120万金币 |
| `vgame-player-progression-simulator` | 影响 | 模拟器读 Excel 会自动获取新值 |

## 3. 落地方案

### 3.1 设计变更

| 章节 | 普通关(改前→改后) | Boss关(改前→改后) |
|---|---|---|
| Ch0 | 5,000 → 10,000 | 10,000 → 20,000 |
| Ch1 | 10,000 → 20,000 | 20,000 → 40,000 |
| Ch2 | 15,000 → 30,000 | 30,000 → 60,000 |
| Ch3 | 20,000 → 40,000 | 40,000 → 80,000 |
| Ch4 | 25,000 → 50,000 | 50,000 → 100,000 |

### 3.2 配置变更

修改 `Drop_dungeon.xlsx` 中对应 DropId 的金币数量字段：

| 章节 | 涉及 DropId 范围 | 需修改字段 | 操作 |
|---|---|---|---|
| Ch0 | 20001001-20001005 | DropItemNumMin / DropItemNumMax | 翻倍 |
| Ch1 | 20001006-20001010(普通) + Boss | 同上 | 翻倍 |
| Ch2 | 20001011-20001025 | 同上 | 翻倍 |
| Ch3 | 20002001-20002015 | 同上 | 翻倍 |
| Ch4 | 20002016-20002030 | 同上 | 翻倍 |

**安全性**：这些 DropId 条目对应的 DropItem_id=1（金币），DropItemNumMin/Max 是纯数据列，无公式，可安全写入。

### 3.3 Skill 或文档变更

- `vgame-player-progression-simulator`：运行 `--from-excel` 会自动读取新值，无需代码变更
- `design/主线关卡系统Design Doc.md`：更新 R-02 中的金币数值表

### 3.4 验证方式

1. 改表前备份 `Drop_dungeon.xlsx`
2. 修改后跑 `更新知识图谱.bat` 更新 config-schema.json
3. 跑 `simulate_progression.py --days 30 --scenario f2p_active --from-excel`，对比改前后 Day 30 的金币存量差异
4. Luban 导出验证（程序协助）

## 4. 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|---|---|---|---|
| 写错 DropId 条目（改了非金币条目） | 低 | 中 | 逐一确认 DropItem_id=1 的条目 |
| 前期金币过量导致经济通胀 | 中 | 中 | 翻倍后总量从 ~120万 → ~240万，仍在单角色升级（96万）+ 5角色（480万）范围内 |
| 公式列被误写 | 低 | 高 | 写入前确认 DropItemNumMin/Max 不在 formula_columns 中 |
